# 我是克里斯蒂安·雷柏，这是《神童》背后的故事

> 原文：<https://lifehacker.com/im-christian-reber-and-this-is-the-story-behind-wunde-1526022898?utm_source=wanqu.co&utm_campaign=Wanqu+Daily&utm_medium=website>

多年来，我们一直是待办事项应用 [神童](https://lifehacker.com/wunderlist-is-a-simple-elegant-and-free-to-do-app-tha-5687089) 的粉丝。它从一开始就以其极简、易用的设计赢得了我们的青睐， [每次更新](https://lifehacker.com/wunderlist-2-brings-recurring-items-shared-to-dos-not-5969253) 都带来了更好的功能:共享待办事项、电子邮件通知、重复任务等等。我们不是它唯一的粉丝:Wunderlist 是苹果 2013 年的年度应用。我们采访了创始人兼首席执行官 Christian Reber，了解我们最喜爱的应用程序之一是如何实现的。

#### 这款应用的创意来自哪里？你是在试图解决你经历过的问题，还是灵感来自其他地方？

在我们开始 [6Wunderkinder](http://www.6wunderkinder.com/en/) 之前，我和我的联合创始人已经在一起合作做一个网络代理。我们在旅途中有多个客户和项目，因此我们需要一个可靠的项目管理工具。我们尝试了几乎所有现有的解决方案，但我们找不到任何我们想继续每天使用的东西。

我们所有可用的解决方案都有非常具体的问题——复杂性、缺少移动和桌面应用程序、不同步——所以我们决定只开发自己的产品 Wunderlist。我们想创造一个产品，满足像我们一样的人的需求——一个美丽而简单的应用程序，帮助你完成工作，无论是在你的业务还是个人生活中。

#### 你想出这个主意后，下一步是什么？

我们没有真正的想法，但我们有一个具体的问题要解决——我认为这是最初的成功因素。Wunderlist 是试图通过构建一个简单、直观、美观和快速的产品来解决这个问题的结果。

我们开始建造原型，并与他们联系了各种投资者。我们很幸运地吸引了天使投资者(高净值个人，愿意投资他们的个人资金来建立新的企业)的注意，两个月后，我们推出了 Mac 和 Windows 的 Wunderlist，随后是 iPhone、iPad、web 和 Android。由于我们在发布期间取得的成功，国际投资者联系了我们，然后我们能够完成对 Atomico(Skype 背后团队的一部分)的投资，以确保 Wunderlist 的持续成功。

#### 你是如何选择目标平台、忽略或等待平台的？

一开始我们没有。我们有一个明确的目标，那就是支持尽可能多的平台，甚至是黑莓、Linux 和 Windows Phone——我们有一个适用于几乎所有可用平台的应用程序。

老实说，我们当时没有意识到，拥有大量的应用程序会极大地降低开发时间。如果你在你的架构上做了一个改变，它会影响到你所有的客户端应用——和只有一个 web 应用完全不同。因此，为了变得更快，我们决定只支持增长最快和最受欢迎的操作系统——iOS、OS X、安卓、Windows——和 web(包括 Chrome OS)。

目前，我们正在对《神童名单 3》进行大规模改写。此次更新不仅支持更多的平台(我们将很快针对 Windows 平台发布一些令人兴奋的公告)，我们还从设计和技术角度对整个产品进行了彻底的重新思考。

在这个过程的开始，我们问自己，我们如何继续构建高质量的软件，同时支持尽可能多的平台。在接下来的几个月里，我们将发布新版本，这对我们来说将是一个巨大的进步，并将把 Wunderlist 带入每个人的日常生活。我们正在谈论一个全新的设计，一个更轻的界面，实时同步，最后，一个公共 API。我们不希望 Wunderlist 仅仅是一个应用程序。

#### 你遇到的最大障碍是什么，你是如何克服的？

哦，那很容易。到目前为止，专注是最大的障碍。我认为这也适用于大多数创业公司和企业。一旦你成功了，你就试图通过做更多的事情来发展它，在我们的情况下，构建更多的功能，支持更多的平台，我们甚至尝试构建更多的产品(你们中的一些人可能还记得 Wunderkit)。我知道“注意力不集中”听起来非常明显，但是如果你每天都在做一个产品，实际上很难把注意力集中在重要的事情上。

更重要的是，当你的团队增长时(我们现在超过 50 人)，将所有的小部分连接起来，专注于构建一个极其完美的产品变得非常困难。随着越来越多真正有技能和雄心的人加入，每个人都想以自己的方式做出贡献(这通常意味着建立自己的特色)。因此，作为一个团队，我们需要不断地重新聚焦，以确保我们所有人都能在所有平台上提供一致的产品体验。

毫无疑问，推出第二个产品是我最大的障碍，但也是我一生中最重要的学习。每当我们为任何事情挣扎时，那次经历帮助我重新聚焦和思考什么才是真正重要的。

#### 对你来说，发布会是什么样的？

这是一个很好的问题。我经常把打造一个产品比作打造一个足球俱乐部。你选择你的球员，你努力训练，你参加比赛，然后你赢或输。对一个公司来说也是一样，你雇佣你的团队，你极其努力地构建第一个版本，你发布(这通常是压力最大的时候)，然后你要么成功，要么失败。

发布产品是最激动人心的时刻，它非常有活力和激励性(如果事情进展顺利的话)。但它也非常紧张、有压力、艰难。足球俱乐部有粉丝，创业公司有粉丝。你从外面看到的，很少能反映出里面实际发生的事情…但是，准确地说，我喜欢它。我真的很喜欢设计产品和开发应用程序。

#### 你如何有效地处理用户的要求和批评？

我们有一个 [功能请求论坛](http://wunderlist.uservoice.com/forums/136230-wunderlist-feature-requests/filters/top) ，在这里我们的社区积极提交他们的想法来改进神童列表。然后，功能请求会提交给整个社区进行投票，因此我们可以快速了解大多数用户希望我们构建什么。然后，这些特性请求成为产品路线图的一部分。

我们也阅读每一封电子邮件、推特和脸书发给我们的信息。我认为我们很清楚用户对我们的期望，我相信《奇迹 3》会反映出用户的需求。

#### 现在，你如何在开发新功能和管理现有功能之间分配时间？

现在，我们 100%的精力都放在了神童 3 的重大更新上。我们将 Wunderlist 3 视为 Wunderlist 的新核心，为我们的用户定期带来令人兴奋的新功能奠定了基础。

我们管理新功能的方式很简单，我们总是选择最重要的一个，并启动它。然后我们选择下一个，下一个，很明显。现有的要困难得多，因为它更多的是关于抛光，或者改变架构。改变现有的功能通常需要更多的工作，而且直接的结果还不清楚。如果我们发现某个特性没有按照我们想要的方式工作，那么我们就改变它。但我们不断完善神童列表中的每个功能。

#### 你会给那些想参与类似项目的人什么建议？

总的来说，解决你自己的问题。那对我来说总是有效的，我只制造我自己非常想用的产品。但是，这也取决于我给建议的人的背景。我是程序员和设计师的混合体，这帮助我理解和连接两个世界。

打造产品需要激情和耐心，如果你不喜欢，就不要做。如果你决定这样做，那么总是试着先做一个你能做的最简单的产品。在功能的优先化和设计的简化上要残忍。然后制造一种能给人们的生活增加价值的产品，忽略趋势。
