# 软件的供应链——增量:API

> 原文:[https://increment.com/apis/apis-supply-chain-software/?UTM _ source = Wanqu . co&UTM _ campaign = Wanqu+Daily&UTM _ medium = website](https://increment.com/apis/apis-supply-chain-software/?utm_source=wanqu.co&utm_campaign=Wanqu+Daily&utm_medium=website)



三十年前，世界上软件公司的数量很少——想想微软、SAP 和甲骨文——他们编写了组成他们产品的每一个软件。没有构建软件的供应链。然而，随着公司开始发展、繁殖和竞争，行业也随之发展。而且每个成熟的行业都有供应链。

以生产实物商品的行业为例。由于来自底特律，我的脑海中浮现出汽车行业。汽车制造商不会自己制造汽车的每一个零件。他们从钢铁公司购买钢材，从座椅公司购买座椅，从速度计公司购买速度计。所有那些在高速公路上从你身边疾驰而过的凯美瑞和 F-150 皮卡都含有由几十家，也许是几百家主要供应商提供的零部件。这些供应商反过来又从全球供应链上数百甚至数千家较小的零部件制造商那里获得支持。随着行业的成熟，它们的供应链也在成熟。随着越来越多的公司专注于该过程的特定部分，整个行业可以变得更加高效和多产。

在过去的 10 年里，随着软件业的成熟，它已经发展了自己的供应链。现在每个公司在某种程度上都是软件公司，大多数公司都不能从零开始建立一切。就像福特和丰田一样，他们需要一个供应链，将行业划分为专业领域，并允许每个参与者专注于一项核心能力。

但是软件供应链看起来不同于汽车行业。软件供应链公司不是专注于速度表或方向盘，而是交付可重用的代码块，开发人员将这些代码块拼凑在一起，制作出完整的应用程序。这些是 API。亚马逊网络服务提供数据中心。Twilio 提供通讯。Stripe 和 PayPal 支持支付。现代应用程序将许多这些组件集成到一个独特的客户价值主张中。

API 是新的供应链，使每个公司都能够成为软件公司，而无需重新发明轮子。欢迎来到 API 经济。

## 我们是怎么到这里的？

API 经济的真正先驱是亚马逊。21 世纪中期，亚马逊决定租赁计算能力和存储容量，而不是作为应用程序，而是作为开发人员和其他公司可以用来构建应用程序的构建模块。这项服务将是灵活的，能够随时扩展和缩小。你只需为你所使用的东西付费，就像你的手机和电费一样，每月都会收到账单。这就是后来的亚马逊网络服务。

现收现付模式是一个巨大的突破，也许和技术本身一样重要。AWS 将创办一家新科技公司的成本降至几乎为零。在 AWS 之前，您需要购买昂贵的服务器、存储系统、路由器和数据库软件。你可能会花 100 万美元购买和安装硬件，仅仅是为了支持你的想法。有了 AWS 和它的现收现付模式，一个企业家可能只需要花 100 美元，在几分钟内就可以上线——这是填写一张表格和输入信用卡号码所需要的时间。

低创业成本的结果是更多的创业公司——那些可以更快进入市场的公司。他们可以在几个月内从餐巾背面(或者在 Twilio 的情况下，比萨饼盒子的背面)勾勒出的想法变成发货。他们可以在没有任何基础设施摩擦的情况下扩张和成长。他们可以快速移动并建造东西。

## API 经济中的生活

如果你看看构成 API 经济的公司，他们都采用了这种模式。他们解决了一些真正棘手的问题，花了几年时间编写解决方案，现在向其他人提供这种服务。对于包括 Twilio 在内的许多人来说，这项服务本身就是一个黑盒子。客户不知道也不关心它是如何工作的。他们只是把我们的代码插入他们的，然后就走了。

API 经济催生了新一代创业公司，它们比传统公司更快、更精简。这些敏捷的新贵颠覆了各行各业的公司，迫使更大的传统公司开始开发新软件以跟上时代，进一步加快了创新的步伐。

API 是开发人员的新构件。在软件供应链成熟之前，构建的提升是如此之大，以至于购买一个解决方案往往更容易。为了开发软件，你必须像微软或甲骨文一样精通软件。API 使得开发应用程序变得更快更便宜。它们可以扩展到支持数十亿用户。最重要的是，它们降低了建筑商的准入门槛，让各行各业的每家公司都可以进行试验。这在 10 年前是不可想象的。

我相信这种向组件软件的转变是行业发展的下一个大飞跃。这是下一个伟大的软件时代——继 SaaS 之后的时代。
