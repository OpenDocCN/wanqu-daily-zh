# “无代码”的兴起

我用 Dreamweaver 建立了我的第一个网站，这是一个所见即所得的网站生成器，在我十几岁的时候就开始流行了。我为我的创作感到骄傲。很丑，但我做到了。虽然我的傻网站没有像我梦想的那样成功，但它是在互联网上创造事物的入门药物。

Dreamweaver 和类似的工具(FrontPage、Flash 等。)极大地降低了创作的障碍，尤其是对我这样一个当时才 15 岁的高中生来说，我对 HTML 和 CSS 只有非常基本的了解。它们对今天的技术生态系统的影响被低估了，今天我们看到了新一波工具，这些工具使创作变得更加容易，并彻底改变了为互联网构建事物的方式。

## 每个人都是创造者

从前，只有一小群软件工程师——一些自称的网站管理员——在互联网上构建东西。这些雄心勃勃的人在发布哪怕是最基本的网站之前，都会花上几个月的时间学习编码。



今天，任何人只要有一台电脑并能接入互联网，就能使用比 20 年前的 Dreamweaver 强大得多的工具来建立一个网站。但是这些基于 GUI 的工具已经远远超出了静态站点，扩展到了全功能的应用程序。不到一个小时，您就可以创建:

还有“无代码”资源，如 [Zeroqode](https://zeroqode.com) 、 [NoCode](https://www.nocode.tech/) 和 [MakerPad](https://www.makerpad.co/) 。

这些工具减少了将想法转化为人们可以使用的东西所需的时间和编码专业知识。你不再需要成为一名程序员来在互联网上构建东西，从而为来自不同背景和视角的新一波创客提供支持。

## 批评家们

可以预见，许多人会批评和评判那些使用“无代码”工具的人。虽然它们带来了权衡，但不可避免的是，更多的产品将在不编写代码的情况下被构建——或者至少是 MVP 包括由能够编写代码的程序员编写的产品。

我最近和一家 A 轮创业公司的副总裁聊过，这家公司使用 Salesforce 和 Zapier 寻找 B2B 销售线索。当然，该公司有一个工程团队，可以构建他们想象的任何东西，但他选择使用“无代码”工具，因为这样构建更快，维护更容易。

我们常常认为很难做的事情做得更好。有时候确实如此。但是随着这些“无代码”工具的进步，用老方法做将是愚蠢的。

托管基础设施已经经历了这种转变。要把一个网站放在互联网上，人们必须购买昂贵的物理服务器，并在开发运营上投入大量精力(有时是压力)。如果雅虎把这个网站放在主页上，它很可能会因为意想不到的流量泛滥而崩溃。

现在，我们有 AWS、Heroku、Google Cloud 和其他解决方案来简化这一过程。即使是最有经验、最有才华的开发运营工程师也使用他们的服务来起步和扩展。对他们来说，把时间花在自己的服务器上是愚蠢的。

## 制作的未来

随着在互联网上创造东西变得越来越容易，更多的人将成为创造者。它不再局限于<1% of engineers that can code resulting in an explosion of ideas from all kinds of people. We see “no code” projects on Product Hunt often, including [本届金猫奖得主](https://www.producthunt.com/posts/diversify-tech)。

五年前，当我们开始寻找产品时，这种趋势是主题的一部分。随着越来越多的人成为创客，人们更加渴望社区在他们的旅程中提供反馈和支持。[2]

随着每个人都成为创客，创客社区将会越来越受欢迎。帮助制造商赚钱、寻找项目伙伴和传播信息的工具也是如此。这些创造性的解决方案可能不用写代码就能构建出来。
