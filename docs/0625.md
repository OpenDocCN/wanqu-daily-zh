**即将到来的比赛**
随着苹果手表发布的临近，我们看到媒体报道达到了疯狂的速度——报道了产品、[手表的设计师](http://www.newyorker.com/magazine/2015/02/23/shape-things-come)、销售[预测](http://www.reuters.com/article/2015/02/17/us-apple-watch-orders-idUSKBN0LL18W20150217)和[零售展示](http://9to5mac.com/2015/01/31/apple-stores-to-be-outfitted-with-safes-to-house-gold-apple-watches/)。作为消费者，这对我们来说很有趣。但是对于我们这些从事新产品开发的人来说，更大的消息是*我们有了一个可以玩的新平台！*

Apple Watch 的推出将创造一个机会来开发第一个“手表优先”的杀手级应用程序，如果成功，它可能会创造新一代的应用程序和创业公司。

为什么新平台如此重要——劣质点击率定律
普通读者会知道我对新平台着迷不已。原因是因为[劣质点击率](http://andrewchen.co/the-law-of-shitty-clickthroughs/)定律，该定律声称，在竞争、垃圾邮件和客户疲劳的驱动下，任何渠道的总体表现都会随着时间的推移而下降。

当你有一个大的新平台时，你可以避免所有这些。因此，每一个新平台通常都会催生一批价值数十亿美元的公司也就不足为奇了。在移动领域，有优步、Whatsapp、Snapchat 等。借助脸书平台，我们看到了 Zynga 等社交游戏公司的崛起。在网络方面，我们经历了网络泡沫。以 Apple Watch 为首的可穿戴设备也很有可能有那么大。

有了 Apple Watch，我们有了新鲜的雪:

*   在发布之后，会有一段试验和新奇的时期，人们会兴奋地尝试新的应用程序，不管它有多琐碎
*   来自技术和主流媒体的兴奋，这将宣传所有添加集成的大应用程序
*   一个围绕着通知和“T0”眼神互动而构建的设备，加上新奇的效果，将导致参与率高得离谱
*   应用商店将会以巧妙的方式推广与手表整合的应用
*   健康、支付、新闻等方面独特的 API 和场景。，导致这些类别中的创造性新应用

与此同时，竞争将会减少:

*   许多应用会对平台采取“观望”的态度
*   一些团队根本不会尝试 Apple Watch，因为将他们的应用程序的价值塞进可穿戴设备的格式并不容易——例如，你不能随便在上面塞任何游戏
*   关于入职、成长和敬业度的最佳实践仍有待发现，因此新人发现的可能性更大

上述动态意味着手表发布会将导致一些令人兴奋的结果。苹果公司在推出新产品方面考虑周到，非常挑剔，所以对于手表，我们知道他们会付出真正的努力和营销能力。结合传闻中每月高达数百万台的销量，你可以想象高价值用户的临界质量正在迅速形成。

**Apple Watch 将增强通知功能**
阅读 HIG 最大的好处之一就是通知用户界面的重要性。虽然你可能会发现自己漫不经心地浏览 Glances UI 来查看发生了什么，但似乎最常见的交互方式之一是获得通知，在手表上查看，然后从那里采取行动。这将是许多订婚戒指的核心。

出于这个原因，苹果设计了两种风格的通知——一种是新通知的摘要“简短外观”,另一种是实际上与多达 4 个操作按钮交互的“长外观”。下面是一个长期待通知:

[![longlook_calendar_2x](../Images/07438ee83c597b8292e3b47d9ada2e7e.png)T2】](http://andrewchen.co/wp-content/uploads/2015/02/longlook_calendar_2x.png)

因为查看手表的通知非常容易，而且你会一直带着手表，所以我认为我们会看到 Apple Watch 通知的表现比推送通知好得多。结合发布前后的新奇期，我认为我们会看到集成 Watch 的应用程序的保留率、参与度和使用率更高的报告，这些案例研究将推动更多开发者采用。

等待手表优先的杀手级应用
成为手表优先的应用仍然是一个引人注目的思想实验。我们看到，在智能手机出现几年后，“为什么这个应用程序只适用于移动设备？”是一个重要的问题。

基本上是现有网站的端口的应用程序最终都成了垃圾——塞满了功能，呈现出比仅仅使用网站更糟糕的体验。相比之下，突破性的移动应用程序利用了内置摄像头、永远在线的互联网、位置或其他可用的 API。换句话说，许多“X 的优步”都失败了，因为叫出租车需要不断地在新的/未知的地点消费服务，并且这种消费的频率足够高。不是每个 web app 都应该是手机 app。同样，手表最初发布时的大多数应用程序可能会认为这只是一种更花哨的方式，用来显示烦人的推送通知，并推动现有 iPhone 应用程序的使用。

更吸引人的问题是，在 iPhone 应用程序交互最少的情况下，什么应用程序本身会在手表上引起高参与度？这就是手表优先的杀手级应用的样子。我怀着激动的心情等待着这个行业解决这个问题。
