# 我们作为开发团队升级的 4 种出色方式

格兰特·阿蒙斯

作为一名技术领导者，你应该有一个主要的目标，那就是帮助你的团队提高能力。

为了做到这一点，你需要有意识地减少工程师的时间，让他们远离实际工作，专注于学习。故意的。知识工作者需要认可的时间来发展自己，他们必须在工作中感到舒适。

俗话说，

> 首席财务官问首席执行官:如果我们花钱培训员工，然后他们离开了，会发生什么？CEO: **如果我们不走，而他们留下来，会发生什么？**

当然，你的工程师会留下来，因为你关心他们的幸福。以下是你的团队作为一个小组一起学习的 3 个极好的方法。预先警告:这些都需要时间和努力，主要是你自己。

# 1.主持常规午餐和学习。

每周召集你的工程师一次，请他们吃午饭，然后一起吃午饭学习。在 [PipelineDeals](https://www.pipelinedeals.com) ，我们每周三从中午到下午 1 点都这样做。我们已经这样做了很多年，而且非常有益。

安排午餐和学习有多种方式，但我们发现非常松散的结构是最容易的。任何需要大量承诺或计划的事情都会很快让你筋疲力尽。

通常我们会以谈论有趣的科技新闻或当前项目发生的事情结束。我们谈论的事情通常分为以下几类:

**时事讨论。**我们经常讨论黑客新闻、播客或其他来源的有趣文章。它促使我们探索新技术，特别是在 Devops 和 Javascript 领域。

**展示和讲述**。有时，工程师会分享他们的屏幕，谈论他们修复的一个特别棘手的问题，展示一个很酷的库，或者演示一些有用的东西。这些都是没有计划的和非正式的，而且总是会导致一场大讨论

**追溯流程**。有时，我们会讨论一个过程以及如何使它变得更好。上周，我们就招聘流程进行了一场精彩的讨论，讨论得出了许多有用且可行的信息。

**一般戏谑**。我们的一个队友对桌游有一种不健康的痴迷，所以我们总是喜欢听他在玩什么，他刚刚订购了什么游戏。另一名工程师非常喜欢 MMA 格斗，并编写了一个机器学习算法来预测格斗的获胜者。谈论这类话题真的很有趣，它只会加强工程师之间的联系。

举办这些活动需要纪律和毅力，但好处很多:

**远程开发者面对面时间**。我们公司几乎所有的工程师都是远程的。一个大家聚在一起聊天的固定场所是建立友谊的好方法。从某种意义上来说，这是一个虚拟的饮水机。

它促进学习。我总是带着新的知识，或者对我走进来之前不知道的东西的好奇心离开。

**社交的时间**。记得小学食堂吗？你和你的朋友坐在一起，放松一下，聊聊天。你可以从繁重的课程中休息一下，花些时间去社交。这就是我们午餐和学习的精神所在。

# 2.一起看视频。

一起观看会议演讲或教学视频，然后进行讨论是一种很好的共同学习方式。

这可以亲自完成，也可以通过 Hangouts 这样的远程工具来完成。观看视频的一个好处是可以边看边聊天。我们一起在 YouTube 上看会议视频的时候，看到开发者在群聊里写评论很有意思。它在实时观看一个“啊哈”时刻！

我*高度*推荐两个视频系列。

**看鲍勃大叔的《干净的编码者》**。*没有什么比这些视频更能提升我的团队和我自己的技能了*。[罗伯特·c·马丁](https://twitter.com/unclebobmartin)(“鲍勃大叔”)是你职业生涯中从未有过的老迈、白发苍苍的导师。他写了两本关于这个主题的开创性的书，分别是[干净的代码](http://www.amazon.com/Clean-Code-Handbook-Software-Craftsmanship/dp/0132350882/ref=sr_1_1?s=books&ie=UTF8&qid=1452431645&sr=1-1&keywords=clean+code)和[干净的编码者](http://www.amazon.com/The-Clean-Coder-Professional-Programmers/dp/0137081073/ref=pd_sim_14_3?ie=UTF8&dpID=512NzCU0wfL&dpSrc=sims&preST=_AC_UL160_SR123%2C160_&refRID=1WE2AV5EXA9R8B6RHS1T)。他会教你和你的团队如何正确地编写代码。

鲍勃大叔通过频繁使用傻乎乎的服装，绿色的屏幕，有趣的道具来保持题材不干涩。

可能要花几个月的时间才能看完，而且不一定便宜。但它们物有所值，你的团队将获得的投资回报远远超过成本。他的视频将巩固你在编程或软件架构书籍中读到的概念。

我个人记得一个非常鲜明的感觉，是在看[功能](https://cleancoders.com/episode/clean-code-episode-3/show)视频的中途。我记得我对自己说，“哦，我的天。我这样做错了这么久！”这就是升级的感觉。当你完成这些视频，你会完全不同的编码。

观看加里·伯恩哈特的《摧毁所有软件》截屏。 Gary 的视频短小精悍，充满了惊人的信息。他会教你如何正确地*测试你的代码。我们发现我们不得不多次暂停他的视频，以便让他的课程深入人心。*

*另一方面，加里是一个速度惊人的打字员，真的知道他在维姆周围的方式。我个人从来没见过有人编辑代码这么快。这令人震惊，也是 Vim 力量的真实证明。*

***看一段会议视频**。参加会议既费钱又费时。下一个最好的事情是前往 http://confreaks.tv/的[观看一场精彩的会议视频。不知道从哪里开始？我建议去看看“总是受欢迎的”部分，然后从那里开始。](http://confreaks.tv/)*

# *3.主持一个读书俱乐部。*

*读书俱乐部是一个团队提升技能的好方法。在调查了一些其他的技术领导者之后，我发现大多数团队在技术和领导书籍之间移动。格式通常是:*

*   *定期会面(每周或每月)，30 分钟到 1 小时*
*   *一次讨论一个已知量。对于周刊来说，30-50 页是合适的数量。*
*   *讨论技术性和非技术性(商业或领导力)书籍。*
*   *有些有一个“书籍管理员”，他收集谈话要点并引导讨论。其他人保持更自由的形式，每个人都带来讨论点。*
*   *如果大家一致认为对话不会有成效，可以随时取消会议。*

*99designs 的读书会形式取得了成功，推荐阅读。*

## *我的团队可以担保的书籍:*

*务实的程序员是成为更好的开发人员的极好资源。它教你如何通过设计模式、更好的开发过程和陷阱来编写更简洁的代码。迪夫·托马斯和 T2【安迪·亨特都在软件开发行业有几十年的经验，并将他们学到的经验浓缩到了这本书里。*

*桑迪·梅斯在教学效率方面有着惊人的技能。她因给开发者的 [4 条“规则”](https://gist.github.com/henrik/4509394)而备受推崇，这些都是极好的建议。她的书非常清晰易读，你会对面向对象设计有更深的理解。尽管这本书关注的是 Ruby 语言，但是这些概念适用于大多数语言。*

*这本书非常容易阅读，将让你和你的团队熟悉所有主要的设计模式。尽管这本书主要是用 Java 编写的，但是这些模式适用于大多数过程语言。当心——读完这本书后，一切开始看起来像一个模式。抵制到处贴模式的冲动。在使用一个模式之前，向他人证明你的推理是正确的。*

## *优秀策划书单:*

*德里克·西弗斯的书页是一份令人惊叹的商业书籍清单。它也不仅仅是一个列表。德里克对他读过的每本书都有详尽的评论。他的网站是寻找最佳非技术性商业书籍的绝佳资源。*

*杰夫·阿特伍德无需介绍。他整理的编程书籍列表是寻找优秀编程书籍的绝佳起点。*

# *4.在当地集会和会议上发言*

*鼓励你的团队成员参加聚会和会议或在会上发言是促进学习文化和建立团队品牌的一个很好的方式。你需要带头树立榜样。*

*许多开发人员都很内向，在众人面前讲话可能是他们最不想做的事情。然而，这是一项重要的职业技能。他们的级别越高，就越有可能需要在大群人面前讲话。*

*发表演讲迫使工程师深入了解某一主题。他们将很容易成为该领域的专家。*

*扎克·霍尔曼有一个很棒的网站，当你需要在众人面前演讲时，它有很棒的资源。他介绍了如何准备您的演讲、幻灯片设计，以及如何为这个重要的日子做准备。强烈推荐阅读！*

*我发现的另一个很好的资源是杰夫·莱克的，他给出了一些关于演讲的很好的建议。*

# *结论*

*就是这样！以上 4 种方法将帮助你的团队更上一层楼。以上为我的团队创造了奇迹。*

*给你的开发人员时间来完善他们自己，你会对结果感到惊讶。*
