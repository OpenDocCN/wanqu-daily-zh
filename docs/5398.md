# 公司内部电子邮件营销和通知的演变

> 原文:[http://caseyaccidental . com/email-marketing-notifications-evolution/？UTM _ source = Wanqu . co&UTM _ campaign = Wanqu+Daily&UTM _ medium = website](http://caseyaccidental.com/email-marketing-notifications-evolution/?utm_source=wanqu.co&utm_campaign=Wanqu+Daily&utm_medium=website)

“不要像营销人员一样发送电子邮件。开始像私人助理一样发送电子邮件。”

我在很多关于增长的演讲中都提到了这一点，但我并没有深入讨论如何实现这一点。公司在发展电子邮件和通知方面有一个明显的演变，从根本不发送电子邮件和通知，到向所有受众发送一次性消息，再到创建一个生命周期，再到拥有一个全面的个性化消息平台。在我最近的两家公司工作过这些系统之后，我认为在流程的早期为人们概述这些转换是有益的。

第 0 阶段:我们讨厌电子邮件

“我们讨厌电子邮件，所以我们不会把它发给我们的客户。”

几乎我工作过的每家公司都在早期的某个时候沟通过这一点，但总是错的。除了非常特定的群体，人们并不反感电子邮件；他们不喜欢糟糕的电子邮件。判断你的客户希望收到什么样的电子邮件的方法主要是问他们在我的产品中看到了什么样的价值，以及我能否以电子邮件的形式传递这种价值。

**第一阶段:群发促销邮件+个性化订单通知**
如果你是交易型服务，你必须发送个性化订单通知，所以大多数公司都是从这里开始的。之后的某个时候，公司开始向更广泛的受众发送大量电子邮件，内容涉及新功能、折扣等。这项工作将显示关键指标的改进，但它非常简单。

促销训练用户等待促销订单，降低盈利能力。此外，使用这种方法，营销是假设客户的步调，而不是适应客户的步调(并最终改善客户步调以增加终身价值)。

如果您受到工程设计的限制，可以对这种方法进行一些简单的优化，以提高您的性能:

*   开发额外的电子邮件，旨在推动习惯的形成(而不仅仅是及时购买)。示例包括趋势商品、商品销售、新增商家、推荐商品
*   如果电子邮件是成功的，测试他们作为推送通知
*   利用现有的确认电子邮件，并在其中添加营销信息(其他要购买的物品、重新订购、最受欢迎的商品等)。)
*   发送每封非事务性电子邮件，并调整确认电子邮件，作为对启用组和对照组的实验，以证明对生命周期价值的影响以及对退订/推送权限/应用程序删除的影响

**第二阶段:转向生命周期信息传递**
在第二阶段，这些在第一阶段获得成功的额外电子邮件和通知开始形成一个自动化程序，持续推动客户的额外参与。为了解决消息传递疲劳问题，这些电子邮件和通知模板根据团队对好客户的预期价值每月进行一次管理。这个频率不是基于数据，但如果每个人都正确使用产品，理想的频率会是什么样子。目标是使用信息提醒人们这项服务并强化习惯。它们配有个性化的折扣电子邮件，旨在推动新的使用案例并增加使用频率。

这些电子邮件和通知模板也是相互管理的，因此消息传递不会过时。例如，如果您在确认电子邮件之外有三个模板，并且您上周发送了模板 1，您将尝试在再次发送模板 1 之前发送模板 2 和 3。此外，每周，这些电子邮件都有新的主题行来呈现它们，看起来与前几周的电子邮件一样。

第三阶段:全面的个性化信息传递
随着第二阶段的方法在它所能带来的额外影响方面趋于平缓，公司转向更全面的个性化系统。这是一笔可观的投资，是我们在 Pinterest 做的。本质上，产品和工程为每个客户确定:

*   正确的内容
*   发送的正确时间(星期几和时间或日期)
*   正确的数量(发送多少邮件和推送)
*   正确的渠道(电子邮件、推送或两者兼有)

这需要一个团队记录发送给订户的每封电子邮件/推送，包括发送时间、打开时间、点击时间、点击导致的下游参与以及模板。所有电子邮件和通知都通过与产品变更相同的实验仪表板运行，以了解对所有关键指标的影响。由此，它需要确定:

*   向每个用户发送消息的最佳日期和时间
*   根据历史点击率和/或购买率确定要发送的模板的优先级
*   一般时间范围内的消息数量最大化每个用户的生命周期价值

这通常从基于规则的方法开始，最终由机器学习提供动力。如果您缺少足够的用户历史数据，例如新用户，您可以将以前看起来像这些用户的人(即以前的新用户)分组，并为他们寻找最佳表现方法。在这一点上，电子邮件不再被认为是营销。它被认为是核心产品的延伸。

该团队还开始通过选择更好的消息传输代理合作伙伴和对不同模板的 IP 地址进行分段来隔离问题，从而优化传送能力。该团队还可能开始投资更高级的安全措施，如 DMARC。

这是一项相当大的投资，这就是为什么大多数公司只在每天发送数百万封电子邮件时才开始建立这种关系，这些电子邮件有很多来自最初两个阶段的运营历史。在这一点上，公司知道电子邮件的价值，并可以证明投资的合理性。

—

在我看来，每个公司都应该在某个时候进入第三阶段。问题是到达那里需要多长时间。这取决于工程约束、规模以及电子邮件和通知在之前阶段的额外参与方面趋于平稳所需的时间。将这项工作外包给营销技术公司也很成问题，因为它需要访问您的所有用户数据，并且数据在系统之间的任何迁移都会降低性能。在一定规模上(像 Pinterest)，甚至是不可能的。

如果你没有达到 Pinterest 的复杂程度，不要沮丧。很少有公司是这样的。只要开始考虑长期发展，以及什么时候是推动电子邮件和通知性能与持续优化相比发生重大变化的合适时机。从一个阶段转移到另一个阶段是一项巨大的投资，但回报通常是值得的，这些电子邮件和通知在当前阶段的影响，以及改善其性能的努力，应该是推动决定进行额外投资以进入下一个阶段的原因。
