# 几乎每个公司 95%以上的时间都在做其他公司做的事情。Auren Hoffman 的总结

几乎每个公司 95%以上的时间都在做其他公司做的事情。它花在公司独有的事情上的时间不到 5%。

那没有意义。

理想情况下，它应该翻转。你的公司应该把大部分时间花在公司特有的事情上。

疯狂的思想实验:想象一个新型的公司，它决定只做自己真正擅长的事情，基本上把其他事情都外包出去。T3】

**招聘即服务？**

几乎每个公司都花大量时间招聘。招聘是一个巨大的时间吮吸。是的，有些公司比其他公司做得更好……但是有一套所有好公司最终都会掌握的最佳实践。

如果你的公司可以将招聘(包括面试)完全外包给另一家服务公司，会怎么样？我知道这看起来很疯狂…但是想象一下如果这真的会发生。想象一下节省的时间。

 **办公空间即服务？**

公司对他们的办公空间考虑很多。随着 flex space(如 WeWork、Knotel 等)的出现，这种情况开始发生变化。但它仍然非常重要。如果你从没想过呢？如果你不担心会议室，食物，音乐等等，会怎么样？如果您将所有这些决策(包括空间大小、位置和成本)外包给一个 API 会怎么样？还是外包服务？

似乎完全疯狂…但是想象一下节省的时间。

**营销即服务？**

营销是每个公司都要做的另一个领域。有些公司真的很擅长营销。有些很一般。一切都可以变得更好。如果你的公司没有人做营销会怎么样？

**金融即服务？**

金融也是。难道财务不能成为众多公司的共享服务吗？真的每个公司都需要立一个财务部吗？除非你是一家银行，否则金融可能不是你工作的核心。

还记得 compute 吗？

虽然这一切看起来很疯狂…但不久前，大多数公司还在管理自己的计算。我在 liver amp——我们拥有 metal，实际上必须处理服务器。工作量很大。现在 LiveRamp 在云中(他们使用 Google Compute ),可以花更多的时间专注于业务。成千上万的其他公司经历了完全相同的过程。

当我们(在 2016 年)开始使用 SafeGraph 时，我们当然决定使用计算即服务(我们使用亚马逊网络服务)。几乎所有在 2013 年之后成立的公司都做出了同样的决定。

我们不外包招聘、营销、财务、办公空间、人力资源等的部分原因。没有很好的服务来做这件事。这在未来可能会改变。当然，有大量的软件工具可以提供帮助(并且公司可以从中获得巨大的优势)…但是公司的员工仍然需要该领域的大量知识来组合工具(并且花时间来选择工具)。

如果你开始剥离所有对你的公司来说不独特的东西，你只剩下几个人来创造公司独特的部分。然后再补充几个需要向市场解释其独特好处的人。

想象一下，你 100 人的公司只有 6 个人。想象一下，你 1000 人的公司发展到 20 人。你能移动多快？
