# 我们刚刚发布了开发者新闻。以下是你如何使用它——贡献者——自由代码营论坛

> 原文：<https://www.freecodecamp.org/forum/t/we-just-moved-off-of-medium-and-onto-freecodecamp-news-heres-how-you-can-use-it/279929?utm_source=wanqu.co&utm_campaign=Wanqu+Daily&utm_medium=website>

2019 年 8 月 13 日更新:我已经写了[一篇详细的文章，在这里展示了我们对开发者新闻的愿景](https://www.freecodecamp.org/news/the-new-way-forward-for-developer-news/)。

2015 年，freeCodeCamp 社区将我们的博客工具从 Blogger 改为 Medium。

Medium 是写博客的便利工具，但是社区已经不再需要它了。所以现在我们已经转向一个自我托管的开源博客工具，叫做 Ghost。

我们不再在旧的[medium.freecodecamp.org](http://medium.freecodecamp.org)子域上发布社区文章，而是在新的[www.freecodecamp.org/news/](http://www.freecodecamp.org/news/)子目录上发布，我们称之为开发者新闻。

在过去的一年里，freeCodeCamp 团队一直在开发者新闻上发表文章，我们很高兴现在向 freeCodeCamp 社区的其他贡献者开放。

作为一名贡献者，以下是对你的一些直接好处:

## 改进 #1: 更干净、更快捷的体验

根据谷歌自己的灯塔工具，开发者新闻的速度明显快于媒体。

开发者新闻也更容易获取，对 SEO 更友好。

我们正在稳步努力挤出更多的优化。

## 改进 #2: 更好的分析

作为一名已发表的作者，我们将为您的文章提供对开发者新闻谷歌分析数据的完全访问权。

你可以准确地看到有多少人在读你的文章，读了多长时间，你的读者在世界上的什么地方，他们是如何发现你的文章的。你可以实时地做这些事情。

## 改进 #3: 读者多了很多

我们的 NGINX 在一个典型的工作日早晨记录日志。是的——有 2816 人同时使用 freeCodeCamp 其中超过一半的人正在阅读开发者新闻。

开发者新闻有更好的 SEO，更快的页面加载，更简单的用户体验。

它还拥有加速移动页面(AMP ),这意味着即使在移动连接不好的情况下，页面加载速度也快如闪电。

没有读者帐户。没有“登录”或“付费绕过付费墙”的提示。

所有这些好处加在一起意味着你的读者会有更好的阅读体验，最终会有更多的读者。

## 改进 #4: 你可以用 Markdown 来写

你现在可以用 GitHub 风格的 Markdown 来写了，开发者新闻会把它转换成富文本。

## 改进 #5: 内置语法高亮

Developer News 为大多数主流编程语言提供了语法高亮显示，而不是必须嵌入 GitHub gists 或 IDE 截图。

## 我如何提交我的文章？

我们的风格指南有更多这方面的信息。

是的。我们正在研究公开这些数据的方法。目前，只有已发表的作者可以通过谷歌分析看到这些。
