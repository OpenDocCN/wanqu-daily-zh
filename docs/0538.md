# 重要的中等指标:总时间读数

## 我们如何衡量敬业度

*一百万页面浏览量！
5 万报名！
五百万帖子！
1.65 亿活跃用户！*

网络公司喜欢指标——尤其是当大数字可以用来吸引科技媒体报道我们的时候。

除了硅谷科技博客的宣传，每个网络公司都应该有一些不那么扯淡的指标来指导业务，并提供其健康状况的指标。理想情况下，有一个数字来统治他们所有人。[乔希·埃尔曼称这是唯一重要的指标](/what-i-learned-building/ab24a585b5ea)。

在 Medium，我们的数字是总时间读数，或 TTR。

# 唯一重要的指标

我们先退一步。为什么要有一个数字？如果你接受数字是衡量企业成功的好方法，为什么只有一个呢？

除了基于互联网的公司，大多数企业都以美元来衡量他们的成功。但是媒体行业总是有点不同。通常情况下，广告商根据受众的规模付费。各种各样的技术被用来测量听众的数量:广播使用日记，听众在日记中写下他们听了什么，什么时候听的。印刷媒体将发行或销售的总份数加起来，然后猜测每份有多少人看过。

当网络站稳脚跟时(电子商务只是它眼中的一个亮点)，只有事件——比如页面浏览量和后来的点击量——可以被测量。随着 cookies(和 Google Analytics)的广泛使用，我们开始谈论用户。对于不产生收入的初创公司来说，用户是唯一的货币:T2 注册用户、T4 注册用户和活跃用户。

“大数据”带来了能够测量用户与应用程序的任何(和每一次)交互的奢侈。我们可以记录用户做了什么，用了什么设备，什么时候，持续了多长时间。这些数据存储起来很便宜，处理起来也相对容易。

我们已经跨越了一个点，数据的可用性已经超过了质量指标的要求。我遇到的大多数数据科学家告诉我，他们收集的数据比他们希望使用的要多得多。然而，在许多情况下，他们仍然没有有用的指标。

企业(那些有收入模式的企业)仍然在为钱而优化。今天丰富的数据有助于更好地理解是什么推动了他们的收入。数据分析师可以将最早的用户互动(如营销活动、推荐来源等)联系起来。)和漏斗末端活动(比如花钱或点击广告)。这些数据还可以洞察产品多样化或潜在的新收入流。

没有收入的公司仍然需要优化仍然有价值的用户行为。在 Medium 的案例中，这种有价值的行为是在我们的平台上吸引我们的用户。

## 约会

几年来，参与一直是成长型营销人员的流行语。当用户使用你的平台时，你就吸引了他们的注意力。注意力是超级互联时代的珍贵商品。

我把争夺用户的注意力看成一个零和游戏。由于硬件创新，在清醒的一天中，几乎没有一刻不是被(排名不分先后)书籍、社交网络、电视和游戏占据的。令人惊讶的是，我们有时间工作和家庭。

关于“参与”的确切含义以及如何衡量它(如果可以的话)有很多令人焦虑的问题。当然，这取决于平台，以及你期望你的用户如何在上面花费时间。

对于内容网站(例如,*纽约时报*)，你希望人们阅读。然后回来，读更多的书。

匹配服务(例如 OkCupid)试图匹配伙伴。成功匹配的数量应该让你对企业的健康状况有一个很好的认识。

一个结合了这两种想法的网站怎么样？我有时把媒介描述为内容匹配:我们希望人们写好文章，让其他人读好文章。这是双方面的:一个不能离开另一个而存在。连接双方的核心活动是什么？是读书。读者不仅仅是浏览一个页面，或者点击一个广告。他们*读*。

在 Medium，我们优化了人们阅读的时间。

# 测量阅读时间

TechCrunch 的格雷戈里·费伦斯坦写道:

> 为新闻编辑说句公道话，我们确实知道读者在一篇文章上花了多少时间:我们知道不到 60%的人会读一篇文章的一半以上，而相当一部分人根本不会读任何东西。

我认为这是乐观的。诚然， [Chartbeat 的分析](https://chartbeat.com/)会告诉你用户对内容的参与程度。根据他们的数据，*平均不到 60%的用户阅读超过一半的文章。我们有不同的看法:对我们来说，没有普通用户，也没有普通帖子。*

我们衡量每篇文章的每一次用户互动。这大部分是通过定期记录滚动位置来完成的。我们将这些数据通过管道传输到我们的数据仓库，在那里离线处理汇总花费在阅读上的时间(或我们对它的最佳猜测):我们推断读者何时开始阅读，何时暂停，以及何时完全停止。这种方法允许我们纠正不活动的时期(比如在不同的标签页打开一篇文章，遛狗，或者检查你的手机)。

总阅读时间(TTR)是一个帮助我们了解媒体平台整体表现的指标。我们可以用很多方法来分割这个数字(登录与注销，新帖与旧帖，等等)。).

我们正在考虑利用这些数据来了解媒体用户的其他方式——以及他们与特定帖子的互动。例如:

*   我们如何激励用户增加阅读他们所写文章的总时间？
*   我们用预期阅读时间来衡量帖子的长度。那么，用户花三分钟读完六分钟帖子的一半，和用户花两分钟读完两分钟的帖子，哪个更好呢？
*   如果一个用户花四分钟阅读一篇六分钟的帖子，她浏览了吗？她只是一个超快的读者吗？还是我们的时间估计错了？
*   眼睛记录一幅图像需要多长时间？
*   如果我们想最大化 TTR，帖子的最佳长度是多少？

还有更多。
