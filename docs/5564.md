# 触摸屏的隐性成本

2012 年，我在一次汽车会议上试驾了一款全新的豪华汽车。这是一款极简主义的欧洲车型，看起来没有什么不合适的——直到我尝试使用车内娱乐系统。整个东西是一整块长方形的反射平板玻璃。触摸屏软件既奇怪又笨重。我花了五个步骤把我的手机和汽车配对，我不得不把所有的注意力都放在显示屏上，只是为了弄清楚它。没有基本的物理按钮，比如改变音量或打开收音机。我无法想象晚上开着它会是什么样子。

我向坐在我旁边的人表达了我的沮丧。“我不敢相信公司会让这东西带着这种可怕的展示上路！这就好像这整个系统是在实验室的理想条件下测试的，但从来没有一次在路上遇到一个困惑的司机！”

那个人开始嘲笑我的爆发。然后他道了歉，给了我他的名片。事实证明，他是公司的负责人。



Tesla’s 2012 touchscreen display (via [Car Connection](https://www.thecarconnection.com/overview/tesla_model-s_2012#image=100179523))



“触摸屏的质量是我的错”，他承认，“我们从来没有在路上测试过”。显然，管理层如此确信娱乐系统的蓝色矩形触摸屏界面是“未来”，以至于该公司在发布它之前甚至没有在路上使用它。毕竟，特斯拉最近首次亮相(广受好评)，这是其第一款拥有类似触摸屏系统的车型。现在，这家新公司自己的汽车已经大规模生产，这将需要许多年来消除损害。

物理界面对汽车可用性至关重要。操作依靠简单的一瞥或肌肉记忆。相比之下，触摸屏迫使司机看。因为按钮不是固定在特定的位置，屏幕会抑制肌肉记忆和可发现性。触摸屏与驾驶过程争夺注意力，增加了分心驾驶的危险。

> ***严重界面* — *那些由知识渊博的专业人员反复使用的和/或在潜在危险情况下使用的界面，不应基于触摸屏。如果必须使用触摸屏，它应该嵌入在一组支持肌肉记忆和单一动作的固定物理按钮旁边。***

现在车内接口发生了什么变化？五年后，我们看到一些汽车模型坚持物理按钮和拨号，这是一个很大的解脱。

除了安全考虑之外，物理控制还能提高生产率。当我在大学期间在一个邮件收发室工作时，我用一个非常难看的键盘系统处理包裹。我只用了一天就学会了这台机器，它很快就成了我的一部分；它能够快速处理的数据密度是巨大的。就像高中生在微积分课上使用图形计算器一样，我们可以与这些物理界面纠缠在一起，而不会强迫我们在使用它们时进行思考。我们与他们合作，他们也与我们合作。它们可以被认为是唐娜·哈拉威的“伙伴物种”之一。

触摸屏并不完全可怕。在正确的情况下，它们非常有用。它们在服务相关行业已经很普遍了，但是这些屏幕通常有简单易懂的软件，颜色编码和专业。它们不应该用在行驶的车辆上！



Example of an image-based touchscreen point of service system



触摸屏设计可以受益于一些基本的设计原则。浏览基于颜色的界面时，解析时间会更短。基于图像的界面需要大脑花更长的时间来处理，缺乏对比度可能会令人困惑，因为每个项目都必须与相邻的项目区分开来。当这么多图像看起来相似时，服务人员必须依靠位置和肌肉记忆来快速使用。



Example POS system that is ugly but functional: High contrast, color-coded buttons take less time to mentally digest.



当我在食品服务和邮件收发室工作时，越丑的触摸屏越容易使用。它们用明亮的对比色进行颜色编码，使得数字或项目之间的界限非常明显。我发现颜色减少了错误。我通常只看一眼界面就点击正确的项目。过了一段时间，在提交订单或打印收据之前，我只会在流程结束时检查屏幕上的错误。

大多数触摸屏界面不使用高对比度颜色或锁定的静态按钮来实现基本功能。它们将操作隐藏在多个按钮下，这让我们在试图开车时危险地寻找正确的按钮，或者让我们沮丧的乘客试图通过蓝牙帮助我们连接手机。



顾客和员工都使用 Square 的销售点系统，系统看起来不错很重要，但是如果图像看起来太相似，处理订单和出错仍然是可能的。我知道一些咖啡店已经用迷因或高对比度卡通人物取代了他们的产品形象。这是一个可爱的策略，对员工和客户都有效。

我们会看到模拟接口的回归吗？我当然希望如此。虽然模拟接口并不适用于所有情况，但它们确实会迫使设计人员做出永久性的决定。因为必须为物理按钮位置做出具体选择，所以设计不可用的模拟接口更加困难。设计决策必须是最终的。软件界面可以在没有相同过程的情况下快速改变和部署——这个世界充满了嵌套的、神秘的菜单和令人困惑的用户流。

上周，我家附近的杂货店把所有的键盘式机器换成了触摸屏。现在，杂货商必须花额外的时间看着屏幕，点击一些不同的图片来选择正确的商品。除了生产力的损失，设计不良的系统也让员工不开心。我曾经看着资深超市员工熟练地在键盘上输入 PLU 农产品代码。他们一只手拿着装水果的袋子，另一只手敲着密码。他们的精确和记忆往往是一种骄傲和轻微的荣誉。现在，那一丁点儿的精通已经荡然无存。更糟糕的是，新屏幕需要不同程度的关注，这使得一些员工很难阅读文本。我曾看到工人们在点击正确的产品图片之前，摘下眼镜凝视着显示屏。我甚至发现他们代表机器道歉。

我问一位收银员，他是否认为自己可以在触摸屏上找到和在物理机器上一样的舒适感。

“绝对不行，”他抱怨道，“当这一次变得可以忍受时，他们会改变我身上的软件。”

这些问题只是暗示了触摸屏固有的问题。在后续的帖子中，我们将看看它们如何进一步加剧我们已经在自动化的兴起中看到的社会摩擦。
