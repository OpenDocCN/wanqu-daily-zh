# HTML 就是网络~皮特·兰伯特

> 原文：<https://www.petelambert.com/journal/html-is-the-web?utm_source=wanqu.co&utm_campaign=Wanqu+Daily&utm_medium=website>

前端工程师和 HTML 这几天有什么故事？我最近和很多人谈过，他们似乎并不理解。我是说，他们理解其中一些。他们明白什么是`div`和什么是`span`，只要当他们点击它的时候，它看起来很好并且可以工作，他们就足够高兴了。我交谈过的很多人都用“哦，这些天我在 React 或 Vue 中做任何事情”来回答关于 HTML 的问题。但是真的，如果你写的都是 Javascript 也没关系，因为如果你在做网站，那么你做的最重要的东西就是 HTML。HTML *就是*网络

一切都是关于消费者消费了什么。这是用户界面和 UX。这是整套的。按照重要性降序排列，依次是 HTML、CSS 和行为(可能由 Javascript 提供，也可能没有)。

我最担心的是技术金字塔的底部。网络的最小公分母。基金会。节奏部分。网蛋糕里的手指。是 HTML。我越来越清楚，有一大群前端工程师不知道或不理解最前端的技术。

网页是一个文档。一个组件，无论是博客模板、新闻网站、营销统计仪表板还是注册表单的一部分，都是文档的一部分。文档是有结构的。在 Web 上，这不仅仅是关于你的框架所提供的视觉效果或架构，而是关于选择语义上正确的元素，以确保你的网页、组件等等在结构上格式正确。标题应该是标题，列表应该是列表，按钮应该是按钮，表格应该是表格。你可以随心所欲地设计它们的样式——标题不一定要大而粗，还要有下边距。这取决于你，但它肯定应该是一个标题，如果你把它变成一个`div`，我就和你打。

正确学习 HTML 并不难，尤其是如果你已经习惯了学习 Javascript 框架。我没有数过，但我很确定只有大约 116 种元素，而且大部分都是你通常不需要的。你为什么不学呢？

我是那种“前端中的前端”的人。我的专长是 HTML 和 CSS，所以我很容易抒情地谈论为什么每个人都应该学习我已经知道的东西(声明一下，我并不知道所有的东西——我们仍然在办公室里就标记某个组件的最佳方式进行激烈的辩论)。这不是关于“我的工作比你的更重要”。如果你正在编写在浏览器中渲染的代码，这个*就是*你的工作。

这是关于可用性和可访问性。如果你认为你的网页或应用程序的语义结构不重要，那么你实际上是在说“好吧，它在我的浏览器中对我有用，把它装上吧”。我不认为你会用你的 Javascript 这样做，你当然也不应该用你的 CSS 这样做。搜索引擎需要阅读你的内容，而不是欣赏你的动画或花哨的渐变。屏幕阅读器软件需要阅读你的内容。键盘用户需要阅读你的内容。谁知道接下来会出现什么技术，以及它会如何消耗你的应用程序，但我敢打赌，如果它能轻松读取、解析和遍历你的内容，它会工作得更好。这些东西读取你的内容的方式是，它们知道这实际上是内容，而不仅仅是包装在无意义标签中的文本字符串。他们知道什么是表格以及如何表示它，他们知道什么是列表以及如何表示它，他们知道什么是按钮，什么是复选框。让一切从`div` s 开始，他们将不得不非常努力地工作才能搞清楚。

> “但我的选择框架考虑到了所有这些。我只是写`.jsx`模板。”

没有。在你的 JSX 中写适当的 HTML。你能做到的。仅仅因为你在使用 React 或 Vue 或其他什么，你就不必从头开始做所有的东西。

> “这个库为所有东西都添加了 WAI-Aria 属性，所以我知道它是可访问的，因为他们已经完成了这项工作。”

太好了。如果您编写了合适的 HTML，这些属性中的大部分根本就不需要。通过使用一个真正的`button`而不是一个带有`onClick`监听器的`div`，你可以免费获得一大堆辅助功能。免费的。这是可访问性、性能和用户体验点，免费。免费！

这东西真的很重要。不做这些事情是慢慢地(实际上没那么慢)打破万维网。至少，这会让使用你产品的人更难使用它。如果你称自己为前端工程师，学习和使用基础知识是你的责任——这是所有可以访问网络的浏览器、平台、设备或家用电器都通用的东西。

请这样做。负责任地建设网络，让网络变得更美好。网上有很多资源可以帮你做到这一点，但这里有几个入门资源:

*   了解如何用 HTML 标记文档。尝试一些小的思考练习，看看音乐会海报或报纸版面，想象它在 HTML 中的结构。如果你有时间，建造它。在日常工作中运用这些知识。
*   MDN 是博客文章、教程和参考资料的绝佳资源。
*   接触社区里的人。阅读博客文章(比如安迪·贝尔最近关于使用语义 HTML 的文章，[保持简单](https://andy-bell.design/wrote/keep-it-simple/))和[观看视频](https://css-tricks.com/video-screencasts/58-html-css-the-very-basics/)。
*   我上来的时候，*查看源*还是有用的。我们已经为现在和未来的几代人共同打破了这一点，但是我想让你记住知道[如何使用浏览器的开发工具](https://developer.mozilla.org/en-US/docs/Learn/Common_questions/What_are_browser_developer_tools)所带来的力量
*   了解[辅助技术如何访问网络](https://www.smashingmagazine.com/2019/02/accessibility-webinar/)
*   看看 [HTML 规范](https://www.w3.org/TR/html52/)，或者仅仅看看 [HTML 元素](https://developer.mozilla.org/en-US/docs/Web/HTML/Element)及其用途的列表。
*   如果你在一个开发团队中工作，讨论一下标记。就*某物*应该是`table`还是`dl` ( [描述列表元素- MDN](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/dl) )进行热烈的讨论。很有趣，我保证。
*   找出你团队中的 HTML 专家，让他们检查你的代码。如果那个人是我，我总是很乐意进行这样的讨论。
