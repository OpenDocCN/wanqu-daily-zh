# 在 Stripe 工作感觉如何| Kalzumeus 软件

> 原文:[https://www.kalzumeus.com/2019/3/18/two-years-at-stripe/?UTM _ source = Wanqu . co&UTM _ campaign = Wanqu+Daily&UTM _ medium = website](https://www.kalzumeus.com/2019/3/18/two-years-at-stripe/?utm_source=wanqu.co&utm_campaign=Wanqu+Daily&utm_medium=website)

两年前，我加入了 Stripe，目的是让创办互联网企业变得更容易，主要是通过制作 T2 的 Stripe 图集 T3。在创办了四家小型软件公司后，我希望同行们少处理一些无聊的事情，要么将解决方案产品化，要么写下我希望在 2006 年创业时就知道的东西。本着同样的精神，这里有一些我在加入 Stripe 之前不知道的事情。

## 你到底是做什么的？

Stripe 的使命是增加互联网的 GDP。我的公司正在增加成功的软件公司的数量。这基本上是工作描述的全部。在日常生活中，选择自己的冒险是非常重要的；我写过代码、需求文档、战略备忘录、 [Stripe Atlas 指南](https://stripe.com/atlas/guides)并为 Stripe 员工、合作伙伴和用户提供建议。

这是…写在名片上不是一件简单的事情。

一方面，它给了我很大的自由来决定我的日常方向，并利用了广泛的技能组合。相对来说，很少有人能够独自完成一项需要文案、反应、ETL 管道和商业谈判的计划；事实证明，这是非常有价值的，减少它的上市时间(相对于不得不组建一个五人团队来做同样的工作)是有用的。

同时，职位描述存在也是有原因的；它们有助于将一个人的立场传达给公司的其他人，澄清关于依赖图的讨论，在制定计划时给决策者一个把柄，等等。成为`Stripe::Atlas::Patio11`的单一实例有时是一件喜忧参半的事情。

## 你还做什么？

除了“继续做我在互联网上通常(希望)乐于助人的自己”，我还试图在我的正式角色之外做出贡献。Stripe 的工作文化中，我最喜欢的一点是“没有什么不是我的工作”这一理念我在 Stripe 的第一个真正的拉请求是修复一个安全漏洞；我很高兴地得知他们是认真的，他们在 spinup 中说过，安全团队花时间让我的代码达到标准，尽管我看起来像“随机业务(？)日本办事处的家伙”。

Stripe 触及了人类努力的一个大子集(基本上，“只是涉及到钱的事情”)。我们的经营规模相对于互联网上的商业来说很小(互联网本身还没有接近它的渐近线),但已经非常重要了。

我最喜欢的一些项目已经毁了坏人的日子。我们的用户每天都有大量资金通过我们流动；网上每个坏人都想要；我们一直在和他们赛跑。有很多聪明的人，每天整天工作来挫败他们；我已经帮过一次或十次忙了。

一个例子:我们有一个内部红队演习运行作为一个竞赛:找出如何窃取所有的钱，奖金点，如果一个可能逃脱它。整个公司的人都做了安全研究，并写下了他们的发现；然后，负责任的团队得到了大量高质量的补救材料，而我们实际上没有损失一分钱。我以几十份作品赢得了比赛。由于这一努力，我们关闭了漏洞类别。

这里的各种努力(安全、风险管理等)受益于理解技术上的可能性，能够快速实现它，以及对小型企业有一个好的心智模型(“对于一个基于互联网的软件企业，什么是正常的行为？”)和大型(“全球金融基础设施的弱点在哪里，我们如何管理它们的风险？”).

我还参与(视具体时间而定)产品战略、写作、公司文化建设、组织问题、支持日本团队、工程招聘、为 Stripe 客户提供建议(通常，真的会收取更多费用)等。

## 你可以自己做生意，为什么还要在 Stripe 工作？

杠杆作用。

杠杆意味着增加你提供影响力的能力。例如，写代码是对你时间的一种极度杠杆化的使用；你离开继续你的生活，但你的代码继续运行。我做过的一些有杠杆作用的事情是写作，写软件，做生意，雇佣别人而不是我，和聪明人一起努力解决我想解决的问题。

我经营自己的企业已经有几年了，包括与联合创始人一起创业，我非常享受这种经历，但我觉得我已经最大限度地应对了经营小型软件公司的挑战。

我还觉得这限制了我对这个世界的绝对影响力。我最喜欢的工作部分并不是我的实际工作(编写和销售软件，及时报税等)，而是帮助其他软件企业家优化他们的业务或工程师应对职业挑战。

[郭凯文](https://twitter.com/kevinakwok)问道“利用什么杠杆？”。在这个职业生涯的 15 年里，我明确地知道我未来 30 年的成功函数是什么:我在我所服务的社区中帮助的人数(广义地说是软件人)乘以我的努力所带来的平均寿命增量的乘积。如果一个人使用收入作为衡量标准，因为它足够脆，可以让一个人保持诚实，那么我做过的最有效的事情(根据投资回报率，由*到目前为止*)是写关于[工程师薪资谈判](https://www.kalzumeus.com/2012/01/23/salary-negotiation/)。几百个人给我发了电子邮件，告诉我他们对那篇文章的评价；这使得对该论文分数的影响下限为大约 7，000，000(一年)。

Stripe 提供了一个大得多的平台，为创业者提供可扩展的教育和建议，与我自己的公司不同，它在结构上直接受益于这样做。(我写了几百万字给 Bingo Card Creator 带来了一些间接的好处，但是 Stripe 从我们生产的每一件产品中直接受益*，这使得 N 家企业在边际上更加成功。公司级的使命是“增加互联网的 GDP 每当我们提高企业形成的速度、企业生存的可能性或企业增长的速度，我们就赢了。)*

 *在过去的 12 年里，我的很多专业成果都是让企业变得更好 1%,并不断重复。对于 Bingo Card Creator 来说，这是一个有趣的游戏，但是为咨询客户做这个游戏更有利可图。Stripe 的规模允许一些决策*被充分利用，无论是直接利用还是在我们客户的业务中。*

(举一个例子，这只是一个关于世界的令人困惑的事实:你凭直觉猜测信用卡支付失败的百分比是多少，错误代码是我不知道，有时在信用卡领域会出问题？提示:比你想象的要高。那些失败的付款在边际成本转换。当 Stripe 将这个数字降低*一个基点*，这将永远为我们的整个投资组合创造价值。)

Stripe 的人脉远比我广，在适当的时候，我可以利用我们的组织因缘。这让我可以做一些需要人们合作的事情，而我可能无法以个人身份打电话给他们。举个例子:我可能会在 HN 关于如何推介投资者的评论中瞎猜，但作为一个例子，我可以直接与 YC 合作，优化他们的合作伙伴在考察公司时真正需要的建议。

我花在低价值业务管理上的时间从一周的 10%下降到了 20%(真的！)到功能零。经营我自己的企业的一个令人惊讶的部分是合同谈判，确保供应商得到报酬，并及时将利润汇至租金。我们有专门从事这些事情的人，所以我不再需要。(我怀念被叫醒的感觉，因为服务器已经…开个玩笑，我一点也不怀念。)

杠杆是有成本的。

一个代价是，组织最终会达到这样一个点，即“让一群聪明人呆在房间里，让他们做任何事情”不再导致自发的秩序，而开始导致混乱。我们已经过了那个阶段。这意味着我们需要某种计划和管理过程，虽然我与他们有古怪的互动，但我确实与他们有实质性的互动。这种互动是个人生产力的一种负担。这是一笔不小的开销。

寻求杠杆作用也意味着在其他条件相同的情况下，偶尔不得不选择我不想做的项目。Stripe 可能会让我把自己锁在房间里写 N 年，但这可能会大大低于影响的理论极限，所以我有时会做一些对公司来说更紧迫但对个人来说更没有成就感的工作。

我偶尔会怀念做自己的事吗？是的，至少一个月一次，尤其是和企业家直接打交道的时候。我没有足够的带宽来制作任何实物原型，但我感觉到了它的诱惑；我很确定我的下一次冒险将是再次经营软件业务。

我*绝对*觉得在 Stripe 工作增加了我对下一个业务的渴望。部分原因是通过接触生态系统中各种各样的公司；企业家的创造力总是让我惊叹。(在其他条件相同的情况下，*绝对*选择一个客户群，他们对你的产品的创造性使用会让你惊喜。)我也花了一些时间在互联网商务的管道中爬行，这是一种美丽的分形，由一个能够从超越时空的地方召唤恐怖的方程描述。

或许自加入以来，我的信念发生的最大变化是，如果利用得当，雄心可以成为世界上巨大的生产力。这在很大程度上得益于与 Stripe 和我们的客户中非常有抱负但又有良好基础的人一起工作。“造一个信用卡处理器？那是*不可能的*”和“建立一个信用卡处理器？这可能包括遵守一套可列举的规则，并编写有限的代码行。”你希望在你的生活中有更多的人说第二个版本，可能最多也就是差一点点。

与此相关，我认为我同意泰勒·考恩的观点，即[提高他人的期望](https://marginalrevolution.com/marginalrevolution/2018/10/high-return-activity-raising-others-aspirations.html)是提高生产率的有效方式。

硅谷大规模做到这一点的一种方式是在文化中创造一个空间，让人们在预见潜在影响时有一点点狂野的目光，然后，这很重要，实际上是运送易处理的工程工件来对抗愿景。

当它被用于人们认为不严肃的目的时，或者当它被那些闲聊的人使用时，它经常被人取笑。我以前可能开过这样的玩笑。我想我已经开始后悔了，因为这些话也被那些有着严肃目标、一丝不苟的人听到了。这些笑话，以及更广泛的文化，阻碍了这些人尝试雄心勃勃的事情。

如果我有一个重来的机会，比如经营自己的企业十年，我可能会跳过相对平淡的前两个(我这么做只是因为我知道我有可能做到)，直接尝试我舒适区之外的事情。(在我心中，我仍然对小型软件企业的美学有着巨大的热爱，但是如果可以重来，我可能会比我实际做的更早地做更有趣、更有影响力的小型软件企业。)

人们可以同时持有两种想法:不严肃的目的和谎言不是积极的，但我们可能低估了以创业方式做非常重要工作的团队的数量。他们名义上是不合格的，他们经常以小本经营，他们看起来不像好莱坞会塑造的明显重要的人物，然而他们总是表现出色。我工作中最好的一部分就是帮助像这样的人建立他们的企业。

这可能是我职业生涯 15 年来最大的一个发现，所以，把它放回瓶子里，给任何一个刚开始的人。我们可能还没有时间旅行，但互联网是下一个最好的东西。

## 作为高级个人贡献者

让一些从外部了解我的人有些惊讶的是，我不是一名经理(Stripe Atlas 或其他任何公司的经理)。我是一名 IC(“个人贡献者”，这是一个在大到足以拥有职业轨迹的公司中广泛使用的行话)。

在职业生涯的早期，集成电路主要通过“工作”来创造价值。在一个人的第一年，例如作为一名 IC 工程师，你编写代码支持不同复杂性的项目。

随着技能曲线的上升，集成电路继续做“工作”,同时也在非管理元工作上花费越来越多的时间，如以定向方式为组织文化做出贡献，设定战略方向，指导他人(包括经理),建立未来完成工作的系统和流程等。

Stripe 为工程和其他领域的集成电路提供了明确的职业道路。它旨在避免相对行业常见的失败模式，即随着职业生涯的发展，要么开始管理，要么对职业影响和结果做出相当大的妥协。管理是一个有用的专业，但如果其他专业更能充分利用你的时间，你可能就不应该去做。

影响力的一个重要来源是其他人。你可以通过指导人们做事来获得影响力(这是一种超级力量，在我独自经营企业时，我可能低估了它的影响)。你也可以通过让他们更有效地做事来得到它。

当我与 Stripe 的人们一起工作时，我试图让他们在组织工作中更加有效，理解各个工作单元的更广泛的业务环境，理解创业生态系统以及我们用户的心态和挑战等。对于特别专注的角色，我会努力让他们更好地完成工作的核心内容。

例如，我在 Stripe 写了很多东西，但 Stripe 现在有很多人认为这句话是对的。我似乎可以在一年内写完 X00，000 个单词，但是最终会达到一个稳定的水平；如果可运输的东西使其他作家更具工具性，就不会有高原。这有时来自直接的编辑工作，但更多的是来自教学，有时是关于写作技巧，有时是更具体的细节，比如如何再现公司的声音。(我还写了大量代码，因为有趣的是，stripe.com 拼写为 CMS e-r-b，这并没有优化作者输入新单词的能力。)

我的同事 Julia Evans 有一篇关于[成为高级工程师](https://jvns.ca/blog/senior-engineer/)的文章，详细讨论了成为高级集成电路工程师的细节。

(旁注:作为一名资深 IC，产生重大影响的一个方法是提供一个例子，让那些在职业生涯中成长的人知道他们渴望成为什么样的资深 IC。另一个是明确地与人们一起工作，加速他们向 it 和其他方向的发展。)

## 写作，写作，写作

Stripe 是一种对书写单词的庆祝，这个单词恰好在特拉华州注册。

我们在内部产生大量的信息，其中大部分在公司内部随处可见。我最喜欢工作津贴可能是图书馆；它包括一切，从关于图书出版现状的噼噼啪啪的备忘录(与[我们的利益](https://press.stripe.com)相关)，到关于使用机器学习来对抗信用卡欺诈的实验性文章(与[我们的利益](https://stripe.com/radar)相关)，到日本企业采用 SaaS 的市场分析(与[我们的利益](https://stripe.com/billing)相关)。

我期望在 Stripe 上写一个 shedload，我做到了。没想到大部分是面向内部的。例证:在 Stripe，需要知道如何对 SaaS 企业进行推理的人，比经营过 SaaS 企业的人多得多，因此我把自己锁在一个房间里，把我 10 年来学到的东西进行数据转储。(我们最终发布了一个[公开版本](https://stripe.com/atlas/guides/business-of-saas)。)

我想我曾经有过典型的工程师对战略备忘录(TM)的鄙视，但是如果你认为它不是一个战略备忘录(TM ),而更像是一个非常复杂的程序的拉式请求，这个程序指导几十或几百人的活动，那就更容易接受了。

我最近写了 Stripe 目前在日本的位置，Stripe 对日本互联网的影响如何才能像对美国互联网一样，以及我们应该在 2019 年采取哪些具体措施来实现这一目标。对我来说，我能在那里做出一些有用的贡献是不太可能的，但该公司做得相当好的一件事是成为一台分布式责任机器。在 BigCo 工作告诉你，在另一个城堡里有一些专家已经写好了行军命令，你所要做的就是执行你分配到的那一小部分。在 Stripe，你可以点一份隐喻性的[披萨](https://www.google.com/search?q=google+bezos+two+pizza+rule)并说“好消息:Stripe 的确有聪明人在为世界第三大经济体制定上市计划。我们刚刚给他们买了一个大披萨。”

人们当然可以把战略备忘录写得相当于缩进规则上的自行车脱落。尽量不要这样做，或者和喜欢这样做的人一起工作。

关于缩进规则的一个题外话:处理所有风格参数的正确方法是用代码来执行它们，并使*唯一的*支持的风格参数接口针对 linter/编译器拉请求，而不是对单个工程师的工作进行评论。由于这个原因，这可能是 Golang 最大的特点。

在 Stripe，有很多东西是合法的，但却是非法的。一个简单的例子就是`@improper_casing_for_ruby`。一个更有意义的例子是从定义的边界之外调用另一个团队的代码。因为棉绒基本上抓住了所有这些，如果一个人发现自己对拉请求吹毛求疵，他通常应该要么 a)修理棉绒，要么 b)找一些有成效的事情来做。

这是被认为使用技术有助于改善人类问题的许多方法之一。在科技公司工作的乐趣之一是，他们似乎在探索这一点上，以及在自我修改工具以更好地适应他们想要的工作方式上，拥有相对优势。这是我预计 2019 年工程师就业人数将是其余时间最低的一个原因:每个关心工作的社会现实的组织都将受益于能够组成自己的工具。(这听起来像是一个有争议的说法，但基本上每个组织都认为领导力太重要，不能外包，领导力只是为组织编程，这在社会上没有争议。)

## (非常)遥远

Stripe 在国际上迅速扩张，但我日常接触的大多数人都在旧金山。这有时很有挑战性。

出于取悦他人的渴望，在第一年，我没有在可接受的会议时间周围建立很大的界限，最终在比如说太平洋标准时间上午 11 点结束了很多会议。在我的睡眠时间表上拔了不少毛之后，我最终在日历上划掉了我的睡眠时间；这是生活质量的一大进步。这些天，我很早起床(一般是早上 7 点)去开会，在上学前休息一会儿和孩子们在一起，然后去办公室。

人们经常问我关于工作/生活平衡的问题。我(坦率地)担心，在这一点上，我并不总是为我的团队或办公室树立一个好榜样；我有一些被间歇打断的相当脆脆的时期，甚至在一个不计算时间或为了面子而面子的商店里，我发现自己太频繁地回到工薪族的模式。我很高兴地告诉大家，在 Stripe，那些更善于自律和保持界限的人确实做得很成功，而且通常不会牺牲实际或感知的效率。

我还认为，特别是随着公司的成熟，它们应该普遍发展组织炼金术，让工作风格迥异的人发挥出大于部分之和的效力，就像[企业联合不同风险承受能力的投资者](https://www.bloomberg.com/opinion/articles/2019-02-13/santander-didn-t-pay-its-non-debt)为共同的企业提供资金一样，任何承受能力水平都无法独自提供全部资金。

有一个问题，通情达理的人可能会有不同意见:定期在晚上 7 点开会可以吗？(在这个问题上，我同情“哦，见鬼，不”，但请注意，如果你想支持传统管理的日本企业，那不会是最有效的回应，还有许多其他合理的理由让某人说，考虑到他们的目标、价值观和个人情况，“实际上，晚上 7 点的会议对我来说听起来很合理。这不是一个疯狂的要求，比如说，谁会在早上 9 点出现在办公室里。”)

如果你的公司有四个人，你可能需要*几乎完全同意*晚上 7 点的会议是否是例行公事。如果你有 400，你可能可以容忍群体/角色/个人等在这个问题上以不同的均衡结束。将您有限的总协议强制性积分留给您文化中的关键差异化价值。(Stripe 用[这份文件](https://stripe.com/us/jobs/candidate-info)向应聘者描述其组织文化。)

我大约每季度飞一次总部，主要是为了“见见老朋友，交交新朋友”每次呆在一个有数百人的餐厅里，这既非常有用，又有点不和谐，上次我去的时候，他们都不在那里。超增长是一种透过玻璃的体验，尤其是当你一点一点地看待它的时候。

对我的团队大喊，这是我参加过的最支持我的团队。我们最近增加了一些新的远程工程师，整个团队在家工作了一周，以更好地了解体验，并捕捉改进我们通信规范的需求。

对于一家似乎总是依赖电子邮件和懒散的公司来说，了解以下情况是很有用的:大量信息分布在午餐桌上，决策是在走廊谈话中做出的，这两者对于只能通过互联网了解总部的人来说都不太好。最近有人做了一个小东西，我很喜欢在这方面提供帮助:如果你安排条纹之间的会议，视频会议链接会自动添加到会议中，这样任何可能需要它的远程员工都不会觉得他们必须不断打断会议，以提醒某人打开摄像头并将链接分发到远程设备。

我们正在相当积极地扩大远程条纹的队列，部分原因是改进的协作技术(Zoom、Google Docs 等)，部分原因是许多团队尝试添加远程的愉快体验，部分原因是希望吸引和留住有才华的人，无论他们在哪里，部分原因是希望我们的客户更加本地化，这些客户分布非常广泛。

## 快速和慢速

出于显而易见的原因，我不能告诉你 Stripe 的任何增长率，但让我们来谈谈一家名为 2X，Inc .的风格化公司，其中每个图表都以每年 2X 的速度增长。

以 2 倍的速度工作打破了许多对世界应该如何运转的直觉理解。

当你从 2X 开始的时候，你一半的同事只有不到一年的经验。一年后，当你开始适应海上生活时，你一半的同事只有不到一年的经验。一年后，当资深同事开始忘记你在洪水期间不在现场时，你一半的同事只有不到一年的经验。

过一会儿你就会对图表麻木了。数字从一只耳朵进，从另一只耳朵出；保留任何东西几乎是没有意义的，因为当一个人沉入你的意识时，它不仅是错误的，而且是明显错误的。每年翻两番，每个人都在笑规划部门怎么又做对了；太诡异了。

这既令人兴奋又令人恐惧，因为自然界中几乎没有任何东西在持续的时间内以每年 2 倍的速度增长。2X 一直在寻找新的机会来维持增长，但在寻找的同时也面临着挑战，那就是在图 T1 的背景下做任何有意义的事情。

Stripe 不是 2X，Inc .，但偶尔会遇到一些这种风格化公司所面临的挑战。

你必须为还没到公司的人写仪表盘和备忘录。大多数人都不是。你必须在会议上举手说:“嘿，你刚刚说了‘销售漏斗’:你能给我一个最学究式的解释吗？”(这对资深人士来说是一件好事，因为强迫上周加入的人打断他们问一个基本问题不会让他们感到舒适或成功。)你必须雇佣专业的教育工作者(最终真的要雇佣历史学家和人类学家)，因为你实际上是在一所小大学的规模上管理一个研究生学习项目。

你必须不断修正你对什么是重大影响的理解，因为希望你在寻找重大影响，而三年前的重大事件可能不再起作用。三年前不可思议的事情现在可能很有可能，而且值得去做，但前提是你能在接下来的六个月内做到。

你必须保持这种文化，为一系列非常值得做但不会明显改变的事情拼命奋斗。我每年飞往 [MicroConf](https://www.microconf.com) 的一个原因是，确保我们遇到那些第一个月收入达到 1 美元、1 千美元或 1 万美元的人，并且*完全*了解他们需要我们做什么。我们不仅要防止他们的体验退化，还要让这种体验比 5 年前有所改观。

## 创造重要的东西

作为 Stripe 的长期用户，我一获得 git 访问权限，就开始探索产品页面背后的神奇之处。(如果你对此不熟悉的话:Stripe 在为新产品页面提供精心设计的处理方面有一个当之无愧的名声。这里有[一个](https://stripe.com/us/billing)一般参考。)

剧透:在这个话题或其他任何地方，几乎没有什么神奇之处。登陆页面是一个 ERB 文件，开始是空白的。它吐出 HTML(和一些 CSS 和 Javascript)，和互联网上的其他页面一样。这些网页之所以非凡，是因为极少数非常有才华的人不可理喻地认为他们是非凡的。

通往伟大工作的道路通常是 a)雇佣非常有才华的人组成的小团队，他们非常专注于某件事，b)在众多其他有价值的优先事项中激励伟大的工作。

人们经常称之为“保护工作出色的团队”，但我认为这并不完全正确。平庸不会发生，因为恶棍扼杀好作品，然后疯狂地咯咯笑。BigCo 并不打算平庸… BigCo 只是有太多的人有太多的事情要做，他们中的许多人受到激励，以牺牲竞争的全球目标为代价，做出局部最优的决策。

进行组织设计和激励结构设计，从内部解决委托人/代理人问题，并促使人们大规模地优化全球结果，这可能是最难解决的问题。(在管理方面？在资本主义？任何事？你挑吧。)我们正在努力；具体来说，它通常包括为人们留出空间，让他们说“这个项目的最有可能的版本是什么样的？”以及接受例如将事情推迟到它们准备好的时候。(请注意，当一个人的业务和环境以 7 级速度发展时，这样做隐含着紧张。没有一个问题是简单的，否则就没有必要让聪明人来解决这些问题。)

“我是创始人”认为，在大型(或成长中的)组织中的一点经验对于纠正一个人的世界观和预测大型组织中未来客户、交易对手、合作伙伴等行为的能力非常有用。作为一名创始人，这是一种有用的能力，而且(有点令人惊讶的是)我们在学校花很少的时间来教授大型组织实际上是如何做决策的。

例如，公民课可以教你对[立法过程](https://www.youtube.com/watch?v=tyeJ55o3El0)的正统理解，但它掩盖了现实世界中真正重要的细节，比如被监管者如何直接与监管者接触，以塑造未来的立法。我同样认为，在我参加过的管理或资本主义课程中，没有一门课会说“有时，令人愤慨的结果的原因不是委托人/代理人的问题，也不是‘恶的最大化’或无能，而是编写一个能够及时解决问题的仪表板超出了最直接负责的个人的 SQL 能力。”

说到 SQL，我喜欢我们的内部工具。我们有许多工程师专门从事这项工作；我想我们大概可以忍受五倍于现在的人口。他们开发的一些东西显然会对我工作过的每一家软件公司有所帮助:一种自动执行(从而确保安全)某人需要打开控制台风格的任务的简单方法，[在整个代码库中进行即时代码搜索](https://github.com/livegrep/livegrep)，以及一种订阅关于某一组对象的状态变化的每日电子邮件的方法(例如，“对于我管理的所有帐户，每天早上给我发邮件，列出任何达到$MILESTONE 的人，这样我就可以向他们表示祝贺。”)我喜欢我们的工具的原因之一是工程驱动的对可组合性的强调，这样我们添加的每一个额外的工具都会使我们的工具(和操作)的大部分变得更有价值，通常是以意想不到的方式。我不能大力宣传投资内部工具。
