## 为了乐趣和利润的应用资本主义

我是资本家。我的一个朋友是忠诚的马克思主义者。我想我们都同意，考虑到任何一个特定的员工，停止出售工时并尽快开始出租他积累的资本是符合该员工个人利益的。

我指的不仅仅是货币资本——在你的 401k 账户中拥有 10 万美元是很棒的，但这不是我真正感兴趣的类型，因为这种资本的回报率非常低。有许多类型的资本并不因为你不能方便地将它们简化为一个数字而变得不真实。

人力资本——你随着时间的推移积累起来的技能，以及你因此能够创造的价值。

社会资本——给信任你的人打电话，让他们为你做一些事情的能力，比如给你推荐一份工作。

声誉资本——当你的专业话题出现时，你的名字在你不在的房间里响起的方式。(希望是好的方面！)

许多日常工作从结构上抑制了资本形成。如果我是一个马克思主义者，我会说“这是资本渴望让劳动服从于它的预期结果”，但我真的认为这是真的，即使没有人需要捻动他们的胡子。

杰克韦尔奇有一句名言，大意是“你工作一周，周五领薪水，然后你和公司就扯平了。”美国企业已经疯狂地接受了它。我太年轻了，不记得在美国，“公司忠诚”不是一句妙语。

如果对公司的忠诚是一个有利可图的提议(它可能仍然存在于一些地方——我知道有一两家较小的公司，在那里“我们像对待家人一样对待我们的员工”的意思与标签上写的一模一样)，你每周工作就会得到一点资本。你的老板对你的好印象又增加了一周。离你的养老金还有一周。离那块金表还有一周。

日本工薪阶层仍然有这样的安排。

在我的前雇主那里，我意识到我不可能一直干到退休，因为那将是我的死期。(我不会对我生命中的那段时间进行过多的描述，因为那段时间相当艰难，但我只想说，如果你连续 6 个月每周工作 90 个小时，在接近尾声时，周期性的昏厥开始变得有点令人沮丧。)

一旦我得出结论，我可能会退出，因此不考虑公司特有资本的缓慢积累，我意识到许多日常工作的基本事实。从长远来看，我在工作中所做的一切都无关紧要。

当然，从短期来看，我正在编写 XML 文件和 Java 类，幸运的是，它们成功地让我的雇主向他们的客户(一所主要的大学)发送了一个考试管理系统。我是一台非常有效的图灵机，它接受电子邮件和票证作为输入，并(偶尔)发送工作代码和 Excel 文件作为输出。但是无论我怎么旋转，我的处境都没有改变。我工作了一周，到了周末，却没什么可展示的。下一周会有更多的电子邮件和更多的罚单，就像前一周一样。在那之后的一周将会是同样的情况。我的生活绝对不会改变。我会一无所有地结束这个星期。

不要空手结束一周。喜欢做你能展示的事情。我喜欢在人们能看到你的地方工作。更喜欢做你能拥有的事情。

## 更喜欢做你能展示的东西

开发者如此热衷于 OSS 的原因之一是因为它给了你公司之间的可移动资本:如果你的工作在 Github 上，即使你离开了一份工作，你也可以把它带到你的下一份工作。以前这种情况发生得相当普遍，但一般都是私下进行的。(有没有程序员没有 snippets 文件夹或者他们自己的私人库来挠痒痒？)OSS 在这方面的一个大问题是，OSS 默认是公开的，这改变了游戏规则。

为什么？因为当你的作品在公共场合时，你可以展示给人们看。这通常是证明你有能力做好这份工作的最佳方式。

告诉人们你可以做伟大的工作是很容易的:任何白痴都可以做，许多白痴都这样做。让人们告诉人们你做得很好是一种进步。它之所以遭遇困难，是因为衡量团队工作中的个人生产率是出了名的困难，而且人们通常没有特别的理由去相信做背书的人的陈述。

(快速提问:如果你有可信的证据表明，名古屋一家你从未听说过的公司的一名中层工程经理认为我是一名非常能干的员工，这是否会让你明显更有可能雇用我？对，没有了解他的背景，那个推荐几乎没用。)

不过，你可以炫耀的工作是你技能的初步证据。在你的投资组合中包含它之后，你推销自己技能的能力会明显提高。鉴于大多数人的净资产几乎 100%投资于他们的个人资本(即，如果你是一名年轻的工程师，所有未来工资的净现值绝对会淹没你银行账户中的一切)，这是对你目前状况的一个相当彻底的改善，而不是对你做事方式的一个非常彻底的改变。

因此，我的第一条建议是:如果你可以在多份工作中选择，在其他条件都相同的情况下，选择一份你能展示自己工作经历的工作。这可能意味着在一个语言堆栈上工作，其中工作双产品通常是有交叉的(例如 Rails)而不是没有交叉的(例如 C#)。这可能意味着在组织内从事喜欢外部可见性的特定项目(例如 Android)而不是不喜欢的项目(例如 AdWords plumbing——大概谷歌会付给你一大笔钱来做这件事，但把它当作不能谈论它的补偿)。这可能意味着在默认开放的行业工作，而不是在默认关闭的行业工作。

OSS 并不是展示你工作成果的唯一方式。在创意产业中，最终产品是客户可以看到的，人们会密切关注谁的名字会出现在演职员表中。学者们花费大量时间担心引用计数和有向图。

更通俗地说，尽早建立一个预期，你只是想谈谈你正在做的事情。我想在 Fog Creek / Stack Exchange，他们称之为“生产工件”——以工作为中心的会议演示、博客帖子、OSSed 软件等等。即使在非常开放的公司，也存在许多秘密，但是公司的大部分有价值的工作并不是特别敏感，而且其中许多有着广泛的普遍性。当你学习这些课程时，把它们写下来。如果可能的话，发表你写的东西。即使它不面向任何人发布，你也可以在以后把人们引回来。

就职业发展而言，我最有成效的一些作品是在 2006 年至 2008 年，当时我正在为不理解自己正在做的任何事情而挣扎，而且我的读者数量——毫不夸张地说——比我弟弟写超级英雄小说的博客还少。为什么在互联网上默默无闻地工作仍然有价值？因为我能够指出我在 2008 年开始的特定实验，然后指出 2009 年和 2010 年的后续实验，这些实验显示出*非常成功*。大多数人对失败和错误的开始并不感兴趣，但是在你的腰带下有一些成功可信地证明了你有能力在未来复制它们或者在你的新环境中尝试你的新成功。

如果你不能制作在工作时可以展示*的东西，你应该制作在*工作之外可以展示*的东西。我们行业中的公司对知识产权转让条款逐渐变得更加合理——如今“我们拥有你在工作中任何时候想到的一切”这种废话越来越少了。即使在我非常刻板的日本大公司，他们也愿意在雇佣合同中加入一个例外条款，a)我在工作时间之外做的 OSS 工作，b) Bingo Card Creator。作为交换，我向他们提出:“如果你让我继续从事这些工作，我将学到很多技能，可以为公司所用。通常情况下，你会投入大量资金让工程师参加会议和专业培训。这对您来说更好:我将在没有运营支出和计费效率没有下降的情况下了解更多信息。”这是一个你可以向任何雇主发出的邀请。*

我更喜欢对人坦诚，而不是像很多人建议的那样“请求原谅比请求允许更容易”。当然，你可以孤注一掷，假装你的雇主不太可能注意到你的兼职项目。不幸的是，如果你的副业成功了，他们注意到你的副业的几率会急剧上升，然后你对它的不坦率会给你带来无限的责任，延伸到很远的将来。随便问问。他们能说的最糟糕的话就是“不”

你可以考虑在更一般的薪酬讨论中提问，而不仅仅是“嘿，老板，我能做 oss 吗？”这样，如果他们说“没有副业”，你会说“好吧，为了代替副业，我需要更多的钱。”当坚持通常的样板文件对公司来说完全没有成本时，坚持股票协议会更容易，但在样板文件上的微小让步往往比在公司账面上实际出现的事情上的让步更容易。

## 更喜欢在人们能看到你的地方工作

我曾经把这称为“在公共场合工作”，但是当人们想到在公共场合工作的人时，他们会想到摇滚明星，并且想“好吧，我永远也不会成为摇滚明星。”

在我们这个行业里，几乎没有人有摇滚明星的形象。他们仍然可以在“与他们专业相关的人”的观众中拥有实质性的个人资料这可能与“在我所在的大都市地区有权招聘前端开发人员的人”的范围一样严格，这可能是一组，什么，几十个人？

你是如何建立这种档案的？我建议，在同等条件下，在能见度高于平均水平的地方和项目中工作。

许多工程项目深深植根于晚期工业资本主义的内部。然后是编写脸书手机应用程序。我不知道实际上是哪些工程师在开发脸书移动应用，但我敢打赌，如果我是硅谷 iOS 或 Android 开发部门的招聘经理，我会 a)知道他们的名字，b)让他们位于或接近我个人挖人名单的顶端。

旁注:挖墙脚名单是我对“如果我有无限的钱，他们没有其他承诺，我会雇佣他们为某个特定项目工作”的非正式称呼。我有几个智力挖墙脚的名单——我所知道的在 Rails 编程、A/B 测试、写电子邮件等方面最优秀的人。当人们问我如何处理这些话题时，我经常说“你知道谁在这方面最棒吗？你不可能浪费时间带他们出去喝咖啡。”对于去喝咖啡的人来说，安排咖啡约会不会有什么不好的结果。(我的兴趣？帮助别人是一件有趣的事情，而且非常有趣的是，当你为他们找到一份工作或一名重要员工时，他们似乎会记得。)

你不必为“性感”的项目进行优化。你知道，性感的项目:我不知道如何描述他们，但我知道当我看到它。大多数工程工作本质上并不性感。然而，我会**优化影响和可见性**。

不要试图以优化 SQL 查询为职业，在一家没人听说过的公司的业务线应用程序上显示偏好页面。这不是让别人知道你有能力做有意义的工作的直接途径。相反，在知名度更高的公司/组织工作——AmaGooFaceSoft，初创公司或知名度极高的小公司(本地，全国，无论什么地方),或者在本质上你会接触到很多人的职位上工作。

我有几个朋友是开发者传道者，这是 API 公司创造的一个有趣的工作，你的任务基本上是“去给一群开发者演示我们的产品”。现在，在接下来的几年里，每天都这样做。”对这份实际工作的看法显然是复杂的。基思·凯西在这里给出了一个很好的解释。

(补充说明:如果我没有注意到 Keith 刚刚让 10，000 人分享了他的作品，那我就是失职了*，因为 Keith 做了这些工作，并使其易于分享*，也因为 Keith 通过他在 Twilio 的前一份工作认识了我。我是那里的顾客。在我们的行业中，每个人都可以玩六度凯文·贝肯，但实际上投入工作会让其他人更有可能代表你玩六度。)

总之，开发者福音。一个观察:我认识的每一个开发者福音传播者在他们不再是一个福音传播者后，马上进入了一个更好的工作。对于其他名声不佳的工程工作来说，情况就不一样了，比如建筑工人。为什么开发者福音传道者得到了升级，而构建人员却没有？我敢打赌，因为福音传道者确实花了数年时间会见成千上万的人，并向他们展示“嘿，我将在你们面前编写代码，同时还能让我的雇主赚一大笔钱。你经营一家公司，可能既需要工程师也需要钱。你也许应该记住我的名字，你知道，以防万一。”除了当 maven 触底或 RubyGems 出问题时，无论如何都是构建人员的错之外，构建人员忍受着不受重视的孤独。

如果你不能在你的日常工作中获得曝光率，试着在工作之外获得一些曝光率。积极联网。去参加当地的技术人员聚会，但是*也*去参加(通常是单独的)活动，在这些活动中，你所在行业的商业方面进行了交流。在会议上发言。把你创造的东西(见上)拿出来，主动展示给人们，征求反馈。你不一定要有成千上万的听众才会有价值——对于获得一份新工作来说，有一个招聘经理的听众要比没有听众好得多。写博客并收集电子邮件列表。这是一个老掉牙的建议，但它*非常管用*，尤其是当你可以多年复合改善的时候。

艾米·霍伊对此有一个很棒的比喻——[“堆砌砖块”](http://unicornfree.com/stacking-the-bricks)。从外表上看，你可能会说“那个事业辉煌的人？就像他们有一堵用牛逼做成的墙。我不能希望有一面这样的墙。”从内部来看，这看起来像是用一天的时间发表一个好的会议演讲，用几周的时间编写一个 OSS 库，用一天的时间写一篇关于让多个 Ruby 版本一起播放的权威博客，用几个月的时间发布一个被很多人使用的产品，用一个小时的时间录制一个播客。**一砖一瓦，一块石头一块石头，墙越建越高。**

## 更喜欢做你能保存的东西

雇主/雇员关系通常是“你给我们一小时，作为回报，我们给你一小时的报酬。”作为一名员工，你很少能掌握时间，为未来做准备，或者在几年后让它们为你带来收益。

我一般不喜欢硅谷模式，但我会为他们辩护:企业的广泛员工所有权是资本主义历史上最棒的创新之一。非管理层员工拥有 Twitter、脸书等公司正负 20%的股份。他们拥有几乎所有其他上市公司的正负“舍入误差”,只有极少数例外。

我认为这是对“无股份”雇佣模式的一个改进，但我不认为这是最终结论。首先，它将员工财富过度集中在一家公司。作为一名员工，你的短期现金流与雇主的持续健康息息相关。如果你的净资产有很大一部分与他们的股票挂钩，一旦发生长期或特定公司的冲击，你就放大了冲击的影响。(这也是为什么我不喜欢购买公司赞助的滴滴或个人退休账户中雇主的股票。你已经对他们的未来有了足够的了解，而无需用自己的钱购买更多。)

专业投资者的明确理解是，早期创业公司 90%的股票一文不值。职业投资者告诉员工“如果我们建议将整个基金押在一项投资上，即使我们认为这是板上钉钉的事情，我们的普通合伙人也会嘲笑我们，尽管如此，你会成为幸运的人，你应该*当然*将你的净资产的 99%套牢在一家特定公司的非流通股票上。”

那么如果不是雇主直接授予的硬资产，那又是什么呢？

显然，像每个理财顾问告诉你的那样存钱。(以下是你需要知道的一切:在你的税收优势账户中购买广义市场指数基金。如果这听起来太复杂，那就买一个 Vanguard target 退休基金，它的数字最接近你退休的大概时间。)

还有一个回报更高、更难的选择:副业。你可以用汗水“购买”它们，一次一粒。它们为你提供了许多好处，包括直接的财务收益(如果你把东西卖给别人赚钱，你就能得到钱，这可能是有用的)，投资财务收益的复合收益(我从 Bingo Card Creator 那里得到的第一笔 2000 美元变成了平均价格为每股 50 美元的 Chipotle 股票——不要买股票，买指数基金，但这个决定对我来说相当不错)。

拥有一件属于你的物品也有无形但同样真实的好处。这也是为什么，虽然我喜欢 OSS，但我建议人们不要马上把他们的 OSS 扔在 Github 上。这使得开发人员很容易使用您的代码，但却不容易向其他人展示代码的影响，尤其是向非技术利益相关者。从某种程度上说，人们的生活因您的代码而得到了有意义的改善，这种荣誉(和可观察到的引用)通常属于 Github，而不是您。如果你打算花几周或几个月的时间来编写有意义的 OSS 库，为它们创建一个独立的网站。

例子:我的 [A/Bingo](http://www.bingocardcreator.com/abingo/) 曾经可能是 Rails A/B 测试的最佳选择，因为它是 Rails A/B 测试唯一认真的选择。现在有点老了，但作为 A/B 测试员，我得到了几份咨询工作。制作文档、快速入门指南、徽标和品牌网站的努力胜过让潜在客户的初级工程师复制我的 Github URL，而根本不将我的工作展示给那里的决策者。

(非常喜欢 Github，伙计们。伟大的公司，伟大的产品，对行业的巨大影响。我只建议不要在一个人的项目中使用它们，原因很简单:我不为他们工作，我为我自己工作。如果我不为自己工作，别人也不可能为我工作。)

如果你想了解更多关于构建一个兼职项目的实际机制，我的博客会详细介绍。对于它的更简短的概述，我真的推荐 [Jason Cohen 在 Microconf 2013 上的演讲](http://www.microconf.com/videos-2013.html)。他的公式是“可预测的经常性收入的获得，带有年度预付费选项，产品解决了企业可论证的持久痛点。”这个想法在上面的链接中发展了一个小时，给出的许多建议都是具体的和可操作的。我强烈推荐。

## 消费有时是有价值的，但创造会推动你前进

我将以我通常给同行的建议作为结束:阅读这封邮件是有价值的。看杰森的视频很有价值。卷起袖子，实际运送一些东西要有价值得多。如果你再也不听我的其他建议，那就运些东西吧。你从一次失败中学到的要比从一千次成功中学到的多。你很有可能成功——人们大大高估了这有多困难。

只是不要一无所获地结束一周。
