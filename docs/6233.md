# 克里斯·萨卡的《还剩下什么？》:提议交易的提示。。。(或者，“我的天，这个收件箱一团糟”)

> 原文：<https://www.whatisleft.org/lookie_here/2005/09/want_to_do_busi.html?utm_source=wanqu.co&utm_campaign=Wanqu+Daily&utm_medium=website>

作为谷歌新业务开发的负责人之一，我站在引入业务提案的第一线。我想我每天能看到 40-50 个。Google Talk 的推出可能又增加了每天 15-20 个。

因此，当我战战兢兢地坐在那里，考虑到这个周日下午我的收件箱里有多少封邮件被标记为后续邮件时，我想我应该花一分钟时间打出一些提示，让我们双方都更容易，并增加你我公司完成一些业务的可能性。这些是随机排列的，随着时间的推移，我可能会扩展这个列表。一如既往，欢迎您的评论。我错过了什么吗？(请记住，这是我的个人博客，其中的任何内容都没有得到我的雇主的批准。)

<u>电子邮件规则</u>——电话很昨日。语音信箱坏透了。除非有人能真正让我轻松地从语音邮件中提取号码，将信息存档以备后用，给它贴上标签，快速地再次找到它，搜索它，转发它，在我方便的时候回复它，等等，否则我会坚持使用电子邮件。电子邮件棒极了(尤其是 [Gmail](http://gmail.google.com) )。这让我能确保我能回到你身边。这也有助于我把公司里所有对你的话题有影响的人都召集起来。所以，请，请，请-电子邮件。如果你坚持打电话，你会听到一条信息，说我不会检查你的语音信箱。所以，在你认为我粗鲁之前，我恳求你发电子邮件。
 <u>论文陈述</u>——以你想要的为先导。请放在第一两句话里。告诉我我们如何合作。你可以以后再详述，但请不要让我说得太远。当我不得不将你的信息转发给许多其他也在寻找论文陈述的高管时，这尤其困难。
 <u>你在解决什么问题？</u> -与论文陈述携手的是确定你正在解决的问题。很多时候，我与无法回答这个问题的人交谈。我们执着于解决能够提升我们最终用户体验的问题。我们很幸运，因为公司甚至没有要求我们通过每次改进来赚钱。相反，我们会因为对用户的影响而获得奖励。所以，请告诉我他们的体验现在坏了什么，我们如何一起修复它。(顺便说一下，我们对此非常谦虚，并意识到有无数种方法可以改善体验。请具体帮助我们。)此外，请记住，我们试图解决大问题。巨大的问题。每天影响数百万用户的问题。有鉴于此，你的一些利基建议可能很酷，但对我们来说，这往往归结为一个优先事项的问题。

<u>与众不同</u> -马上告诉我为什么你们与众不同，你们在市场上有什么比较优势。为什么你最能帮助我们？90%的时候，我们意识到我们需要帮助，但是我们需要帮助找到谁最适合做我们的伴侣。马上开始并加速这个过程。

跟进——重复发邮件跟进对我来说完全没问题，如果我落后了，可能还会有所帮助。这样做时，请提出具体的下一步措施。也就是说，当你越过我们去找其他高管时，没有什么帮助。在一天结束时，那些发给大人物的随机电子邮件会马上回到我的身边，只是让我对合作不太感兴趣。所以，如果需要的话，不要害怕给我发邮件。

<u>谷歌是自下而上的</u>——最后一点提醒我，谷歌的购买/合作决策是自下而上的。也就是说，产品经理和工程师是你的最终客户。当然，交易可能由副总裁签署，但他们只是认可项目经理的建议和领导。我们的管理团队真正让这里的人们能够在他们的空间内完成所需的交易。你想和 [Google Talk](https://www.google.com/talk/) 合作？我们的首席执行官不会去推动它。Google Talk 的产品经理将是你的人。因此，我的建议是避免与大人物会面，而是了解你的观众。我向您保证，您的交易会更快完成。

<u>会议并不总是必要的</u> -通常，供应商们都急于亲自见面。他们想带一个大团队来施压。我这样做了很长时间，很快就发现我的日程陷入了僵局。事实是，会议可能是低效的。先说邮件。给我们一副牌。也许接下来我们可以打个简短的电话。没必要和我开会建立关系。我发誓我在即时通讯网络上做过生意。
 <u>新发展协议不是一个有益的开始</u>——作为一家知识产权公司，新发展协议可能令人恐惧。在我们真正进入合作关系的实质之前要求我们签署一份协议可能会使事情变得复杂。我要求你不要和我们分享任何机密。我认为 NDA 在硅谷已经变得自动化了。然而，当人们停下来考虑它们和陷阱时，他们意识到它们在很多情况下是多余的。

<u>从工程开始</u>——我宁愿在房间里和技术人员开会，而不是只和业务人员开会。(注意:这是一个反对自我利益的声明，因为我只是一个想成为极客的人，而不是工程师出身。)我们 99%的合作决策的基础是工程师的评估。我们喜欢深入研究目前出现问题的具体细节，我们将如何解决这些问题，以及这种协作将会是什么样子，所有这些都需要具体说明。双方对技术的深刻理解是开始任何关于谁付钱给谁等问题的对话的前提。因此，提前感谢您将您的聪明人带入我们的聪明人的圈子。

公关是一种干扰——原则上，我们倾向于不和我们的伴侣做公关。事实是，我认为公关只是分散注意力。我知道作为一个初创企业，渴望一些公众意识是什么感觉。我绝对去过，可以同情。但是，PR 可以很空洞，不会给任何人增加多少具体的价值。专注于构建用户真正喜欢的酷东西是我们双方成功的最佳途径。相信我，当那发生的时候，你的公司和我的公司将会得到所有我们想要的曝光率，甚至更多。所以，现在让我们避开公共关系的讨论，只专注于让酷的东西工作。如果它发生了，世界很快就会知道。

威胁不起作用——数量惊人的人写信给我说“如果你 5 天内不采取行动，我就把这件事交给微软。。. "或者“这封信会转给特里·塞梅尔。。. "我非常倾向于放弃那些提议。对我来说，伙伴关系不仅关乎技术，也关乎合作伙伴。我不喜欢和那些想强迫我采取行动的人一起工作。微软和雅虎都是很棒的公司，拥有优秀的工程师。他们都在解决大问题，并在许多方面取得了很大的成功。我认为我们每个人都有自己的长处，最终，他们中的一个可能会成为你更好的伴侣。虽然我不会提前寻求你的排他性，请让我休息一下的威胁。我宁愿你把精力花在解释为什么谷歌有独特的优势成为你的好伙伴上。谢了。

<u>不要假设我们已经想到了 X</u>-对我来说，最有趣的事情之一就是阅读博客，看看人们对我们在特定领域的所谓下一步行动给予了多少信任。他们认为我们在某个地方有一份宏大的总体规划文件，并一步一步地制定未来几年的计划。事实是，我们在不断学习。我们倾向于尽早发布，并且经常发布。然而，这并不意味着我们已经想通了。你有什么好主意吗？我们是不是忽略了大局？你能帮助我们吗？开始吧。例如，你们这些已经考虑 VoIP 很多年的人，如果你是谷歌，你会怎么做，你会如何与我们合作来实现这个目标？

<u>考虑为我们工作</u> -如果以上所有这些都是你本能地想到的，并且你有很多绝妙的想法、完成事情的经验，并且正在寻找一个有趣的环境，在这个环境中你可以塑造/做出影响数百万互联网用户的决策，那么我强烈建议你来为我们工作。查看我们的[招聘页面](https://www.google.com/jobs/)，如果你发现任何有趣的东西，给我发电子邮件。

底线是我/我们希望与您和您的公司合作！合作伙伴关系是我们战略的重要组成部分，也是谷歌创造巨大价值的推动力。我们谦卑地承认，如果不与他人合作，我们就无法开始完成我们想做的事情的一小部分。也就是说，你可以想象，我们非常忙，所以如果有时我不能及时回复你，我请求你的原谅。希望记住以上所有内容，我们将一起完成更多工作。所以，发送这些建议，让我们做一些很酷的交易！谢了。
