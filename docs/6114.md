# 变革管理:来自战壕的建议

> 原文：<https://qz.com/work/1542486/change-management-advice-from-the-trenches/?utm_source=wanqu.co&utm_campaign=Wanqu+Daily&utm_medium=website>

有人曾经对我说“你擅长变革管理”，我当时就想，什么？

我对这个想法想了几个星期，最终意识到:哦！这就是我的工作。我带领正在努力的团队，帮助他们关注、调整并开始发挥他们的潜力，即变革管理。

关于变更管理的事情是，它涉及许多无形的工作，很难从外部跟踪。人们在变更管理中看到的两件事是变更(如果他们注意的话，是在接近尾声的时候)和何时发生了(有时是可怕的)错误。无论是作为管理者还是被变革者，以下是我从变革管理的战壕中学到的一些经验。

## 1.没有英雄

当我在一家失败的初创公司工作时，一位新的首席执行官进来做了一个小时的自我介绍(把我从那些认为自己很有趣的人那里救出来)，并给每个人发了一本书，讲述了一些英雄人物给公司带来的变革。

确实发生了变化。不到六个月，我不得不解雇了一半的团队成员，并在我的老板非常明智地参加工作面试的时候，监督产品的关闭。不久之后，我也离开了，不久之后，整个公司都关闭了。对我来说，这是一次学习经历。我的队友非常能干——我们所有人都做了令人敬畏的事情——但我怀疑首席执行官和他在一切崩溃前招募的人是否有同样的感觉。

在公司内部，最大的努力是团队努力，在转型时期更是如此。那些在困难时期轻而易举地吹嘘自己的资历并声称拥有某种“远见”地位的高管们，似乎永远不会成功。当有人声称自己是英雄时，人们听到的是他们不需要——或不想——别人站在他们一边。人们不会因为你告诉他们而接受你的想法；他们必须相信你。这需要更多的前期工作和倾听，而不是吹嘘。

## 2.高绩效者和低绩效者最挣扎

这个花了我一些时间去琢磨。当然，表现差的人会努力奋斗——他们是这样的(这是可以理解的！)害怕改变对他们意味着什么。然而，高绩效者似乎挣扎得更厉害，最终我意识到了原因:高绩效者已经找到了在现有系统中取得成功的方法，因此他们倾向于将问题视为其他人需要团结起来并变得有效。结果，变化看起来像是不必要的开销，很容易妨碍他们的实际工作。

本质上，低绩效者需要知道“是什么”——在新的事物秩序中的期望是什么——而高绩效者需要解释变化的“为什么”。

但是在你尝试向团队引入任何一种“绩效管理”之前，第一步总是引入标准、支持和责任。一旦你有了这一点，你就可以清楚地传达人们需要发展的地方，给表现差的人他们需要的帮助，帮助他们取得成功，如果还是不行，就让他们走。这不是一个简单的过程，但它是一个相对简单且有据可查的过程。

努力应对变化的高绩效员工对经理来说更具挑战性——首先，因为你可能觉得你更需要他们，而不是他们需要留下来，这使得反击变得更可怕；其次，因为高绩效员工往往在团队中拥有更多的社会资本，如果他们持怀疑态度，其他人也会怀疑。

那么，团队中每个人都喜欢并向其学习的人，你担心团队没有他就无法运作的人，出现在你面前并告诉你所有出错的事情的人？这是你需要获得认同的人。这会很难，但会非常有效，所以抓紧时间。完善你对他们的解释，倾听他们的意见，努力赢得他们的信任。给他们看你会带他们(还有其他人！)和你在一起。在这个过程中，对这个人来说，你可能会觉得你做的一切都太少，太晚了。然而，最终，当你们一起成功时，他们的反馈和认可会告诉你，你终于成功了。

## 3.所有过程都是上下文相关的

有时，人们观察团队并诊断问题是缺少过程，而不是缺少价值观、凝聚力或交付。他们制定了某种程序——例如，每周一次的站立会议——然后想知道为什么人们不相信它，为什么它不是他们所希望的奇迹。

缺乏过程很少是真正的问题，而是潜在问题的症状——这是你需要解决的问题。您仍然可以创建相同的流程，但是您应该确保该流程是对上下文问题 [的上下文答案，以服务于结果](https://qz.com/work/1389468/how-to-run-an-effective-meeting-avoid-process-performances/) 。

当我加入 Automattic 的移动团队时，没有人站着说话。但这不是问题所在。问题是有些人不互相交谈，表面问题，互相帮助。其他人有，所以对这些人来说，单口相声似乎是一个毫无价值的过程。但是让每个人都去一个可以做这些事情的地方真的很重要。

回顾是另一个过程，变更经理会试图强加给还没有这样做的团队。虽然回顾是一个有价值的目标，但它们很少是开始的地方。当团队努力进行回顾时，通常问题不是缺少回顾，而是缺少有效的回顾所需要的心理安全。这可能是由于团队缺乏信任或凝聚力，或者其他没有解决的问题。您需要首先解决这些事情，否则回顾将不会有成效，并且这个过程将是徒劳的。

站立和回顾是很好的例子，因为你不能把它们强加于人；大多数人必须出席并参与。你可能会觉得花了太多的时间让人们接受一些非常标准的东西，但有时这就是重点。如果你把精力花在一个过程上，人们会看到结果的改善，他们会更开放——希望开始暗示自己——接下来会发生什么。

## 4.你会犯错误

我作为变革经理犯的最大错误是在 2017 年，当时我在四个月内两次重组了我的团队。首先，我们按照部门和时区进行调整，而不是按照项目。我们做出选择的时间比我们应该做的要晚，因为恐惧和我们没有完全处理好的事情而拖延，紧接着我们比预期更快地招聘了员工——我们选择的结构显然不适合我们必须要做的入职数量，因此需要进行第二次重组。

当然，这不是我唯一的错误，但你不需要在这里列出它们的完整清单。

我向团队坦白了，解释了限制和理由，并且道歉了。虽然这对他们来说是破坏性的，但他们是理解和支持的。他们的原谅帮助我原谅了自己。

领导有一定的做法和原则，但它是一门艺术而不是一门科学，在团队转型的工作中更是如此。如果我们不是反社会者(我真的希望你们当中没有人是反社会的领导者)，我们允许团队中的人有犯错的空间，其中一部分就是承认，尽管我们有最好的意图，但我们也会有自己的意图。我们需要诚实、公开地承认这些错误，并为我们造成的破坏道歉。不这样做是一种煤气灯，破坏我们可能完成的其他一切。

没有办法绕过错误——只有通过。所以，尽你所能从他们身上学习，尽你所能向更多的人承认他们。下次你会做得更好；你的团队也会更愿意相信这一点。
