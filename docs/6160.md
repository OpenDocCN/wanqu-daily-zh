# 微软、Slack、Zoom 和 SaaS 的机会

> 原文:[https://strate Chery . com/2019/Microsoft-slack-zoom-and-the-SaaS-opportunity/？UTM _ source = Wanqu . co&UTM _ campaign = Wanqu+Daily&UTM _ medium = website](https://stratechery.com/2019/microsoft-slack-zoom-and-the-saas-opportunity/?utm_source=wanqu.co&utm_campaign=Wanqu+Daily&utm_medium=website)

讨论企业软件很难不提到微软，现在是最好的时机:上周，该公司(短暂地)成为继苹果和亚马逊之后第三家市值超过 1 万亿美元的美国公司，也是目前世界上最有价值的上市公司。

这是一个显著的转变，而且，由于我在前首席执行官史蒂夫·鲍尔默离职前几个月开始了 Stratechery，这是一个我能够逐步记录的转变。关键的突破是三重的，而且，正如经常发生的那样，这三个突破实际上是关于同一个存在主义问题——窗户在哪里:

*   首先，微软的领导层[承认其本质是一个横向公司](https://stratechery.com/2017/microsofts-monopoly-hangover/)，而不是围绕 Windows 建立的高度差异化的纵向公司。
*   第二，微软[拥抱了一个 Windows 是众多](https://stratechery.com/2016/the-curse-of-culture/)客户端之一的世界，并将其服务定位于 iPhone、Android、Linux 和 Mac。
*   第三，也是最重要的一点，塞特亚·纳德拉在内部出色地引导了 Windows T1 的终结，让微软员工可以自由地开发客户真正想要的产品，而不是微软需要的产品。

然而，使这一切成为可能的最重要的因素是，尽管由于软件即服务(Saas)的兴起，企业市场面临着所有的中断，但微软非常适合利用这一新的范式，只要他们能够走出自己的道路。

至少部分是。

#### SaaS 的商业模式

任何新的技术范式都有三个部分:更好地完成当前用例，提出新的业务模型，以及创建全新的用例。值得称赞的是，微软实际上很早就开始关注 SaaS 的新商业模式。

以前，企业软件是以许可证为基础销售的:公司购买软件是以席位为基础的(或者后端软件是以服务器或内核为基础的)，当软件的新版本出来时，他们可能会更新，或者不更新。或者没有对任何人来说都不是好事:公司将运行过时的软件，供应商将不会获得新的收入。

微软发现，对微软和他们的客户来说，付费订阅更有意义:公司可以按月或按年支付固定价格，获得最新最好的软件。这并不是万灵药——升级软件仍然是一项重大任务——但至少避免升级的动机被消除了。

从资产负债表的角度来看，这也有微妙的优势:现在，公司为软件支付的费用大致相当于软件在一段时间内的使用量，即运营费用，而不是固定成本。这改善了他们的投资资本回报率(ROIC)测量，如果没有别的。对微软来说，收入变得更加可预测。

#### SaaS 和当前用例

不过，SaaS 更深刻的含义是——为了清楚起见，我指的是通过互联网访问的软件，而不是在订阅基础上付费的数据中心软件——它如何一方面使现有企业客户的当前用例更加高效，另一方面使全新客户可以访问。

从企业客户开始:许多行业的现实是他们的需求是可变的。有时他们需要更多的座位来安装特定的软件，有时他们需要更少的座位；这在基础设施即服务(IaaS)的情况下尤为明显，因为计算需求可能会随季节而变化。订阅付费的好处在于，公司可以在需要的时候购买自己需要的东西，仅此而已。微软，再次归功于鲍尔默，正在用 Office 365 这样做:席位可以按月提供，不需要大量的前期支出。

然而，前期支出的缺乏也扩大了市场:在 Office 365 上购买一个
Exchange 席位意味着雇佣微软来运行你的电子邮件服务器，这在以前需要在内部完成。现在，各种中小型公司都可以使用企业级软件，而不需要他们自己的 IT 部门。

#### 微软的 SaaS 挑战

当然，这对微软来说也是一个挑战:“雇佣”SaaS 提供商意味着更容易找到真正关心现代用例，尤其是移动用例的提供商。我在 2015 年的 [Redmond 和【Reality 中写了关于云存储的文章:](https://stratechery.com/2015/redmond-reality/)

> 一旦您消除了支持和维护的负担(这由服务提供商处理)，仅仅因为它们是集成的，就从一个供应商那里购买就没有意义了。有更多的自由来评估一个特定产品的不同特征，比如说，它有多容易使用，或者它对移动的支持有多好。正是在这一点上，微软的产品，尤其是令人讨厌的 SharePoint，被发现有所欠缺。

这就是纳德拉最大的不同之处。从象征的角度来看，值得注意的是，如果没有其他原因的话，他的第一个公开活动是为 iPad 发布 Office:

> 这就是首席执行官的权力。他们不能做所有的工作，也不能影响他们无法控制的行业趋势。但他们可以选择是否接受现实，这样做可以影响他们所领导的所有人的世界观。这就是为什么塞特亚·纳德拉出现的第一场公开活动是 iPad 的发布会。这就是为什么尽管 Windows Touch 版本还没有完成，微软还是发布了它。这就是为什么微软放弃了 Windows Phone 许可证支付的伪装(实际上已经为零),而只是让它免费，这很重要。

对于 SaaS 提供商来说，这是唯一可能的途径:重点是在一个地方托管所有的基础设施，这意味着最大可能的收益来自于扩大可寻址市场，这进一步意味着服务于所有设备，而不仅仅是公司拥有的设备。不过，这就是现任的负担:在公司现金流，尤其是公司文化方面，从外部看来明显正确的东西，往往与明显正确的东西背道而驰。

#### 放大并变得更好

雷德蒙与现实是关于文件共享软件的，但更广泛的想法——SaaS 将竞争平面从易于集成转变为易于使用——也许最好的例子是 Zoom 的崛起。事实证明，视频会议软件是一个异常困难的技术问题，Zoom 在解决这些技术挑战方面做得比任何人都好。它比其他选择更好。

即便如此，如果所述视频会议软件必须通过内部软件安装来交付，那么 Zoom 是否会像现在这样成功就值得怀疑了:同样重要的是，注册 Zoom 只需要一个电子邮件地址；付费计划只接受信用卡。摩擦的减少意味着质量比以前更加重要，这也是 Zoom 如此成功的原因。

对现有公司的挑战，包括微软和其他竞争对手，如 Citrix、Cisco 等。多年来，他们利用与企业的现有关系建立业务，这使他们容易受到像 Zoom 这样专注于提供优质产品的公司的攻击，至少曾经有一次，SaaS 的架构使分销变得如此容易。毫无疑问，企业软件仍然需要一支销售队伍，但是从已经发现并尝试过该产品的客户开始比销售没有任何预先存在关系的产品要容易得多。

#### 松弛和新用例

然而，新范例还有最后一个含义，这是最深刻的:全新的用例。这是 Slack 试图在上周公开的 S-1 中强调的。

首先，该公司认为 Slack 改变了内部沟通方式:

> 对 Slack 最有帮助的解释通常是，它在组织内部取代了电子邮件的使用。像电子邮件(或互联网或电力)一样，Slack 具有非常普遍和广泛的适用性。它不是针对任何一个特定的目的，而是人们在工作中一起做的几乎任何事情。
> 
> 然而，与电子邮件不同，这种活动大多发生在基于团队的渠道中，而不是在个人收件箱中。通道提供了与项目或主题相关的对话、数据、文档和应用程序工作流的持久记录。随着人们加入或离开一个项目或组织，渠道的成员资格会随着时间的推移而改变，用户可以从积累的历史信息中受益，而这是员工从空的电子邮件收件箱开始工作时所无法获得的。根据组织的规模，这可能提供比在以电子邮件为主要通信手段的环境中工作的个人多几十倍、几百倍甚至几千倍的信息访问。

其次，Slack 认为它改变了集成软件的含义:

> 与电子邮件不同的是，Slack 的设计初衷是为了与外部软件系统集成。Slack 为用户提供了一种简单的方式来共享和聚合来自其他软件的信息，对通知采取行动，并在众多第三方应用程序中推进工作流，Slack 应用程序目录中列出了超过 1，500 种应用程序。此外，Slack 的平台功能超越了与第三方应用程序的集成，并允许与组织内部开发的软件轻松集成。在截至 2019 年 1 月 31 日的三个月内，我们超过 1000 万的日活跃用户包括超过 50 万的注册开发者。在截至 2019 年 1 月 31 日的三个月中，开发人员共同创建了超过 450，000 个第三方应用程序或自定义集成，这些应用程序或集成在典型的一周内使用。此外，我们目前正在开发低代码解决方案，以完全在 Slack 中创建集成和工作流，适合所有用户，并基于简单的非技术用户界面。

当引入一种全新的工作方式时，有两个方面的挑战:首先，你必须让公司相信新的工作方式更好，其次，你必须切实帮助他们实施。正是在这一点上，互联网对企业软件的影响最为深远:

*   首先，互联网本质上是病毒式传播的，这要归功于信息可以零边际成本传播的事实。就 Slack 而言，告诉别人它的好处只需要在社交媒体上发一篇帖子，随着时间的推移，邀请加入 Slack 团队。
*   其次，与前面的观点相关，Slack 提供免费产品实际上是划算的:不需要客户安装，只需要数据库中的几个条目。
*   第三，实施是一个付出的问题——仅此而已。使用稀缺的 IT 资源没有任何顾虑，只是一个关于成本的问题，这个决定通常基于最初免费的实现。

这就解释了为什么我认为 Slack 是互联网对企业软件市场影响的典型代表:在某些方面，Zoom 是一个更令人印象深刻的业务，但它的用例是预先存在的。另一方面，Slack 引入了一种全新的工作方式，并基于其 S-1，这样做的方式将随着时间的推移产生一个非常有利可图的公司(Slack 正在亏损，但亏损率远低于其收入增长；这是一家对其成本有杠杆作用的公司，未来将非常盈利)。

#### 微软缺少什么

毫无疑问:推动(近)万亿美元估值的微软乐观主义是有道理的。当然，Azure 是最大的原因，但 Office 365 受益于我上面描述的所有动态:[正如我上周](https://stratechery.com/2019/microsofts-earnings-microsofts-growth-opportunities-facebooks-earnings/)指出的那样，它的市场在现有客户、它已经服务的公司的新用户以及向所有这些用户追加销售新功能方面都在增长。

与此同时，使用微软的*原因*在很大程度上是基于过去的:人们熟悉 Office 文档，Exchange 仍然是企业电子邮件的标准。与微软合作的好处是一切都和以前一样。不过，这提出了一个纳德拉的微软尚未回答的存在主义问题:为什么一家与微软的工作流程没有任何关联的新公司会选择 Office 365？

请注意，这是一个关于团队(Office 365 对 Slack 的回答)是否可行的单独问题:分发在企业软件中仍然很重要，团队具有宝贵的优势，这些优势来自于与微软其他产品的集成。

与此同时，即使是对团队有利的情况也是它抓住了微软现有用户群的一个*部分*:

这正是你对利用现有用例和现有客户关系的产品的期望。相比之下，Zoom 首先解决了现有用例与获取新客户关系的需求:Zoom 在建立初始客户群方面面临挑战，但在此基础上，他们在新用例以及深化与客户的互动方面都有增长机会。

Slack 的机会更加引人注目:由于从新客户和新用例开始，吸收现有用例(总是比创建新用例容易)以及深化现有客户的效用的机会是巨大的。这就是为什么你会得到这样的 IPO 图表:

[![Slack's cohort growth](../Images/8a205fbb355a625b25f9b7fead5c9568.png)T2】](https://i0.wp.com/stratechery.com/wp-content/uploads/2019/04/Screen-Shot-2019-04-30-at-7.46.35-PM.png?ssl=1)

Slack 不仅在增加用户，随着时间的推移，它还在增加这些用户的货币化，预计这两者都将继续。这正是微软所缺乏的:该公司充其量是将现有的微软用户转移到 SaaS 模式，并让他们远离 Zoom 或 Slack 这样的公司。然而，从长远来看，这并不是增长的良方。

#### 企业成长框架

您可以在这两个向量上绘制这三种产品的图表——预先存在的客户关系和预先存在的客户用例:

[![Use case versus existing customer relationships in enterprise software](../Images/9e2fa0fcdabeafc74a49db07937b3439.png)T2】](https://i0.wp.com/stratechery.com/wp-content/uploads/2019/04/Stratechery-2019-04-30-19.29.28.png?ssl=1)

这就是纳德拉的微软的不足之处。该公司做得很好，利用其现有的优势与现有客户建立了更有价值的关系，并成为新客户的可行选择，正如我在上文提到的，它确实进入了新的用例；团队明显位于上图的右下方:

[![Teams expands the use cases within Microsoft's existing userbase](../Images/f5c12ba0e6d6f0b6ec88db2937e12caf.png)T2】](https://i0.wp.com/stratechery.com/wp-content/uploads/2019/04/Stratechery-2019-05-01-19.31.23.png?ssl=1)

问题是，Teams 之所以成功，是因为它利用了微软现有的客户群，而不一定会赢得那些一开始就不会考虑微软的客户。没有足够的行业领先技术(如 Zoom)或新用例中的创新(如 Slack ),无法让人相信公司可以超越现有的客户关系长期发展。这就是像 Zoom 和 Slack 这样的公司如此有价值的原因:它们创造了准备增长的新客户；与此同时，微软大部分时间都把现有客户留在公司内部。

那么，这就是纳德拉的新挑战:该公司本可以在纳德拉任期早期收购 Slack，并考虑 Zoom，但在这两个方面都等了太久。微软已经找到了如何利用现有用户群的方法:如何扩大用户群仍是一个悬而未决的问题。
