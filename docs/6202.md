# 战略字母 V–乔尔谈软件

> 原文:[https://www . joelonsoftware . com/2002/06/12/strategy-letter-v/？UTM _ source = Wanqu . co&UTM _ campaign = Wanqu+Daily&UTM _ medium = website](https://www.joelonsoftware.com/2002/06/12/strategy-letter-v/?utm_source=wanqu.co&utm_campaign=Wanqu+Daily&utm_medium=website)

当我在大学时，我选了两门经济学入门课程:宏观经济学和微观经济学。宏观充满了像“低失业率导致通货膨胀”这样的理论，从来没有完全站得住脚的现实。但是微型的东西既酷又有用。书中充满了关于供求关系的有趣概念，这些概念确实有效。例如，如果你有一个降低价格的竞争对手，除非你和他们竞争，否则对你产品的需求就会下降。

在今天的节目中，<font>我将展示其中一个概念是如何解释一些熟悉的电脑公司的。一路走来，我注意到开源软件的一些有趣之处，那就是:大多数公司花大价钱开发开源软件是因为这对他们来说是一个好的商业策略，而不是因为他们突然不再相信资本主义，爱上了[言论自由](http://www.gnu.org/philosophy/free-sw.html)。</font>

市场上的每一种产品都有*替代品*和*补充*。替代品是第一种产品太贵时你可能会购买的另一种产品。鸡肉是牛肉的替代品。如果你是一个养鸡户，牛肉价格上涨，人们会想要更多的鸡肉，你就会卖出更多。

补充品是指通常与另一种产品一起购买的产品。汽油和汽车是互补的。计算机硬件是计算机操作系统的经典补充。保姆是高级餐厅晚餐的补充。在一个小镇上，当当地的五星级餐厅推出买一送一的情人节特价时，当地的保姆会将价格翻倍。(实际上，9 岁的孩子被拉去做早教。)

在其他条件相同的情况下，当一种产品的补充品价格下降时，对这种产品的需求就会增加。

让我重复一遍，因为你可能打瞌睡了，这很重要。当一种产品的补充品价格下降时，对这种产品的需求就会增加。例如，如果飞往迈阿密的航班变得更便宜，对迈阿密酒店房间的需求就会上升——因为更多的人飞往迈阿密，需要一个房间。当计算机变得更便宜时，更多的人购买它们，它们都需要操作系统，所以对操作系统的需求上升，这意味着操作系统的价格可以上升。

在这一点上，人们试图通过说“啊哈！但是 Linux 是免费的！”好的。首先，当经济学家考虑价格时，他们会考虑总价格，包括一些无形的东西，如设置、再教育每个人和转换现有流程所需的时间。我们喜欢称之为“总拥有成本”的所有东西

第二，通过使用啤酒中的免费论点，这些倡导者试图相信他们不受经济学规则的约束，因为他们有一个很好的零，可以乘以一切。这里有一个例子。当 Slashdot [问](http://interviews.slashdot.org/interviews/02/06/07/1255227.shtml?tid=156) Linux 开发者 Moshe Bar 未来的 Linux 内核是否会兼容现有的设备驱动程序时，他说不需要。“专有软件的价格是每行调试代码 50-200 美元。这样的价格不适用于开源软件。”Moshe 继续声称，每个 Linux 内核版本都可以淘汰所有现有的驱动程序，因为重写所有这些现有驱动程序的成本是零。这是完全错误的。他基本上声称，花费少量编程时间使内核向后兼容，相当于花费大量编程时间重写每个驱动程序，因为这两个数字都乘以它们的“成本”，他认为成本为零。这是一个明显的谬误。开发人员花费数千或数百万小时来修改每一个现有的设备驱动程序将不得不以牺牲一些东西为代价。在此之前，Linux 将再次在市场上受到阻碍，因为它不支持现有的硬件。用所有“零成本”的努力让 Gnome 变得更好不是更好吗？还是支持新硬件？

被调试的代码不是自由的，无论是私有的还是开源的。即使你不为它支付现金，它也有机会成本和时间成本。可用于开源工作的志愿者编程人才是有限的，每个开源项目都与其他开源项目竞争相同的有限编程资源，只有最性感的项目才真正拥有超过他们可以使用的志愿者开发人员。总而言之，我对那些试图证明免费软件的疯狂经济价值的人印象不深，因为就我而言，他们得到的只是被零除的错误。

![Eazel closed.](../Images/b67b113bc5a89512e49b420c99bc103b.png) 开源并不能免除万有引力定律或经济学。我们从 Eazel(T4)、ArsDigita(之前被称为 VA Linux 的公司)和许多其他尝试中看到了这一点。但是，开源世界中很少有人真正理解的事情仍然在发生:许多非常大的上市公司，有责任使股东价值最大化，正在投入大量资金支持开源软件，通常是通过支付大量程序员团队来工作。这就是互补原理的解释。

再说一次:当一种产品的补充品价格下降时，对这种产品的需求就会增加。一般来说，一家公司的战略利益是让他们的产品价格尽可能低。理论上可持续的最低价格是“商品价格”——当你有一群竞争对手提供不可区分的商品时出现的价格。所以:

| 聪明的公司试图将他们产品的补充物商品化。 |

如果你能做到这一点，对你的产品的需求就会增加，你就能收取更多的费用，制造更多的产品。

当 IBM 设计 PC 架构时，他们使用现成的部件而不是定制的部件，他们在(革命性的) [IBM-PC 技术参考手册](http://news.com.com/2100-1001-271354.html?legacy=cnet)中仔细记录了部件之间的接口。为什么？以便其他制造商也能加入这个派对。只要配接口，就可以用在 PC 里。 **IBM 的目标是将插件市场商品化，**这是对 PC 市场的补充，他们做得相当成功。在很短的时间内，出现了数十家提供存储卡、硬盘、显卡、打印机等产品的公司。廉价的插件意味着对个人电脑的更多需求。

当 IBM 从微软获得操作系统 PC-DOS 的许可时，微软非常小心地不出售独家许可。这使得微软有可能将同样的东西许可给康柏和其他数百家使用 IBM 自己的文档合法克隆 IBM PC 的原始设备制造商。微软的目标是将个人电脑市场商品化。很快，个人电脑本身基本上成了一种商品，价格越来越低，功能越来越强，利润率极高，很难盈利。当然，低价增加了需求。对个人电脑需求的增加意味着对其配套产品 MS-DOS 需求的增加。在其他条件相同的情况下，对一种产品的需求越大，它给你赚的钱就越多。这就是为什么比尔·盖茨可以买下瑞典而你不能。

今年，微软试图再做一次:他们的新游戏主机 XBox 使用商用 PC 硬件，而不是定制部件。该理论(在本书中解释)是，硬件商品每年都变得更便宜，所以 XBox 可以压低价格。不幸的是，这似乎适得其反:显然商用 PC 硬件已经被挤压到了商品价格，因此 XBox 的价格下降速度没有微软希望的那么快。微软 XBox 战略的另一部分是使用 DirectX，这是一个图形库，可用于编写在各种视频芯片上运行的代码。这里的目标是使视频芯片成为一种商品，降低其价格，从而销售更多的游戏，真正的利润发生在那里。为什么世界上的视频芯片供应商不设法将游戏商品化呢？这比 T2 难多了。如果游戏*光环*卖得很疯，那它真的*没有*任何替代品。你不会去电影院看《星球大战:克隆人的进攻》,而是决定看一部伍迪·艾伦的电影就满足了。他们可能都是伟大的电影，但他们不是完美的替代品。现在:你更愿意成为游戏发行商还是视频芯片供应商？

将你的补充商品化。

理解这一策略实际上对解释为什么许多商业公司对开源做出巨大贡献大有帮助。让我们检查一下这些。

**头条:IBM 斥资数百万开发开源软件**。

神话:他们这样做是因为娄·郭士纳读了 GNU 宣言，决定他其实不喜欢资本主义。

事实:他们这样做是因为 IBM 正在成为一家 IT 咨询公司。IT 咨询是企业软件的补充。因此，IBM 需要将企业软件商品化，而最好的方法就是支持开源。你瞧，他们的咨询部门凭借这一策略大获全胜。

标题:网景开放他们的网络浏览器。

神话:他们这样做是为了从新西兰网吧的人们那里获得免费的源代码贡献。

事实:他们这样做是为了使网络浏览器商品化。

从第一天起，这就是网景公司的战略*。看看[最早的网景新闻稿](http://wp.netscape.com/newsref/pr/newsrelease1.html):浏览器是“免费软件”网景公司免费提供浏览器，这样他们就可以在服务器上赚钱。浏览器和服务器是经典的补充。浏览器越便宜，你卖出的服务器就越多。这从未像 1994 年 10 月那样真实。(事实上，当 MCI 走进门来，给网景公司扔了这么多钱，让他们意识到他们也能从浏览器中赚钱时，网景公司是[感到惊讶的。这不是商业计划所要求的。)](http://wp.netscape.com/newsref/pr/newsrelease4.html)*

当 Netscape 将 Mozilla 作为开源软件发布时，是因为他们看到了降低浏览器开发成本的机会。这样他们就能以较低的成本获得商品利益。

后来美国在线/时代华纳收购了网景公司。服务器软件，本应是商业浏览器的受益者，却表现不佳，被抛弃了。现在:为什么美国在线/时代华纳会继续投资开源？

美国在线/时代华纳是一家娱乐公司。娱乐公司是所有类型的娱乐交付平台的补充，包括网络浏览器。这个巨型企业集团的战略利益是让娱乐交付——网络浏览器——成为一种无人能收费的商品。

我的论点被 Internet Explorer 像啤酒一样免费的事实所折磨。微软也想让网络浏览器成为一种商品，这样他们就可以销售桌面和服务器操作系统。他们更进一步，发布了一系列组件，任何人都可以用它们来组装一个网络浏览器。Neoplanet、AOL 和 Juno 使用这些组件来构建自己的 web 浏览器。鉴于 IE 是免费的，网景让浏览器“更便宜”的动机是什么？这是先发制人之举。他们需要阻止微软完全垄断网络浏览器，即使是免费的网络浏览器，因为理论上这会给微软一个机会，以其他方式增加网络浏览的成本——比如说，通过提高 Windows 的价格。

(我的论点更加站不住脚，因为很明显，巴克斯代尔时代的网景公司根本不知道自己在做什么。对于网景公司的所作所为，一个更可能的解释是，上层管理人员在技术上是无能的，他们别无选择，只能遵从开发人员提出的任何方案。开发者是黑客，而不是经济学家，只是巧合地想出了一个服务于他们策略的方案。但是让我们假定他们是无辜的。)

标题:Transmeta 雇佣 Linus，付钱让他黑 Linux。

误解:他们这样做只是为了引起公众的注意。否则你会听说过 Transmeta 吗？

真相:Transmeta 是一家 CPU 公司。CPU 的自然补充是操作系统。Transmeta 希望 OSs 成为一种商品。

**Headline:孙和惠普出钱让西棉黑 Gnome。**

神话:Sun 和 HP 支持自由软件是因为他们喜欢集市，而不是教堂。

真相:Sun 和惠普都是硬件公司。他们做博兴。为了在台式电脑上赚钱，他们需要视窗系统，作为台式电脑的补充，成为一种商品。他们为什么不把付给 Ximian 的钱拿去开发一个专有的视窗系统呢？他们尝试过这一点(Sun 有 NeWS，HP 有 New Wave)，但这些实际上是硬件公司，软件技能相当粗糙，他们需要窗口系统成为廉价商品，而不是他们必须花钱购买的专有优势。所以他们雇佣了西棉的优秀员工来做这件事，原因和 Sun 收购 Star Office 并开源它的原因一样:将软件商品化，在硬件上赚更多的钱。

**头条:孙开发 Java 新的“字节码”系统意味着写一次，在任何地方运行。**

字节码的想法并不新鲜——程序员总是试图让他们的代码在尽可能多的机器上运行。(这就是你如何商品化你的补充)。多年来，微软拥有自己的 p-code 编译器和可移植窗口层，使 Excel 可以在 Mac、Windows 和 OS/2 上运行，也可以在摩托罗拉、英特尔、Alpha、MIPS 和 PowerPC 芯片上运行。Quark 有一个在 Windows 上运行 Macintosh 代码的层。C 编程语言最好被描述为独立于硬件的汇编语言。对于软件开发人员来说，这不是一个新的想法。

如果你能在任何地方运行你的软件，那就使得硬件更像是一种商品。随着硬件价格下降，市场扩大，推动了对软件的更多需求(并使客户有额外的钱花在现在可能更贵的软件上。)

孙对的热情是，嗯，*奇怪的*，因为孙是做硬件公司的。让硬件成为商品是*最不想做的事情。*

哦哦哦哦哦哦哦哦哦！

Sun 是计算机行业的老大。由于看不到过去他们对微软的极度恐惧和厌恶，他们采取了基于愤怒而非自身利益的策略。Sun 的两个策略是(a)通过推广和开发自由软件(Star Office、Linux、Apache、Gnome 等)使软件成为商品，以及(b)通过推广 Java 及其字节码体系结构和 WORA 使硬件成为商品。好，孙，来个小测验:当音乐停止时，你要坐在哪里？如果没有硬件或软件方面的专有优势，你将不得不接受商品价格，这只能勉强支付瓜达拉哈拉廉价工厂的成本，而不是你在硅谷舒适的办公室。

“但是乔尔！”杰瑞德说。" Sun 正试图将操作系统商品化，比如 Transmeta，而不是硬件."也许吧，但是 Java 字节码也商品化了硬件，这是一个相当大的附带损害。

从所有这些例子中你注意到的一个重要的事情是，软件很容易将硬件商品化(你只需编写一个小小的硬件抽象层，就像 Windows NT 的 HAL，这是一小段代码)，但硬件将软件商品化却非常困难。软件是不可互换的，StarOffice 营销团队正在学习这一点。即使价格为零，微软 Office 的切换成本也是非零的。在转换成本变为零之前，桌面办公软件不是真正的商品。即使是最小的差异也会让两个软件包之间的切换变得很痛苦。尽管 Mozilla 拥有我想要的所有功能，我也很想用它来避免打地鼠弹出广告游戏，但我太习惯按 Alt+D 键去地址栏了。所以起诉我吧。一个微小的差异，你就会失去你的商品地位。但是我已经把硬盘从 IBM 电脑中取出来，并把它们猛地放到戴尔电脑中，然后，嘭，系统就完美地出现了，并且运行起来就好像它还在旧电脑中一样。

[Creo](http://www.creo.com/) 的首席执行官阿莫斯·迈克尔逊告诉我，他公司的每个员工都被要求参加一门他称之为“经济思维”的课程好主意。即使是基本微观经济学中的简单概念，也对理解今天正在发生的一些根本性转变大有帮助。
