# 学习编程的两种方法:自顶向下和自底向上

有两种学习编程的通用方法，各有利弊。

## 自上而下的方法

使用自顶向下的方法，目标是通过创建真正软件的重要部分来学习。

大多数自我导向的学习者在他们第一次开始学习编码时采用自顶向下的方法。他们想马上做出一些很酷的东西——也许是一个 3D 游戏，或者一个好看的互动网站。

让事情变得有意义的最快方法是遵循一个教程——特别是一个非常详细的教程。如果你严格按照这些步骤去做，你一定会有所创造。

### 自上而下:好的

自上而下方法的美妙之处在于你可以快速地创造出一些东西。用自己的指尖制作一个真正的软件感觉很好。你可能会沉迷于这种感觉，它可以帮助你完成学习过程。

### 自上而下:坏的

自上而下的方法的问题在于它没有教授基础知识。

在教程结束时，你可能根本不明白你的作品是如何工作的。如果教程不够详细，你可能会努力去填补空白。如果你以最轻微的方式偏离指示，它将会中断，并且你可能无法诊断问题。

这就像你还没学会游泳，就被直接扔进了游泳池的深水区。

阿恩·布拉瑟尔[反思](https://lambdaisland.com/blog/25-05-2017-simple-and-happy-is-clojure-dying-and-what-has-ruby-got-to-do-with-it)教授自上而下的方法:

> 我教过很多人 Ruby on Rails。Rails 非常复杂，但在几堂课上，你就可以教一个人创建一个由数据库支持的交互式网站。教书令人沮丧，因为你在走钢丝。按照步骤来。不要在被子下偷看。不要凝视深渊。但是那种立刻变得富有成效的感觉吸引了人们，让他们想，“嘿，也许我可以做这个”。

## 自下而上的方法

使用自下而上的方法，目标是学习所有编程的基本概念。

自底向上的方法在正式的编程课程中更受欢迎，比如大学或学院中的课程。它包括从零开始，一次学习一个概念。这个想法是为了建立一个通用编程技能的坚实基础，它可以用来制作任何类型的软件。

### 自下而上:好的

自底向上方法的美妙之处在于，你可以学到真正通用的编程技能。不管你是在制作一个 3D 游戏，还是一个互动网站，编程的基本概念适用于一切。

每个单独的概念都更容易学习，因为你可以孤立地学习它。如果你正在学习的自下而上的课程结构良好，你就不会一下子被 1000 个不同的概念轰炸。你应该一次看到一个新的概念，按照容易理解的顺序。基本概念排在第一位(比如值和变量)，在更复杂的概念(比如函数)之前。

### 自下而上:不好的

自下而上方法的问题是它很慢。要想让一件事情变得有意义，需要花很长时间去学习所有必要的技能。典型的自下而上计划可能如下所示:

*   首先你要学习字符串，整数和变量。
*   然后你会学到布尔、分支和循环。
*   然后你学习数组和结构。
*   然后你学习函数和控制流。
*   然后老师告诉你，大部分专业人士已经不这么编码了。
*   所以你开始学习类，实例，实例变量，方法，继承，以及一大堆其他面向对象的编程概念。
*   然后你试着忘记你最初学习写代码的方式，学习用新的面向对象的方式写代码。

你可能要花几个月或几年的时间来完成这个计划，而且它也不会给你制作一个 2D 游戏或互动网站的所有技巧！

当它花费很长时间，而你没有创造任何有趣的东西时，它会变得很无聊。很多人发现花一周时间学习控制流并不如花一天时间做一个网站有趣。如果你没有获得成就感，你就不太可能继续学习。

## 哪种方法最好？

这两种方法都不比另一种好。要成为一名熟练的程序员，两者都需要。

你需要创造软件的经验，即使你并不完全理解它。你需要继续学习的动力，那种成就感会有帮助。仅仅学习概念和理论不足以让你为真正的软件项目做好准备。

你*也*需要基本的编程技能。需要能够理解和调试代码——包括您自己的代码和其他开发人员编写的代码。你需要能够在你从未遇到过的情况下编写代码，因为不会有针对你需要解决的每个不同问题的教程。

最好的方法是在两者之间交替。

感觉不知所措，好像不知道代码在做什么？改用自下而上的方法。回到基础，准确地学习你的代码是如何工作的。试着填补你知识上的空白。

厌倦了学习抽象的编程概念？改用自上而下的方法。创建一些小的东西，比如一个简单的网站或游戏。试着把你学过的概念都演练一遍，获得那种成就感。

最终，这两种方法在中间相遇。您学到了很多关于代码如何工作的知识，这给了您解决任何问题的信心。您还将学习编程概念和理论如何应用于现实世界的项目——在哪里使用它们，在哪里不使用它们，以及如何很好地使用它们。

这个学习过程永无止境。软件开发非常复杂，没有一个人能理解所有的事情，这意味着总有新的东西要学。所以，不要太担心你还不知道的事情——这只是需要时间。我的建议是放松，享受旅程。

## 《编程初学者入门》一书

[初学者编程](https://leanpub.com/programming-for-beginners)是一种自下而上的学习编程的方法。

它从最底层开始，假设你对编程一无所知。基本概念一次介绍一个，顺序是为了让你的大脑容易吸收。

为了保持这本书的趣味性，并提供成就感，每章都以编程挑战结尾。鼓励读者制作一个小的、基于文本的游戏，只使用到目前为止学到的编程概念。因为这本书采用了自下而上的方法，游戏非常简单——它们不是 3D 的，甚至不是 2D，它们只是文本。

编程挑战的存在是为了给你自己编写一个软件的体验，而不需要任何教程。这是一个锻炼你从本章中学到的一般编程技能的机会。即使它不是 3D 的，你仍然可以感受到创造一些互动的东西——一些你可以输入的东西，并让它输入一个回应给你。
