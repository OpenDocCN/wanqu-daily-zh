# 欢迎光临！

你刚刚在日本加入一家日本公司。你将面对工程、语言和各种文化差异。那种兴奋！多么伟大的冒险等待着我们。

# 工程

**注意:**非工程师可以安全地跳过这一部分。

如果你最终在一个合理的公司，你可能会发现一个和你在家里熟悉的类似的设置。持续集成、不同的开发环境，以及使用一些最流行的企业软件，如 Java 等。然后好戏开始了。

**UTF8 vs Shift JIS 和 EUC-JP**

unicode 之争肯定早就解决了，对吗？哈，哈…哈…

伴随着几乎宗教般地使用微软 Excel 来做几乎任何事情，出现了对日语的两种旧的流行编码之一的使用:在 Windows 上使用 Shift JIS，在 mac 上使用 EUC-JP。这种逻辑实际上并不像听起来那么疯狂:大多数人将他们的 Excel 设置为读取 Shift JIS，当你给他们 UTF8 时，他们会得到 [mojibake](https://en.wikipedia.org/wiki/Mojibake) 。当然，您只需要人们将他们的 Excel 默认为 UTF8，但是他们如何处理所有的遗留文档呢？作为一个外国人，你可能没有这个问题。因此，鸡和蛋永远存在。

**此处未发明**

虽然你会发现许多公司使用谷歌工具套件进行在线协作，但你也会发现 [Desknets](https://www.desknets.com/oncloud/service/img/img_desknets01_l.png) 和 [Garoon](https://www.lancork.net/wordpress/wp-content/uploads/2014/08/cybozu-garoon-detail.png) 是用于日常日历和其他管理操作的本土解决方案。他们经常有一些滑稽可怕的错误，例如 Garoon 不允许你在会议结束后添加一个人，除非你克隆会议，添加这个人，保存它，并删除旧的事件(我祈祷在新版本中这是固定的)。你还会发现像 [ChatWork](https://3.bp.blogspot.com/-z4tJm0PigZs/WT5wjNr6JqI/AAAAAAAABGE/-jWyumK70gM-zO3K-j6UiiWIM09caXNEACLcB/s1600/01%2B%25281%2529.png) 这样的本地聊天应用可能会被用来代替 Slack(尽管日本也有相当数量的 Slack 追随者)。这并不奇怪，很可能是因为价格更便宜(忽略总拥有成本)，更倾向于相信本地品牌，或者是对产品的真正热爱。我认为这更多地说明了美国软件在国外的流行，而不是日本。

**实体书**

当然，西方仍在使用实体书，但它们通常是最受尊重和可重复使用的书籍的保留购买品。日本的书店拥有相对强大的追随者，在工作中，你会经常发现人们购买几乎任何书籍和杂志，有时甚至具体到如何正确使用 messenger。老实说，我认为很容易获得参考指南是一件很棒的事情。我只是觉得这是个可笑的时代错误。你还会看到相当多的人在火车上阅读纸质报纸，但那是另一回事了。

在硅谷，我经常看到人们购买知名的技术书籍，并把它们摆在书架上，以显示自己受过良好教育，这也很有趣。我认为这种情况在日本也会发生，但我更经常看到上面提到的超级特定书籍的情况。

**工程师职业**

你会经常发现许多工程师没有计算机科学学位，这在表面上看起来甚至在硅谷的公司中也并不罕见，但你可能会发现一些人在加入公司之前几乎没有任何编程经验。这很可能是日本大学制度的结果，在这种制度下，你的专业和你未来的工作往往没有什么关联。通常在毕业前一年，你穿着西装在城市里四处求职，试图说服一家几乎不择手段雇佣你的公司。如果你幸运的话，你被录取了，而且在最好的情况下，你在开始你现在找到的任何工作之前得到了培训。有时候，这种情况不会发生。

结果你会发现很多自学成才的人(印象深刻！)，但是自学通常不会产生理想的编程实践:复制+粘贴覆盖重构、快乐路径编码、奇数变量命名、强耦合等。这通常会反映在你将要处理的代码和将要进行的讨论中，所以做好准备。

典型的工程组织将会生存下来，这要感谢一些伟大的工程师，他们通常是自学成才的，但也是超负荷工作的。日本有许多伟大的工程师，但他们中的许多人厌倦了与日本的工作文化打交道，并出国了。因此，留下来的人往往是少数，虽然他们有助于推动事情向前发展，但他们的人数远远少于他们的同龄人。如果你遇到这样的工程师，尽你所能借给他们支持。你可能会帮助他们恢复理智。

# 文化差异

迟早你会遇到这样一种情况，你的经历或观点与日本同事不同，不少人会无一例外地求助于最简单、最灵活的谈话结束者:“啊，这一定是文化差异。”（文化の違い)

我曾经看到过这个用来解释:

*   在部署代码之前没有编写测试或者没有足够的测试
*   冗长无用的仪式会议
*   任何他们找不到非理性行为的合理解释的地方

我必须强调，**不是每个人都这样做**，事实上，毫不奇怪，你的更好的工程师/同事总是会努力保持良好的逻辑讨论。也就是说，你会碰到这个，所以值得一提。

# 真正的文化差异

不管是好是坏，日本的工作社会经常被群体思维、对羞耻的恐惧、对失败的恐惧以及避免冲突和维护和平的渴望所驱动。这将导致一些令人沮丧的情况。

**害怕决策**

决策是走向失败的第一步，没有人愿意失败。但是必须做出决定。这种二分法是如何解决的？会议。没完没了的会议和电子邮件、规划文档、前期规划文档、后期规划文档、会议文档，以及所有人无时无刻不在讨论的所有事情。这种想法是，如果每个人都参与到决策过程中，那么当一些事情不可避免地出错时，就没有人可以责怪了！问题解决了。

当然，这解决不了任何问题，还会产生一系列问题，但这也是在日本工作的另一个好处。

**暧昧**

如果一件有争议的事情停止争议，那么这件事情就解决了。如果不可能满足两个相互矛盾的目标，那就一个都不满足，或者不检查你是否满足了它们。当有人最终提出这个问题时，就回到通常的陈词滥调:

*   对不起—I’m sorry
*   そうですね——是的，你说得对(但我不会为此做任何事情)
*   仕方ないです——什么也做不了(这是不可避免的)
*   さあ——谁知道呢(我不知道，也许没人知道？)

对话示例:

**我:**这分明是坏了！
**田中:**是的，你说得对。M: 难道我们不应该对此做些什么吗？
**T:** 什么都做不了。
**M:** 这个很容易拼起来！
**T:** 对不起。M: 谁负责这个代码？团队 X.
**M:** 我们团队的某个人应该和他们谈谈。是的，你说得对。男:嗯，我可以和他们谈谈吗？
**T:** 啊，也不是不可能。男:那是不是说我可以？
**T:** 谁知道。
**男:** …

假设我去与团队 X 交谈，对话可能如下所示:

我:这里有一组补丁来修复这个存在的问题。
**铃木:**啊，我明白了。M:嗯，你能复习一下吗？我们会调查的。M: 但是这些补丁解决了问题，我已经替你完成了你的工作。
**S:** 啊，是的，谢谢。我们会调查的。M:好的……如果有什么事，请告诉我。
**S:** 啊，是的，谢谢。

正如你可能已经猜到的，什么都没有发生，那些补丁就像我年轻时掉落的樱花花瓣一样被遗忘了。

**众口一词**

如果你和一群日本人在一起，问一个问题，你有时会目睹以下一系列事件:

1.  每个人都环顾其他人
2.  一个人开始暗示一些事情
3.  在整个集团的多米诺骨牌效应中轻微或明显的一致

保护社会和谐的需要在社会中根深蒂固，有时甚至在友好的活动中也会发生，不仅仅是在工作中。这通常适用于社交问题，但任何与工作相关的问题最好是一对一地问。

**同伴压力**

你可能会看到老板或同事为了把更多的工作推给某人而给他们施加压力。这在日本通常被称为“[权力骚扰](https://en.wikipedia.org/wiki/Power_harassment#Japan)”。作为一名外国人，你可能对此免疫，你会发现这会影响你与管理层和同事的关系。他们可能会去找一个更容易使唤的同事，而不是让你去做什么。对我来说，我为其他人被指挥感到难过，所以我试图通过在某种程度上顺从他们的突发奇想来阻止欺负者去找他们，以避免其他人的命运，但有时这些要求是如此荒谬，以至于我不得不说不，我很清楚其他人会得到它。这太可怕了，你可以试着教人们说不，但这样做是在与日本社会最强大的力量之一作斗争。

我会尽力而为

有时你会委托某人一项任务，他们会回答“我会尽力而为”頑張ります).这实际上意味着:即使他们认为一项任务是不可能的，他们也会不顾一切地努力去完成它。一方面这是令人钦佩的，但另一方面，这是不现实的。当有人给你这样的答复时，试着重新协商任务的条款，使其更加合理，这样任务才能真正完成。

这种“尽力而为”的文化，即使它可能会破坏你的健康，也是日本文化的双刃剑之一。至少在短期内，它可能会“更快”地完成一些工作，但它会对身体和心理健康造成灾难性的后果。我亲眼看到人们把自己工作到生病的程度，仍然带着面具去上班。在一个案例中，一个人因为压力的副作用去了医院，然后发现很难回到工作社会。人们花了很多时间来欣赏日本人对工作的热情，但我有时觉得破坏优秀人才所付出的代价太高了。

# 办公室差异

这些是我觉得有趣的事情。也许这在其他地方也很常见，但在硅谷的泡沫中，有一些事情是惊人的。

**打印所有内容**

有会吗？打印幻灯片，每人一套。有设计会议吗？为房间里的每个人打印高质量的模型。然后用笔把它们标记出来，用数字方式进行修改，然后重新打印所有内容并重复。哦，可怜的树…

不仅仅是办公室，几乎你做的每一件事都会附带一张纸。我不完全确定人们对打印输出的痴迷是什么，但它可能与你手中的某种东西的触觉有关，某种有形的东西，在像软件这样的世界里，我们所做的许多事情都是短暂的。我欢迎围绕这个话题的解释和讨论，因为老实说我仍然不知所措。

**沉默是金**

作为一名开发人员，我绝对厌恶开放的办公环境，因为它们经常会导致混乱和分心，我惊喜地看到大多数人在工作时是多么安静。然而，我在日本的外国朋友有时会被这种安静到几乎令人不安的程度所震惊，所以我提出了这个问题。当然，如果你身边有销售或客户服务，这可能会有所不同，但一般来说，人们不会经常大声讨论(除了地球上到处存在的麻木不仁的人)。

# 外国人和外国人

这一点在网上和网下可能已经谈得不可开交了，但我只想提醒你，你是个外国人。你差点忘了吗？[盖金](https://en.wikipedia.org/wiki/Gaijin)(外人)字面意思是“外面的人”，是 gaikokujin(外国人)这个词是指外国人。我个人认为，许多日本人使用“gaijin”这个词是因为它更简短，更容易说，听起来更友好，而不是出于恶意。他们有时会开玩笑地说[——](https://en.wikipedia.org/wiki/Japanese_honorifics#San)(外人さん)以第三人称称呼外国人。

你会经常在工作中听到这个词，而且经常当着你的面。虽然这很容易让人生气，但要考虑到大多数人并不觉得这个词有冒犯性或侮辱性。有些人会特意避免使用它，因为他们知道这会引起冒犯，但绝大多数人不会在意。

外国人(外国)的意思是“外国”,但你会经常听到人们说它是一个真正的国家，涵盖整个世界，但通常只指“西方”或美国。相当于你会听到单词 kaigai(海外)意思是海外，意思差不多。如果你很幸运，他们会用瓯北(欧米)这将只涵盖欧洲和美国(非常具体！).

你有时会听到这样的话:

*   他们在 gaikoku 有 X 吗？(你会惊讶于人们会问什么，我曾经被问及在 gaikoku 是否存在漱口液)
*   凯盖对 X 有什么反应？([国外对日本的反应](https://translate.google.com/translate?sl=ja&tl=en&js=y&prev=_t&hl=en&ie=UTF-8&u=http%3A%2F%2Fkaigainohannou.info%2F2017%2F11%2F17%2F20secapology%2F&edit-text=&act=url)通常比最初的事件得到更多的反应)
*   日本和 gaikoku 不一样(这类似于“文化差异”的论点来解释任何不方便的事情)

有时候你听到的事情听起来有点种族歧视，但一般来说，他们并不是这个意思。我有时称日本有“偶然的种族主义”。这并不是说他们想冒犯别人，他们只是很少考虑某件事是否会冒犯外国人。

# 外国人团队

在一家公司工作的外国人被聚集在一起是很常见的，要么是因为他们出于安全和理智自然地聚集在一起，要么是因为当他们都在一个地方并且与其他人隔离时更容易管理外国人。有时候与日本车队的融合更多，有时候更少，但几乎总是你会看到外国人集中在一个地方。

这其实并不那么令人震惊。在美国工作的外国人也有类似的现象。让我觉得有趣的是，由于外国人通常不像日本人那样敏感和克制，他们被认为更难管理。我认为这是你在日本能做出的最大贡献，同时也是最危险的事情。立刻，你有能力帮助过程现代化和帮助改善事物，也有能力破坏和谐以至于导致灾难。明智地使用你的职位，仔细考虑你的行为的后果，而不仅仅是在你的祖国。

# 外国人的名字

在日语中，姓在前，名在后。在正式的场合，比如在工作中，你可以用“田中- [三](https://en.wikipedia.org/wiki/Japanese_honorifics#San)”来称呼一个叫“田中太郎”的人。有时，当非常熟悉、年龄相近或指比你年轻的人时，你可以用“Taro-san”。

你是外国人(你又忘了吗？).你将主要被称为你的名字，可能是它的缩写。有时，如果你的姓很短或者很简单，就会被使用。我通常被称为“Alex-san ”,因为我的姓有些具有挑战性(话说回来，我的名字在美国也具有挑战性，这就是我叫 Alex 的原因)。这基本上是一个不断提醒外国，即使它不是故意的。事情就是这样，接受现实吧。我向你保证，日本人一开始经常被完全陌生的人直呼其名也会感到不舒服(这是亲密、熟悉或优越感的表现)。

# 技术语言

是的，这是一项工程工作，如果你与日本开发人员合作，你会发现一些有趣的事情。

**英语外来词和日英词汇**

最明显的是公司和产品名称。除此之外，还有一整类词有时被称为ビジネス用語(商业用语)，其中一些非常符合英语，如“双赢”，还有一些你可能从未听说过，但人们会继续惊呼“这来自美国”，好像你知道来自美国的每一件事。它们包括:

*   SNS——社交网络服务(我们用英语称之为社交网络)
*   [PDCA](https://en.wikipedia.org/wiki/PDCA) —计划执行检查行动(我们称之为“执行工作”)
*   [KPT](http://code-artisan.io/retrospective-method-kpt/) —保持问题尝试(这个我编不出来，lol。在网上搜索，大多数结果都是日文)
*   听证会——人们主要听到你谈论事情的会议(不，你不是在受审，别担心)

我可以连续几天列出这些，但那是另一天的话题。关键是，要习惯学习很多新的“英语”单词。

来自欧洲语言的英语单词

当西方世界融合成一个大家庭时，人们不可避免地会认为一切都来自英语，当你不理解这些单词时，会感到困惑。一个例子是单词“enquette”(更正:enquette，谢谢你 [Renaud Guerin](https://medium.com/u/f6bb044c8ed0?source=post_page-----aaa9be8919b2--------------------------------) ！)是“调查”的法语单词，经常使用。

**大量词汇** [**汉字**](https://en.wikipedia.org/wiki/Kanji)

其中许多是从数学中传承下来的，当然数学在日本已经存在很长时间了，所以大多数单词来自中文是有道理的。以下是一些常用词的简短列表。

*   関数—功能
*   再帰的関数—递归函数
*   変数—变量
*   引数—参数
*   There is no coincidence-fault/bug
*   再起動—重启/重新启动

然而有些单词使用英语

*   メソッド —方法(“mesoddo”)
*   バグ — bug (“bagu”)
*   デバッグ —调试(“debaggu”)

**等词用** [**【英语】**](https://en.wikipedia.org/wiki/Wasei-eigo)

*   パソコン——个人电脑(“pasokon”，即“paasonaru konpyuuta”的缩写，即个人电脑)
*   ロールアウト —推出(部署新版本的软件；这个并不常见，但是在 web 服务器软件环境中并不常用，至少在我来自的加利福尼亚是这样)
