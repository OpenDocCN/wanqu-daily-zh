# 为什么你应该首先不用颜色来设计的 4 个理由

当涉及到 UX 设计时，在添加色彩之前创建灰度屏幕会迫使你清晰地思考并正确地优先考虑。

# 1.你关注布局和间距

当你开始设计的时候，要记住 B&W 调色板的限制，你大部分的思考时间都花在了如何正确的安排空间上，以便看起来像是组合在一起。你开始考虑一些看不见但更重要的方面，比如可读性(行高、段落和版式)、注意力中心(行动按钮和大小)和可浏览性(元素分组、间距)

# 2.你的客户问了正确的问题

好的设计需要时间。在设计的早期阶段，大多数设计师羞于向客户/用户展示未完成的作品，以收集早期反馈。我再怎么强调从客户那里收集早期反馈的重要性也不为过。这有助于您远离过多迭代的陷阱，避免在不必要的版本上浪费太多时间。大多数客户要么不知道他们到底想要什么，要么不擅长给出建设性的反馈。给他们看一个早期版本，看起来像一个更高清晰度的纸笔草图，会让他们理解这个过程，并问你一些更有建设性的问题。

> 你可能会问“这个按钮适合什么颜色”，这肯定会比“你为什么选择黄色这个按钮”更好。

# 3.你的设计很干净

创造一个简单干净的设计不是一件容易的事情。当你设计黑白图案时，你关注的是可用性、间距、尺寸、布局元素、交互和流程。你的设计的第一个版本应该只有不同的黑色和白色阴影，以显示元素之间的层次和视觉重量。



# 4.创造一致性

在你的设计中最好使用三种或更少的颜色。在你的设计中使用太多的颜色可能会让用户困惑，或者把用户的注意力转移到不太重要的地方。当你有一个单一的品牌颜色，并有选择地用在黑白设计上时，用户的眼睛自然会被吸引到这些“颜色”区域。
