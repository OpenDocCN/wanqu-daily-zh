# 在数年辱骂性的电子邮件之后，Linux 的创造者靠边站了

> 原文:[https://www . new Yorker . com/science/elements/after-years-of-abuse-emails-the-creator-of-Linux-steps-side？UTM _ source = Wanqu . co&UTM _ campaign = Wanqu+Daily&UTM _ medium = website](https://www.newyorker.com/science/elements/after-years-of-abusive-e-mails-the-creator-of-linux-steps-aside?utm_source=wanqu.co&utm_campaign=Wanqu+Daily&utm_medium=website)

著名程序员 Linus Torvalds 的电子邮件就像雷电一样从高处降落到公共列表中，充满了谩骂、侮辱和贬低的语言。“请你现在就自杀吧。世界将会变得更好，”他在一篇文章中写道。“伙计们，这不是一场吹箫比赛，”他在另一场中评论[。“他妈的闭嘴！”他在第三部](https://lkml.org/lkml/2013/2/21/228)中开始了[。](https://lkml.org/lkml/2012/12/23/75)

托瓦尔兹已经公开发布了数千条严厉的信息，针对那些向 Linux 计算机操作系统内核提交他认为有缺陷的代码的程序员，他在超过 25 年前[赋予了 Linux 计算机操作系统内核生命](https://www.zdnet.com/article/linus-on-linuxs-25th-birthday/),现在作为一个合作的开源项目来管理。今天，Linux 内核很有名，运行着谷歌、贝宝、亚马逊和易贝的大型计算机，以及使用 Android 操作系统的 20 亿部手机。不过，托沃兹保留了对每一行宝贵代码的最终决定权，就像他在赫尔辛基大学读研究生时第一次开始研究这个系统一样。多年来，他被称为 Linux 的“终身仁慈的独裁者”

周日，这位仁慈的独裁者[宣布](https://lkml.org/lkml/2018/9/16/167)他将暂时下台，以“获得一些关于如何理解人们的情绪并做出适当反应的帮助。”托瓦尔兹今年 48 岁，和家人住在俄勒冈州波特兰市外，他明确表示自己并没有筋疲力尽。“我非常*非常*想继续做这个我已经做了近三十年的项目，”他在 Linux 内核邮件列表的一篇帖子中写道。“我需要休息一段时间，以获得关于如何改变行为方式以及修复我的工具和工作流程中的一些问题的帮助。”托瓦尔兹任命了一位副手格雷戈里·克罗亚·哈特曼在他不在的时候负责这个项目。

托瓦尔兹决定辞职之前,《纽约客》( The New Yorker)就他的行为向他提出了一系列问题，因为一篇报道抱怨他的虐待行为阻碍了女性从事 Linux 内核程序员的工作。在给《纽约客》的回复中，托沃兹说:“我为我发明的 Linux 代码及其对世界的影响感到骄傲。然而，我并不总是为自己无法与他人良好沟通而自豪——这对我来说是一生的奋斗。对于任何一个我伤害了他们感情的人，我深感抱歉。”

托瓦尔兹的回应由 Linux 基金会传达，该基金会支持 Linux 和其他开源编程项目，截至 2016 年，每年向托瓦尔兹支付 160 万美元[。该基金会表示，它支持他的决定，并鼓励女性参与，但它对托瓦尔兹如何运行编码过程几乎没有控制权。“我们能够在新项目中对这些结果产生不同程度的影响，”声明说。"像 Linux 内核这样更老更成熟的成果更难影响."](https://projects.propublica.org/nonprofits/organizations/460503801/201723189349305852/IRS990)

直到这个周末，托瓦尔兹不仅为自己的过激行为辩护，还坚称这是 Linux 取得巨大成功的原因。“如果你想让我‘表现得专业一点’，我可以告诉你，我不感兴趣，”他在 2013 年写道，作为对知名 Linux 贡献者塞奇·夏普(Sage Sharp)的回应，塞奇·夏普在一份公开的电子邮件列表上要求托瓦尔兹停止在电子邮件中使用“身体恐吓、口头威胁或口头辱骂”。“我穿着浴袍坐在家里的办公室里，”托瓦尔兹写道。“同样地，我不会开始打领带，我*也不会相信虚伪的礼貌、谎言、办公室政治和暗箭伤人、消极的攻击性和时髦的词语。因为这就是“专业表演”的结果:人们诉诸各种真正令人厌恶的事情，因为他们被迫以不自然的方式表现出正常的冲动。”*

尽管 Linux 项目免费分发其产品，但它已经成长为一家蓝筹科技公司。名义上是一个志愿者企业，就像维基百科一样，事实上，Linux 主要是由来自世界大型技术公司的资金和程序员维持的。英特尔、谷歌、IBM、三星和其他公司指派程序员来帮助改进代码。根据[基金会 2017 年](https://www.linuxfoundation.org/2017-linux-kernel-report-landing-page/)的报告，在过去一年对 Linux 进行的 8 万次修复和改进中，超过 90%是由付费程序员完成的；仅英特尔员工就要对其中的 13%负责。同样是这些公司，还有数百家其他公司，覆盖了基金会大约五千万美元的年度预算。

Linux 的精英开发人员，绝大多数是男性，倾向于分享他们领导的积极自信。在最多产的贡献者中很少有女性，尽管基金会和研究人员估计大约 10%的 Linux 程序员是女性。“技术界的每个人都知道这件事，但莱纳斯得到了通行证，”埃隆大学(Elon University)的计算机科学教授梅根·斯夸尔(Megan Squire)告诉我，她指的是托沃兹的虐待行为。"他建立了这种个人崇拜，这种对重要性的崇拜."

在一个研究项目中，斯奎尔使用来自托瓦尔兹的电子邮件来训练计算机识别侮辱。根据斯奎尔的列表，托瓦尔兹在四年时间里发出的 21000 封电子邮件中，有 1000 多封使用了“废话”这个词。在那个时期，“荡妇”、“婊子”和“杂种”的使用频率要低得多。斯奎尔告诉我，她发现很少有性别偏见的例子。“他是一个机会均等的滥用者，”她说。不过，斯奎尔补充道，对于非男性程序员来说，敌意和公开羞辱更加孤立。随着时间的推移，许多女性程序员离开了社区。“女人先认输，”她告诉我。“他们说，‘为什么我需要忍受这些？’"

2013 年，夏普，这位非二进制且使用“他们/他们”代词的人，在他的地盘——公开的 Linux 内核邮件列表——与托瓦尔兹对峙。夏普将托瓦尔兹描述为“在口头上辱骂人们并公开撕裂他们的情感时，最糟糕的罪犯之一。”当时，在俄勒冈州一个小镇长大的夏普年近三十，是 Linux 内核的重要管理员。“人们以为我是科技行业的 cis 女性，因此我受到了很多骚扰，”夏普告诉我。夏普试图以实际的理由吸引托瓦尔兹。“我不是要求你为了帮助少数民族而改变你的沟通方式。我不是在 Google+上咆哮虱子的疯狂女权主义者，”夏普写道。“我正努力为所有开发人员改进内核邮件列表。我们可以给出负面的技术反馈，而不用言语谩骂。”

托瓦尔兹回答说，赌注太高，顾不上礼貌。错误的代码会产生级联效应，需要其他开发人员做大量的工作来纠正。速度和准确性是最重要的。“诅咒发生在‘你错得太离谱了，甚至不值得尝试对它进行逻辑论证，因为你没有可能的借口’的情况下。 . .有时人们会给我一个惊喜，然后带着一个合理的借口回来。我全家都死于一场悲惨的意外事故，我的小马得了癌症，我心烦意乱。然后我可能会告诉他们我很抱歉。不，不是真的。"

Valerie Aurora，一位前 Linux 内核贡献者，告诉我她在 Linux 社区工作了十年，她确信她不能作为一名女性在这个等级中上升。奥罗拉说，托瓦尔兹和其他强大的技术人物是“平等机会的混蛋”的概念是错误的，是性别歧视的:当她和夏普采用托瓦尔兹咄咄逼人的沟通风格时，他们经历了报复。“基本上，莱纳斯创造了一个领导模式——那就是做一个混蛋，”奥罗拉告诉我。“塞奇和我可以告诉你，做一个混蛋对我们来说是不可能的。如果我们是个混蛋，我们会被打耳光，被惩罚，被扣留。我试过了。”

Aurora 六岁时第一次接触编程，当时她和父母住在新墨西哥州。她母亲是对计算机感兴趣的人；她买了一台 IBM PCjr，教女儿*基础*编程语言。十二岁时，奥罗拉一家搬到了一个牧场。“我会写一个计算机程序，然后去挤羊奶，”奥罗拉回忆道。她第一次接触 Linux 是在新墨西哥矿业技术学院。大约在 2000 年，当她的第一个 Linux 补丁被接受时，她举办了一个聚会。几年后，她在[出版了一本在线手册](http://tldp.org/HOWTO/Encourage-Women-Linux-HOWTO/)，“如何鼓励 Linux 中的女性”，其中包括“不要过多批评”、“一定要赞美”和“当女性到来时不要盯着她指指点点”等章节 Aurora 告诉我，她在一次 Linux 会议上被人摸了，一名高级管理员轻视她，说她缺乏成为关键开发人员的才能。她说，她从来没有想过要举报性骚扰。2007 年，Aurora 开始退出 Linux 编程。她现在是科技公司的顾问，致力于包容性和多样性问题。

许多为 Linux 做出贡献的女性指出另一个开源项目 Python 是 Linux 面临#MeToo 时刻的指南。来自荷兰的白人男性程序员吉多·范·罗苏姆发明了 Python 编程语言的代码。然而，Van Rossum 自称是一名女权主义者，他在 Python 程序员的主题演讲中经常穿着一件“Python 是女孩的”t 恤。“一个项目吸引了适应这种文化的人，”范·罗森告诉我，并补充说，如果领导们进行了激烈的沟通，“它将吸引那些要么赞同这种态度，要么至少不认为这有什么问题的人。”Van Rossum 现在住在湾区，她说 Python 社区表明从事开源软件项目的女性人数可以增加。“我正在把我的女权主义思想或倾向应用到一个我自然会产生一些影响的地方，”他说。

Mariatta Wijaya 是一名软件工程师，出生在印度尼西亚，现在在温哥华生活和工作，她说，出于好奇，她于 2015 年在蒙特利尔参加了她的第一次 Python 大会。在 van Rossum 的主题演讲中，他承认核心开发人员中没有女性。“他说他愿意亲自指导女性，如果这是提高多样性所需要的，”维贾亚回忆说，“但我没有联系他。”第二年，她参加了同样的会议。“他做了另一个主题演讲，承认仍然没有女性，”她回忆道，并提出了同样的指导建议。这一次，Wijaya 联系了 van Rossum，他同意提供帮助。一年后，Wijaya 成为第一个拥有核心开发人员特权的女性，这让她有能力帮助决定 Python 将如何发展和变化。

今天，大约 90 名 Python 核心开发人员中有 4 名女性。今年夏天，范·罗森宣布他将辞去 Python“仁慈的独裁者”一职。“我不会任命继任者，”他在一封公开电子邮件中写道。“那你们都打算怎么办？创造民主？无政府状态？独裁统治？一个联邦？”

相比之下，托沃兹长期抵制 Linux 编程团队需要变得更加多样化的想法，就像他抵制降低自己语言的呼声一样。2015 年，夏普倡导了有史以来第一个针对 Linux 开发者的行为准则。至少，他们希望有一部法律来禁止 DOX xing——在网上发布个人信息以煽动骚扰——以及社区中的暴力威胁。相反，托瓦尔兹接受了一个具有挑衅性的名为“冲突代码”的编程修复，这创建了一个更普遍的投诉机制。从那以后的三年里，没有一个开发者因为辱骂性的评论而受到惩罚。当时受雇于英特尔的夏普说，他们此后小心翼翼地避开 Linux 内核工作。

Torvalds 周日的帖子是道歉、内核更新和已获批准的代码更改的大杂烩。他开玩笑说，也许他可以为自己的不良行为找到一个技术解决方案——“也许我可以安装一个电子邮件过滤器，这样当我发送带有诅咒语的电子邮件时，它们就不会被发送出去。”—在承认之前，在他照镜子之后，“很明显，这不是唯一必须发生的变化。”被批准的补丁列表中有一个名为“[行为准则:让我们修改它](https://git.kernel.org/pub/scm/linux/kernel/git/torvalds/linux.git/commit/?id=8a104f8b5867c682d994ffa7a74093c54469c11f)”它宣布“冲突准则”已被“行为准则”所取代，该准则禁止“侮辱性/贬损性评论”和“在专业环境中被认为不合适的”行为。投诉将由基金会的技术顾问委员会审理，该委员会有 10 名成员，均为男性。
