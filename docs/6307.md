# 头版:好的，坏的，丑陋的

发布于 2020 年 11 月 17 日

现代 web 开发请不要用 FrontPage！它充满了安全漏洞和过时的标准。这样做的目的不是要说服你。最新的版本差不多二十年前就出来了！

Microsoft FrontPage。仅仅提到这个名字就让大多数(如果不是所有的话)经验丰富的 web 开发人员抱怨不已。“FrontPage 完全是来自地球城市黑暗时代的垃圾”你说。"它接触到的一切都被可怕的输出和专有的胡说八道毁了！"是的，它是。

但是...作为概念的 FrontPage。作为一个关于可能发生的事情的梦，以及一扇了解*是什么*的窗户。让当时典型的家庭用户创建网站，表达创造力，并通过风暴征服世界，所有这些都不需要被迫学习 HTML 或 CSS 或 JavaScript...在这方面，FrontPage 是无可匹敌的。

让我们来谈谈为什么微软的 FrontPage 在短时间内是互联网的终极内容创建工具，以及为什么它后来失宠了。

## 历史

在谈论善与丑之前，我们需要先谈谈 carphones 和商务会议。因为这是 1994 年，那时互联网还很新。在那个时候，大多数互联网聊天发生在新闻组集团(想想类似 Reddit 的东西)或 BBS 系统(想想类似[老 AOL](../5/)；下图示例)。如果你需要生意上的东西，比如股票之类的，你必须登录任何有你需要的东西的人的 BBS。这是一种痛苦。

![A picture of a BBS (tilde.town) as of today](../Images/8f8e2b8552da7996ac8fb8b53c04a886.png)

兰迪·福加德和查尔斯·h·弗格森出场了。著名的计算机行业顾问 Ferguson 联系了麻省理工学院的毕业生 Forgaard(如果你想知道的话，通过 carphone ),讨论创办一家新的基于互联网的公司。

他的想法是，许多公司，如道琼斯、彭博、苹果等。投入了数百万去建立他们自己的，完全不兼容的拨号上网服务。因此，他们应该创建一个标准化的、完全开放的服务器/客户端组合来取代所有的独立工作。这有望降低这些公司的开发成本，并通过让企业希望在互联网上出现来提供一个增长的市场。

两人决定成立他们自己的公司，维梅尔技术公司。一个月后，还在计划阶段，他们从欧洲粒子物理研究所听到了全新的万维网的风声。它是分散的、开放的，甚至比他们计划的还要强大。简直太完美了。唯一的问题是，如果你只是一个低级的广告经理或其他什么人，制作网站是一件相当痛苦的事情。网络需要一个网站创作工具。

![FrontPage 1.0a, taken from WinWorldPC.com](../Images/13ab6ac0c30d929f4d90aadb74fe6dd8.png)

所以，不知何故，他们设法雇佣了许多专业的程序员，没有任何薪水，在 FrontPage 上工作。(早期创业文化，或许？)显然它起作用了，因为 FrontPage 发布了(只有！)1995 年 10 月，比计划晚了一周。那时，万维网正在爆炸式增长，网站数量每月增长 20%。FrontPage 也获得了巨大成功，获得了许多奖项和好评。事实上，它是如此之好，以至于微软最终买下了它们。据他们说，它不仅感觉像一个原生的 Office 应用程序，而且对他们正在进行的变得更加以互联网为中心的计划来说是完美的。

![FrontPage 97, taken from WinWorldPC.com](../Images/f720e599eede1ffdb050f16dabb7b6a6.png)

多年来，FrontPage 被集成到 Office 套件中，并成为微软生产力产品的旗舰产品。到 Office 2000 时，FrontPage 的 Office 集成已经超出了您的想象；确定 Office 用户界面，并允许在整个套件中导入和导出。那有...其他的也是，但是现在让我们保持乐观。

我将介绍 FrontPage 2002，因为这是我拥有的版本，也是我最有经验的版本。

## 好人

界面...是(在我看来)有史以来最好的程序界面之一。不夸张。我认为只有那个时代的其他微软应用程序如 PowerPoint 和 Visio 能超过它。我可以*看到*它为什么获奖。

![FrontPage's start screen with no webs open](../Images/e8dd394cfc4e38e4cbdfd41c73a967e8.png)

像任何其他 Microsoft Office XP 程序一样，右边有一个侧边栏，上面有一些常见的任务。当然，你可能想做的是创建一个新网站。

FrontPage 相当于“项目”,叫做“网站”。这些网站包含文件(你所有的 HTML，CSS，图片等。)和该网站的偏好设置(如与哪个 web 服务器同步或使用什么兼容性设置)。

![FrontPage's template selector](../Images/a6c1e535d3105e9fef5effd62da61082.png)

从选择一个新的“空网站”开始。弹出一堆模板，包括“空网站”模板。有点奇怪，但是肯定的。

![The "Personal Homepage" template in action](../Images/ec5eddec6e520b3b3d1199b14f18da72.png)

如果你*被*选择一个模板，它会给你一个预填充的填空网站，准备好你的文字和图片。它需要一些布局工作和设计创意，但是*毕竟是*模板的目的。(尽管，作为一句忠告，*不要使用这些。从网络标准的角度来看，它们的设计相当糟糕。你可以看到它勉强适合我的窗户。想象一下在智能手机上。)*

![FrontPage open to a new blank web. Folder list and Views bar are visible.](../Images/e9f869b6d5c6e28dcb8eac61006ae994.png)

不管怎样，一旦你打开你的空白网页，你会得到...没什么！(那是你要求的，不是吗？)不用担心。真的，如果你仔细想想，立即开始制作一个页面是相当愚蠢的。你看，FrontPage 采用了一种面向项目的方法。虽然你很容易*可以*坐下来开始敲打东西了，但这真的不是 FrontPage 想要你走的路。

![FrontPage Navigation editor with some pages created and linked up.](../Images/2edc83532b468138a5b8318f2cce904a.png)

这可能是我最喜欢的 FrontPage 功能:导航编辑器。在这里，您可以创建空白页，并按照逻辑层次将它们链接在一起。这与你设计网站时通常在纸上进行的过程非常相似。

![Navigation bar settings dialog](../Images/22c5345e951c71b50a8cb48755e390d5.png)

通过这种层次结构，您可以在所有正确链接的页面中自动创建导航栏。例如，如果您决定创建一个新的顶级页面，那么您网站上的每个页面都将被更新以突出该页面。(为了在这个站点上复制这个特性，我不得不在我的定制 Flask 服务器上使用一些非常复杂的模板脚本。我非常喜欢这样的东西。)

这个功能也很好，因为它迫使你*考虑你的站点*。你不能像玩微软 Word 一样坐下来，然后敲出几页。你需要一个连贯的计划。在你开始之前，你需要和你的客户/团队/自己坐下来，问“你想在你的网站上做什么？”对大多数人来说，很明显你需要一个网站。每个人都有一个网站。不太明显的是*网站上需要什么。FrontPage 的 web 编辑器允许您在提交任何内容之前，先尝试一下层次结构和布局。*

![PowerPoint Slide Sorter mode"](../Images/bfedeab1525281de9bfd83af008d9902.png)

有趣的是，Word、PowerPoint 和 Access 都(模糊地)遵循相似的模型。Word 有大纲编辑器，PowerPoint 有幻灯片浏览模式(如上图，展示了我 11 岁时做的一个互动游戏)，Access 是一个典型的数据库程序，它要求你在处理表格之前先声明表格。

更有趣的是，除了 Access(没有多少人使用，主要是因为它的成本和相对罕见)，你从来没有被要求在开始之前有一个计划。我见过的大多数使用 Word 的人只是直接打字，或者用一个提纲完成另一个文档。但是选择就在那里！人们可以在他们的文档中完成他们的大纲*，并围绕其进行充实。PowerPoint 也是如此。*

![Highlighted template include code](../Images/6b6ac0a6f4c516e995bab7dfdc5c0a14.png)

网页设计中的另一个常见任务是定义一个“模板”页面，你可以在其中填写所有内容。例如，你现在正在阅读的这个网站上的每一页都有相同的页眉和页脚，每篇文章的顶部都有相同类型的缩略图。这些都是使用模板特性编写的。

FrontPage 作为一个办公产品，支持模板。实现被公认为是超级的，但是它确实存在。您可以将页面包含在其他页面中，这很好，只是设置起来有点棘手。(仅供参考，Word、Excel 和 PowerPoint 也有模板。这是他们鲜为人知的特色之一，IMHO。)

![Report view window](../Images/bb2d31ef4dc964964b16883c0825b276.png)

另一个非常好的特性:报告。就像 IDE 中的编译器一样，FrontPage 为您的站点生成警告、错误和统计信息。一目了然，我可以查看所有断开的链接，孤立的网页，过大的图像等。这是一个*奇妙的*特性，是现代网络开发软件*所不具备的*。

![Hyperlinks view window](../Images/9686737682f97107056ac89c2e54f922.png)

与报告一样，您可以一目了然地看到什么链接到什么。这很像软件开发 IDE 中的调用图。我可以很快看到从任何页面链接的每个网站(这里显示的是[Sonic R color fix 文章](../24/))和任何子页面。

![Tasks view, showing a task labeled "Write FrontPage article"](../Images/84f10171e2a8d026a283d6afa93a2004.png)

最后一个特征，尽管在 2020 年这一点没那么有用。在 tasks 视图中，您可以列出需要完成的任务。这似乎已经很好地记录了你需要做的事情。在某种意义上，它很像 GitHub 或 GitLab 的问题跟踪器。

但是，如果您连接到一个带有 FrontPage Web Extensions 的服务器(我会谈到这一点...)，您可以与其他人共享此任务列表。你可以(再一次，像一个问题跟踪者一样)把一个任务交给别人去做，然后去做一个别人交给你的任务。对于 web 开发来说，这是一个非常好的特性，但是现在这通常由像 GitHub 或 GitLab 这样的源代码控制提供商来处理。

![FrontPage editing the Sonic R color mod article](../Images/50ae6b77694a43bd10b7144f5684491f.png)

当然，还有网页编辑器。这几乎和微软的 Word 一模一样，只是增加了一些网络的味道。你输入单词，它们就会出现在页面上。你把这些词变成蓝色，它们就是蓝色的。*也就是说*，当你编辑这样的元素时，它会插入一个 HTML 3.2 风格的`<FONT>`标签，而不是试图将它匹配到一个 CSS 规则。有很多方法可以解决这个问题，但是你必须非常勤奋地去做。不过，这主要是因为 FrontPage 是它那个时代的产物。

FrontPage 是一个关于计划、执行和审阅的程序。以导航编辑器为例。你计划好你网站的内容。你画出布局草图。然后你反思你的布局是否符合你的要求。或者报告。创建内容，检查报告。或者任务。计划好你需要做什么，去做，然后检查任务是否完成。这就是 FrontPage 脱颖而出的原因。

### 题外话:使用 FrontPage 进行符合标准的 Web 开发

另外，声明一下，用 FrontPage 创建干净的、有点现代的代码是可能的。(我指的是“使用 CSS”和“不需要 IE”，而不是任何涉及 HTML5 或 CSS3 的东西。)你需要做两件事。

您需要做的第一件事是将 FrontPage 的功能设置为更适合现代的功能。为此，请打开任何页面，然后转到工具→页面选项→兼容性。然后使表单看起来像这样:

![Settings to fix FrontPage's IE reliance on Internet Explorer](../Images/fc5f872f22785d94db08f7b8c0384f50.png)

这样做可以删除特定于 Internet Explorer 的代码、过时的 web 技术以及对 FrontPage 服务器扩展的依赖(现在已经没有人运行它了)。请记住，这确实会禁用 FrontPage 的许多功能，但坦率地说，这可能会更好。

你需要做的第二件事有点复杂。很大程度上，您需要创建一个样式表，而不是使用工具栏中的格式化选项。这样做可以实现更多的跨设备兼容性，并在将来更容易进行站点范围的更改。

要创建样式表，请转到文件→新建。打开页面模板并转到样式表选项卡。选择“普通样式表”然后按确定。

![Stylesheet template dialog](../Images/1c8e558871cb28e4a38f52039ae0aa97.png)

现在，您应该有一个禁用了所有格式选项的空白文档。转到格式→样式，在“列表:”下选择“用户定义的样式”

![List of page styles](../Images/1248915c18834347712ba244d22ae485.png)

从这里，按“新建”。当要求输入名称时，键入您希望应用样式的元素名称或类的名称。(这是一个标准的 CSS 选择器，所以所有这些规则都适用。)从那里，使用左下角的菜单来修改样式。重复，直到所有的风格。

要在页面中使用样式表，请转到该页面，然后转到格式→样式表链接。点击“添加”，选择样式表，按“确定”。你的页面应该有风格！(然而，内置的“预览”模式由于某种原因不会显示它们。)

要给一个元素一个类，选择一段文本并在工具栏中寻找第一个下拉菜单。(应该是“段落”、“页眉 1”之类的。)选择您希望选择的类别。您的选择现在应该具有该类的样式。

我绝对承认这是不必要的混乱，特别是考虑到 CSS 早在 1996 年的 Internet Explorer 3 就得到微软的支持，但此时`<COLOR>`和`<FONT>`标签还没有被弃用。

## 坏事

还记得我说过微软收购维米尔是因为它“非常适合他们正在进行的以互联网为中心的计划”吗？让我们多谈谈历史。

![List of WebBots (now called Web Components) in Frontpage 2002](../Images/fd59ed34d98c1db4f9b389cf74dedbfc.png)

当 Vermeer FrontPage 发布时，它支持这些名为 WebBots 的“插件”。网络机器人背后的想法很简单:你把它们拖到页面上，你马上就能得到动态内容，比如搜索页面、注册、目录等。这些“网络机器人”作为评论保存在 FrontPage 制作的 HTML 中。在“标准”网络服务器上，这些机器人什么也不做。然而，如果你在弗米尔(后来是微软)的个人网络服务器上运行它，网络机器人就会活过来。为了保持 WWW 的开放性，Vermeer 为当时所有流行的 web 服务器，如 NCSA、CERN 和 Apache，发布了 FrontPage 服务器扩展。

在我进一步讲之前，我们先来谈谈 ie 浏览器和微软。简而言之，微软决心赢得互联网。大约在 1996 年，Internet Explorer 3 问世了。它做得还不错，拥有大约 20%的市场份额，但是它仍然输给了网景导航公司。因此，为了提高采用率，微软在 ie 4 中加入了 Windows 桌面更新。

![Windows Desktop Update before/after](../Images/7d09e0288b3578c55bb5311f620684c9.png)

这次更新所做的或多或少是用 Internet Explorer 的内核替换了一大块 Windows 界面和 API。尽管它被宣传为“你想要的互联网！”以及“Windows 资源管理器的全新功能！”，它也是推动 IE 应用的前沿阵地。

即使您禁用了桌面更新，一些程序仍然要求系统上有 IE4。为什么？IE4 所做的事情之一是提供了一个方便的 API 来为任何 Windows 应用程序添加嵌入式网页。它还为您的程序提供了诸如 HTML 帮助和更多常见控件等功能。这背后的逻辑是提供一个统一的 API 和一致的特性集，用于向任何 Windows 应用程序添加互联网特性。

不幸的是，他们这样做的方式让使用非 IE 浏览器(主要是 Netscape Navigator)感觉很陌生。为什么要使用奇怪的外部程序，当你可以使用一个看起来和行为*就像* Windows 资源管理器和 Windows 帮助和操作系统的其余部分。另外，许多新电脑用户甚至没有听说过网景公司，他们也没有理由去调查。(奇怪的是，[在可用性测试](http://www.gargaro.com/test.html)中，人们发现网景比 IE 更好*！)*

 *此外，为了加速网景的死亡，Internet Explorer 增加了一些独特的功能。其中最著名的是 IE3 中添加的 ActiveX 控件。ActiveX 控件是放在 web 服务器上的程序，可以通过 web 浏览器下载丰富的动态内容。问题是，ActiveX 控件只能在 Internet Explorer 上工作，而且通常是为 Windows 9x 编程的。这将互联网的很大一部分拒之于微软视窗操作系统的 ie 浏览器之外。还有 VML(矢量图形)，各种 CSS 属性，VBScript(讨厌 JavaScript？Visual Basic 怎么样？)，等等。所有这些最终导致了 2001 年对微软的反垄断诉讼

我的观点是什么？Microsoft FrontPage 广泛使用了这些微软技术。如果你愿意，你可以让所有的脚本都在 VBScript 上运行，因此只能在 Internet Explorer 上运行。您可以通过自选图形绘制矢量图形，但它只能在 Internet Explorer 中矢量化。(尽管，公平地说，SVG[唯一的另一种选择]直到 2001 年才最终确定。)它可以嵌入由 Microsoft Office 生成的网页，这些网页只能在 Internet Explorer 中正确显示。它可能包括只能在 Windows Media Player 上使用的视频。

当然，你必须不怕麻烦去做所有这些事情。然而，问题是，对于这些功能中的许多功能，通常没有 Netscape 的替代品，如果你想要兼容性，这就使程序的功能变得残缺不全。但是等等，还有呢！

大多数网络机器人都需要 FrontPage 扩展。虽然 UNIX 服务器版本仍然存在，并且仍在更新，但它们并不是很好。[首先，阿帕奇版本有巨大的安全漏洞](http://insecure.org/sploits/frontpage.server_side_apache_extensions.html)。显然，微软正尽最大努力掩盖这些问题，正如在[front page Server Extensions 2002 安装文档](https://msdn.microsoft.com/en-us/library/cc767983.aspx)中，它几乎不承认 UNIX。(事实上，今天要获得服务器扩展，你必须去[一些奇怪的第三方网站](http://www.rtr.com/fpse/Win2008R2/index.htm)(由[一个微软官方网站认可！](https://www.iis.net/learn/publish/frontpage-server-extensions/installing-the-frontpage-server-extensions-on-iis))并购买。为了钱。为什么。)

另外，除此之外，还有一些功能需要微软的 IIS 网络服务器。其中最大的一个是数据库集成。与数据库相连的网页是用 ASP 提供的，ASP 是微软的一种专有技术。而且，如果您正在处理任何类型的动态内容的用户交互，数据库不是可选的。此外，除此之外，它支持的唯一非微软数据库是甲骨文的。另外，在*和*之上，FrontPage 中有一个单击选项，可以创建到任何 Microsoft Excel 工作簿或 Access 数据库的数据库连接。另外，除了*和*之外，一些协作功能只有在安装了微软 SharePoint 服务器的情况下才能工作。

FrontPage 虽然是一个很棒的产品，但它只不过是微软工具箱中的一枚棋子。所有这些都奏效了。非常喜欢。2000 年，百思买有了一个首页网站，还有很多很多其他网站。见鬼，他们甚至设法让 [GeoCities](https://web.archive.org/web/20001018043143/http://officeupdate.microsoft.com/services/yahoo/yahoocohome.htm) 、 [AngelFire](https://web.archive.org/web/20001018043143/http://officeupdate.microsoft.com/services/Angelfire/Angelfirecohome.htm) 、Tripod 和 Talk City 来运行 Frontpage 服务器扩展。FrontPage 在互联网上风靡一时。到 2002 年，Internet Explorer 达到了 96%的市场份额(相比之下，1997 年的市场份额约为 18%)。

你知道最棒的是什么吗？因为微软的技术在公司创建互联网网站和公司内部网的时候是如此突出(尤其是在 Y2K 惨败前后)，所以他们被卡住了。被困了很久很久。多年来，由于基于微软服务器技术，内部公司站点需要某种形式的 Internet Explorer 是很常见的。[日本政府*出于某种原因在 2016 年*](https://redd.it/4bc4ri)改用了基于 ActiveX 的技术。就消费者而言，我们仍然停留在试图从 Windows 程序、Office 文档和 Visual Studio 程序开发中过渡出来。当每个人都拥有自己的第一台个人电脑时，它们在 T4 非常流行。

你可能不会在 Internet Explorer 上看到这个的唯一原因是因为一个软件:Mozilla Firefox。这是自 Netscape 以来第一个流行的网络浏览器，提供 IE 没有的东西，如标签浏览和下载管理器。(也有歌剧，但你得付钱。很明显，人们真的想要 Firefox。

Firefox 的兴起使得微软服务器在公共互联网上的使用根本不值得花费几百美元的许可费用，这导致了基于 Linux 的免费服务器软件的广泛使用。完成了推动越来越无用的 IIS、Windows NT Server 和 MS SQL Server trio 的目的后，它的 HTML 输出变得越来越过时(这是推测；微软尚未给出官方原因)， [FrontPage 于 2006 年停产](https://web.archive.org/web/20061116063501/http://www.microsoft.com/office/frontpage/prodinfo/futureoffp.mspx)。

## 丑陋的

尽管 FrontPage 在过去的 13 年里被停止使用，以今天的标准来看，它制作的 HTML 很糟糕，而且完全是微软统治计算机行业的一个幌子，但对于没有经验的用户来说，它仍然是制作网页的最佳程序之一。是的，说真的。FrontPage 绝对不是你在 2020 年用来制作网站的程序*。但是，奇怪的是，没有一个程序像 FrontPage 那样拥有如此强大的用户界面。所有其他的产品要么需要你对 HTML 略知一二，要么就是已经很多年没有更新了。*

### Microsoft Expression Web

微软给出的替代 FrontPage 的推荐方案是它自己的 Expression Web。我试用了一下(它是免费的，主要是因为它从 2012 年就停产了)并且...它实际上只是淹没在 Visual Studio 风格的沉重涂料中的 FrontPage。

![Screenshot of MS Expression Web](../Images/1a25ef9f125dd5ecfd1db6fd5a5c6c41.png)

FrontPage 和 Expression Web 之间有 4 个主要区别。

#1: CSS。你无法避免它。您不再需要`<FONT>`和`<COLOR>`标签。相反，每次你改变字体或者其他什么的时候，它都会为你创建一个新的 CSS 类。我不喜欢*那样的风格，我更喜欢直列风格，但不管怎样。*

#2:他们正在推进 ASP.NET。很难。还记得网络机器人吗？他们走了。你知道你得到了什么吗？...几乎没有！没有计数器或导航栏。你*做的*得到的是自定义验证器和数据表之类的东西，老实说我不知道如何使用它们。此外，正如上面的樱桃，Linux 服务器上的 ASP.NET 支持在最近的开发之前一直是非常受欢迎的。网芯。有人可能会说这可以追溯到整个“统治世界”的事情，但这是一个坦率的令人困惑的产品。

#3:他们去掉了一半的网络管理功能。我的意思是，网络仍然存在。您仍然可以生成报告、查看超链接、发布到服务器等等。但是缺少两个关键特性:导航编辑器和协作特性。我知道取消对 FrontPage 服务器扩展的需求可能很难，但至少可以说它是*有用的*。此外，他们去掉了导航编辑器，因此也去掉了页面中的导航栏。这让你陷入了那个奇怪的石器时代，手动制作链接，并希望它们不会中断或过时。

最后#4:用户界面不是很好。一点也不。与 FrontPage 相比，这里的 UI 布局看起来绝对不合逻辑。你两边都有垃圾加上你可能不在乎的底部。(例如，文件夹视图和 CSS 属性可以很容易地合并到其他窗格中。)CSS 管理器，相比 FrontPage 的格式→样式菜单，意义不大。自动造型容易；只需点击格式按钮，你就完成了。实际上，赋予一个标签一个特定的样式是非常困难的，需要您在标签编辑器中手动设置这个类。此外，和 FrontPage 一样，制作一个站点范围的样式表比它应有的难度要大。网站管理器也有点奇怪，因为 FrontPage 2000+的巨大侧边栏已经被底部的小标签取代。而且，没有逻辑上的原因，尽管它是一个完全独立的东西，但它和页面编辑器共享所有相同的边栏。

总的来说，我的快速评论是:它肯定比 FrontPage 更容易掌握，而且它生成的 HTML 也更好。不幸的是，用户界面似乎受到了一群 Visual Basic 爱好者的冲击，在途中失去了一些关键功能，并获得了一些奇怪的 ASP.NET 服务器东西，没有人会使用。它令人困惑，是同时为两种受众(专业后端网络开发人员和完全新手)设计的，老实说，界面本身就让 FrontPage 更好。此外，除了更加依赖 CSS 和 ASP.NET 之外，与 FrontPage 相比，它并没有增加很多新功能。如果有的话，它只是删除功能。

### 梦想编织者

当然，也有非 MS 替代品。还有 Adobe 的(原 Macromedia 的)Dreamweaver。与微软的产品不同，Dreamweaver 仍在积极开发中。你可以从 Adobe 以每月 20 美元的价格购买。(当然，前提是你别无所求。)

我将在这里查看 Dreamweaver 的 2016 版本，因为这是我在记录自己使用它时使用的版本....2016.(这篇文章已经被搁置了一段时间。)

Dreamweaver 是一个有趣的小东西。当然，与 FrontPage 相比，我会随时使用 Dreamweaver。它实际上产生了当年有意义的网站。它允许您为移动响应创建媒体查询，并且通常产生更干净的代码。如果你懂 HTML 和 CSS，喜欢走捷径，Dreamweaver 完全没问题。但是，不幸的是，对于没有经验的用户来说，与 FrontPage 相比，这个界面仍然有所欠缺。(是的，我还是认真的。)

### Adobe 鼠标

![Screenshot of Adobe Muse from Adobe's website](../Images/bc4bf30e5c806c53c1fb45637841049b.png)

还有一个节目。土坯缪斯。出于各种原因，我不能拿到试用版，所以我会建议你[看看这个演练/教程视频](https://youtu.be/8eDuupJ9Uus)。你会注意到这正是我想要的。它有一个网络层次编辑器。它有导航栏之类的“小部件”。它通过创意云进行协作。它使用 CSS，并在模板和预先规划上蓬勃发展。怎么了?

答案:*厂商锁定*。你看，像 Dreamweaver 一样，你需要每月支付 20 美元来使用 Muse。如果没有，就不能用 Muse。似乎很简单。这么说吧，你一个月都忘了付缪斯的钱。假设你*需要*更新你的网站，*今天需要*。如果你不知何故丢失了你的 FrontPage 安装，你仍然可以用记事本之类的东西编辑这个网站，把它推到你的服务器上，FrontPage 服务器扩展就会启动并处理网络机器人。(如果他们不这样做，也不会是世界末日，因为 FrontPage 会在这种情况下缓存网络机器人的内容。)与缪斯，嗯...

![DIVs. DIVs everywhere](../Images/f050838cae6838aa0e490d8df46a5834.png)

这是来自 Adobe Muse 官方网站的片段，大约在 2016 年。没有缪斯，你*不能*编辑这个。很难理解你在这里看到的东西，因为它只是随机的 DIV 元素，带有 Muse 特有的属性和类。比方说，如果你想用文本编辑器在页面上添加一个新的部分，你就可以了。这种混乱的标签也使得很多事情，比如特定于设备的样式表或者障碍可访问性变得不可能。(需要证明？尝试打印 Muse 网站。从我的经验来看，这是一张白纸。)

不出所料，现在回想起来， [Adobe Muse 已经停产](https://helpx.adobe.com/muse/kb/adobe-muse-end-of-service.html)。值得庆幸的是，即使 Muse 软件已经停产，它仍然可以使用。不幸的是，这样做需要“Creative Cloud 所有应用程序订阅”，这是非常昂贵的。更糟糕的是，如果你想把*从* Muse 换成另一个产品，你必须完全重建你的网站，因为没有 Muse 就没有办法编辑 Muse 网站，也没有办法将 Muse 布局和风格信息导出到可用的 CSS 中。(有趣的是，这一节*的大部分是我在*发生之前写的，我有一个很大的警告，这是*将要*发生。)

### Wix、Weebly、Squarespace 等。

关于这一点，软件即服务网站也有一个新趋势，比如 Weebly、Wix 和 Squarespace。这些声称让一般外行人开发自己的网页。这些问题就像缪斯，但在类固醇。你不仅必须*只*通过他们的网络应用程序编辑你的网站，他们*拥有*你的网站。我的意思是，你仍然拥有你的文字和图片的版权，但是你的网站在互联网上的存在完全依赖于服务提供商。

假设其中一家破产了，无法托管网站。您的网站已经**关闭**。句号。他们可能有一个备份，但你没有办法轻松地将它迁移到其他地方。这是一种完全不同的完全封闭的形式！此外，就功能而言，你是 Weebly/Wix/任何人的心血来潮，有时需要付钱来“解锁”更多。(例如，使用 WiX，您需要每月支付至少 10 美元。如果你想删除广告，17 美元/月。就为了有个网店。)至少有了 FrontPage，如果你需要更多的功能，你可以创建自己的网络机器人。(当然，大多数人可能会为此付费，但至少这不是一个月的费用。)

如果你认为这些都不是问题，我想向你介绍一下 GeoCities。它曾经是一个简单的网络主机，有几兆的存储空间。然后雅虎收购了它。它获得了一些“酷”的、只有地理城市才有的功能，比如它的网站生成器和留言簿，以及一些你可以自己添加的奇特的 PHP schenanigans。然后，在 2009 年，GeoCities 面临了一个非常突然和非常*终结*的命运。Wix，Weebly，Squarespace 和其他人最终都会死。Geocities 是 90 年代的王者，现在看看它。请不要把所有的鸡蛋放在一个篮子里。

如果你留意他们，你会发现更多的网站编辑。这些要么像 Muse、Weebly、Wix 等被严重锁定。或者美化了的文本编辑器，除了语法高亮显示和一些常见 HTML 标签的快捷按钮之外，没有太多功能。

## 结论

好吧。那么秘书制作不出公司网页怎么办？谁在乎呢。请一个知道自己在做什么的网站设计公司，对吗？事情是这样的:我确实同意你应该找一个知道他们在做什么的网站设计公司，如果你在处理，比如说，金融交易，个人信息，大量的数据，或者非常复杂的服务器。但这并不是排除其他 95%的人的理由。

在我个人看来，互联网是一个非常酷的东西。这里充满了无数富有创造力的人。想要证据吗？见 Tumblr，YouTube，Twitter，Soundcloud，[我的链接页面](../../links/)等。但是把互联网锁在一些预先定义好的角落里是一件坏事。[我在以前的一篇文章中详细讨论过这个问题。但是如果你想做一些不同的事情呢？以这个网站为例。当然，它*可以*是一个 WordPress 博客，它*可以*也可以。](../31/)

但是，比如说，我想在这个网站上开始一系列交互式编程教程。假设我正在教你如何[制作一个 Game Boy 游戏](../18/)，我想要一个代码段。在边上会有解释一段代码如何工作的文本，在一个小文本框*中你可以进行试验！除非你掌握了这个概念，否则你不能继续。(关于这类事情的例子，请参见优秀的[着色器网站](https://thebookofshaders.com/)的书籍。)如果我想在我假想的 WordPress 网站上这么做，我就要倒霉了。我需要的和 WordPress 提供的是完全不同的两件事。*

或者说有个音乐人想做个网站。(没什么特别的原因，就说 1983 年左右的丹尼斯·德扬(Dennis DeYoung)吧，他穿着那套粉色西装。)他不希望*只是*一个千篇一律的 Bandcamp 页面。也许他觉得自己在做实验。谁说他不能在歌曲灵感简介和 iTunes 链接旁边放一个现场表演的视频？或者是为了激起兴趣的某种[独特的互动体验](https://musiclab.chromeexperiments.com/)？乐队露营。就是他。

或者说，一个来自阿拉巴马州的加德纳人明天早上突然有跑马拉松的冲动。几百场马拉松。在做了那件事并且没有死之后，他想利用他的知名度赚钱。所以他制作了一个很酷的网站，上面有一张地图，可以记录他跑步的地点，距离，速度等等。然后，他可以在时事通讯或博客或其他东西中获得关于耐力跑和美国各地酷点的提示。(并不是说这个来自阿拉巴马州的*特殊的*加德纳会困扰，但也许他的粉丝会。)

好了，荒谬的假设够多了。关键是，互联网是创新之地。把 95%不懂网页设计的人扔出去，而且*永远也不会*阻止了互联网的许多创造性用途。现在，可怜的老 Dennis DeYoung 除了得到一个 Wix/Weebly/随便什么网页或者雇佣一个网页设计师之外，在 T2 没有一个展示他音乐的地方。《阿甘正传》目前最多只能使用 Tumblr，该网站最近也出现了一些问题。

你看着类似于[neoccities](https://neocities.org)的东西，你会看到无尽的、完全自由的创造力。你有[我的小马插曲追踪器](https://mlpfim.neocities.org/)，[一个夸张的 2000 年致敬网站](https://exo.pet)，[一个喜欢尝试很酷的 JavaScript 东西的家伙](https://mrdoob.neocities.org/)，[非常实验和艺术的网站](https://town.neocities.org)等等。问题是，所有这些人都在某种程度上了解 HTML 和 CSS。想象一下那些想做这种东西的人，但是只有*懂艺术或音乐之类的东西。*

我想说的是，我们需要一种简单的方式，让任何人都能创作任何东西，并把它放到互联网上。卡在“无法控制”和“极其昂贵和耗时”这两个极端之间是不够的。尽管这听起来很傻，微软/弗米尔的 FrontPage 几乎就是那个东西。它让多人在一个网站上工作，并提供规划工具和所见即所得的编辑。问题是，它停留在 90 年代。FrontPage 从来没有产生过“好”的网站；它只是产生了一些与旧的 GeoCities 粉丝网站相当的东西。(此外，它归微软所有，已经完全消亡，这也于事无补。)

我们需要一个新的首页。一个使用 HTML5、CSS3、现代 JavaScript 和很酷的 PHP 的 FrontPage。FrontPage 遵循现代设计原则，如语义 HTML 和强大的 CSS 样式。一个完全开放的首页，完全不是世界统治计划中的一枚棋子，完全不被一个贪婪的供应商所束缚，完全不被抛弃。一个 FrontPage，输出有用的，有感觉的，简单的 HTML 和 CSS。一个 FrontPage，它集成了版本控制软件(如 Git)和计划程序(如 KPlan)以及通信程序(如 email 或 IRC ),以实现有计划、有意义的协作。一个 FrontPage，让人们像今天人们做 WordPress 插件一样写和分享有用的和多功能的网络机器人。(尽管可能使用 JavaScript 而不是定制的服务器端插件。)

总之，我们需要一个首页，让才华横溢的普通人用创造力和独创性重新掌控互联网。因为尽管脸书和推特都很好，但让我们面对现实吧，我们都有点怀念那些糟糕透顶的手工制作网站的旧时光。
