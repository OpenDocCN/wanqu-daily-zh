# 启动想法清单| defmacro

> 原文：<https://www.defmacro.org/2019/03/26/startup-checklist.html?utm_source=wanqu.co&utm_campaign=Wanqu+Daily&utm_medium=website>

我一直在修补不同的创业想法，需要一个好的清单来思考它们。已经有了很好的模板:[YC 的应用程序](https://apply.ycombinator.com/)，亚马逊的[内部新闻稿](https://www.quora.com/What-is-Amazons-approach-to-product-development-and-product-management/answer/Ian-McAllister)，红杉的[撰写商业计划](https://www.sequoiacap.com/article/writing-a-business-plan/)。我发现自己混合和调整这些模板，因为它们不完全符合我的世界模型，所以我写了自己的列表。

我用这个列表来发展想法和过滤它们。如果你采用它，小心使用它作为一个过滤器。记住，在早期阶段，好的想法是非常容易被扼杀的。

# 产品

1.  用户是谁？
    <= 70 char
    [【唐·瓦伦丁】](https://www.sequoiacap.com/article/remembering-don-valentine/):“谁在乎？”
2.  他们不满的本质是什么？如果他们读到这个答案，他们会说“谢谢，我希望我也这么想过”。
    <= 240 chars
    [@ Benedict Evans](https://twitter.com/benedictevans/status/1110538673873805314):“iTunes Store 解决了一个用户问题。App Store 也是如此。Spotify 和苹果音乐也是如此，实际上苹果新闻也是如此。但是苹果委托电视节目解决的是什么用户问题？”
3.  你在为他们建造什么？
    <= 70 chars
    [彼得·德鲁克](https://www.amazon.com/Effective-Executive-Definitive-Harperbusiness-Essentials/dp/0060833459/):正在设计的产品是**给**的客户，还是的**给客户？**
4.  写一条假设客户的推文，解释产品以及它如何消除他们的不满。
    [@BrianNorgard](https://twitter.com/BrianNorgard/status/1110915013085028353) :“没人关心你的产品。谁建造了它，它的特点，它的起源——都是多余的。人们只在你的产品现在能为他们做的事情中发现价值。节省人们的时间。帮人省钱。给人一个退路。**自私的手将永远掌权。彼得·德鲁克:你真的在尽最大努力帮助客户吗？**
5.  为你的产品发布写一篇博文标题。令人惊讶吗？是新的吗？你的目标客户会想点击它吗？他们会想要分享链接吗？第二天他们还会分享吗？
    < = 70 个字符
6.  写下你的产品发布博文的第一段。包括产品名称、产品说明、目标市场、主要优势以及行动号召。
    < = 240 个字符
7.  你的目标客户关心什么样的“好的标准”?在这些指标上，你的产品是否在所有可供选择的产品中独占鳌头？(例如，你能做什么别人做不到的事？)
    <= 240 chars
    **参见:** [越崛起越差越好](https://www.jwz.org/doc/worse-is-better.html)，[越差越差](https://www.artima.com/weblogs/viewpost.jsp?thread=24807)，[Amazon.com 解释了](https://twitter.com/profgalloway/status/1192777681789739008)
8.  你的产品是不是尽可能的棒？大概不会。
    y/N
    **参见:** [埃隆马斯克建议 CEO 们](https://www.wsj.com/articles/elon-musk-advises-ceos-to-stop-wasting-time-on-powerpoint-meetings-11607470065)

# 增长

    < = 240 个字符

    < = 70 个字符
15.  你的产品是迈向这个里程碑的可信的进步吗？

# 意义

18.  达到你的 25 年里程碑对世界意味着什么？这个未来*真的*刺激吗？为了传送到那里，你愿意放弃多少年的生命？如果你发现自己在这个反事实的世界里，你会想回去吗？
    <= 140 chars
    **参见:**[1995 年美国在线商业](https://twitter.com/JonErlichman/status/1181646807924953088)，[史蒂夫·乔布斯在 MSPDC 1996 上展示 web objects](https://www.youtube.com/watch?v=DmCu97u35Ek)
19.  如果是另一家公司而不是你在研究这个想法，你会怎么想？你会加入他们吗？
    是/否
20.  想象你站在你的团队、投资者、家人和朋友面前。你失败了，他们在等你说话。你会说什么？鉴于失败是默认的，你愿意解决这个问题吗？
    <= 480 chars
