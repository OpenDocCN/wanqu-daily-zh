# 我是乔尔·加斯科因，这是《缓冲区》背后的故事

我喜欢在推特和脸书上分享东西，但我完全不一致。有一天我会发几条微博，然后三天没有任何消息，然后疯狂分享和转发。如果你和我一样，是你的新宠应用。它为你安排你的推文和脸书帖子，创造一个一致的社交媒体存在。(它还 [支持](https://lifehacker.com/buffer-streamlines-sharing-across-multiple-social-netwo-5967425)App.net 和 LinkedIn，并在 iOS 和 Android 上可用。)我们和 Buffer 的联合创始人 [Joel Gascoigne](https://twitter.com/joelgascoigne) 聊了聊这款应用的灵感，他面临的最大挑战，以及庆祝 100 万用户的感受。

#### 缓冲区的想法从何而来？你是在试图解决你经历过的问题，还是灵感来自其他地方？

Buffer 的想法是在我使用 Twitter 大约一年半之后产生的。我开始分享我觉得鼓舞人心的博客帖子和引文的链接，我发现我的追随者似乎真的很喜欢这些类型的推文。我经常会被转发，或者最终围绕博文或引语展开一场精彩的对话。就在那时，我决定要更频繁地分享这类内容，因为引发的对话让我能够与一些超级聪明有趣的人接触。

所以，我的目标是分享更多的博客文章和引用，我开始手动做这件事。我很快意识到，将来安排这些推文会更有效，所以我开始使用一些可用的 Twitter 客户端来做这件事。我在这里遇到的主要困难是，我必须选择发推文的确切日期和时间，而事实上，我只想“每天发五次推文”。我只是想把推文分散开来，这样我就不会在每天阅读的时候同时分享了。有一段时间，我用一个记事本记录我计划发微博的时间，这样我就可以每天试着发五次微博。这变得相当麻烦，因此我的想法诞生了:我想让安排“一天 x 次”发推像定期发推一样简单。

#### 你想出这个主意后，下一步是什么？

我花了一年半的时间建立了另一家创业公司，并没有取得太大的成功。我犯了很多不同的错误，比如假设人们想要什么，只是添加功能而没有得到任何验证。在那段时间里，我被 Eric Ries 的作品和他写的 [精益创业方法论](http://theleanstartup.com/) 迷住了。我是他的早期博客用户之一，我看了他的许多演讲。我之前创业的失败加上精益创业的想法给了我一个重要的认识:如果我想让这个新想法不仅仅是我自己的爱好和工具，我需要测试其他人是否会觉得这个想法有用。

我在推特上和我的粉丝分享了这个。我很幸运在这一点上有 1700 名追随者，因为他们积极分享我所有的更新和从我以前的创业中吸取的教训(这是我推荐成为超级透明的的一个原因)，所以我有相当多的人点击这个登陆页面并输入他们的电子邮件。他们一输入邮箱，我就给他们发了一封非常个性化的邮件，我询问他们对 Buffer 的反馈。我跟踪了这里的指标，但是实际的对话才是最有用的。

一堆人在这一点上不了解产品，相当一部分人表示不会用。幸运的是，我遇到了一些人，他们认同我自己的痛点，他们也想要一个解决方案。在构建完整的产品之前，我做的下一个测试是看看是否有人愿意为此买单。所以，我在这两页之间加了第三页:

我又发了一条关于登陆页面的微博，让更多的人分享了他们的电子邮件。再说一次，在这一点上，这真的不是关于数字或指标，而是关于对话。这一次，我试图将一些对话引向定价，特别是弄清楚人们是否会为产品付费。同样，许多人认为这是荒谬的，或者只是没有看到其价值。我们学到了很多关于如何更好地描述缓冲的知识。与此同时，有几个人说，如果这个产品存在，他们会为它付费，这让我看到了一丝希望。该是我开始工作并构建完整产品的时候了。

(实际上，我在之前的一篇名为《7 周内给付费客户的 [*创意:我们是如何做到的*](http://blog.bufferapp.com/idea-to-paying-customers-in-7-weeks-how-we-did-it) 》的文章中写下了这项工作的全部细节。)

#### 你如何选择目标平台，忽略或等待哪些平台？

这成了一个相当简单的决定。我非常喜欢下面这句话:

“尽你所能，用你所拥有的，在你所在的地方。”——西奥多·罗斯福

最终，我们的收入会增长到足以让我和我的联合创始人利奥跳上飞机，在旧金山找到立足之地。然而，首先，我在英国的伯明翰，我决定试着用我所拥有的东西让它工作。这也有助于我决定瞄准哪些平台。我只针对我个人知道如何编码的平台。所以，我纯粹从网络开始，我建立了一个简单的网络应用和一个 Chrome 扩展。我不知道如何建立一个 Android 或 iPhone 应用程序，所以这些必须等待。

碰巧的是，这些限制也是最快速验证和发展创业公司的好方法。你看，如果你尝试在所有平台上发布，它将会推迟发布，这意味着你需要更快地等待，看看是否有人想要你正在构建的东西。我大幅削减了功能集，幸运的是，我能够在七周内推出该产品的第一个版本。

另一个关键决定是是否支持多个社交网络，尤其是脸书。我决定只支持 Twitter，因为这是我想自己解决的首要问题。事实证明，Twitter 和脸书是非常不同的网络，只从 Twitter 开始是明智的，因为我们想做一个只在少数事情上很棒的产品，而不是在许多事情上“还行”。

这也是我在帮助早期创业者时经常遇到的话题。我一直给的建议是，尝试从产品的第一个版本中尽可能多的删减。这样，当你发布时，你将有希望得到相当二元的反馈:要么人们喜欢那个特性，要么他们觉得它没用。从人们喜爱的单一功能中找出要构建的东西要比推出一堆东西并找出哪些有用哪些没用更容易。

#### 你最大的障碍是什么，你是如何克服的？

当我第一次开始 Buffer 的时候，我是一名全职的自由网络开发者。很难找到足够的时间来适应边上的缓冲。起初我在晚上和周末工作。我试图让它工作的一个关键方法是推出一个超级迷你的产品，这在这次挑战中帮助很大。

最终，我有了一个重大突破，我把早期的成功归功于此。我意识到，在为别人工作了一整天之后，我经常在晚上尝试缓冲。我一点也不精神，我经常很累，没有效率。所以我把事情颠倒过来——我开始醒得更早(大约早上 6 点)，在开始自由职业之前工作 90 分钟，而我是完全精力充沛的。

如果你正试图建立自己的创业公司，并且真的想要成功，你应该问问自己优先考虑的是什么。对我来说，建立我自己的产品和公司，并达到可以完全专注于自己产品的程度是我的最高目标。这比其他任何事情都重要，所以我决定将 Buffer 作为我一天的第一项活动。

#### 对你来说，发射是什么样的？

我回顾 Buffer 的推出是一个巨大的成功——对我来说这是一个里程碑式的成就。虽然数字并不大，但我们确实花了很长时间才开始看到巨大的牵引力。在第一个月，我们有大约 100 名注册用户和前三名付费用户。第一个付费用户是在网站上线三天后——他们在我的 PayPal 账户上支付了 5 美元，我在房间里跳来跳去。与我从事自由职业的收入相比，这只是很小的一笔钱，但这是我第一次用自己创作的产品在网上赚钱。我不再用我的时间来换取金钱。向 PayPal 支付 5 美元购买一个我可以扩展的产品，与自由职业者的数千美元相比有着天壤之别，我无法以同样的方式进行扩展。

但是 100 个注册用户和 3 个付费用户并不能被视为一次成功的发布。事实上，我们花了整整六个月的时间才放下我的自由职业工作。我们只是不断向前推进，我的联合创始人利奥努力想出如何通过内容营销获得牵引力。当我们开始采取“多次发射”而不是一次“大爆炸发射”的方法时，我们的思维方式发生了最好的转变。我们的目标是在 3-4 周内发布某种新闻，介绍我们推出的新功能或我们达到的里程碑。这给了我们复合增长。

#### 你如何有效地处理用户的要求和批评？

从一开始，我就非常注重与用户交流，并为他们提供快速有效的支持。这现在形成了我们愿景的一个关键部分，事实上，我们团队的 25%致力于客户支持，团队的其他成员也定期在这上面花费时间。我们有一份每月的快乐报告，分享我们如何在 Buffer 开展客户支持的所有见解，以及我们获得的实际数量。例如，2013 年 8 月，我们进行了 4，335 次电子邮件对话，在 1 小时内回复了 52%的电子邮件，98%的人对他们的体验感到满意。

我们有超过 1，000，000 名用户，我们承诺为每一位用户提供出色的支持。我们做到这一点，部分是通过努力工作和雇用优秀的人才，还有一些工具。我们使用 [帮助侦察](http://helpscout.net/) 进行电子邮件支持，使用 [Sparkcentral](http://www.sparkcentral.com/) 通过 Twitter 进行支持。我确信我们的首席快乐官 Carolyn 会同意我的说法，如果没有这些不可思议的工具，我们会淹没在支持中。它们都允许我们有一个共享的收件箱，这样整个团队可以一起处理收件箱，它们也帮助我们保持超级个性化。例如，Help Scout 添加零票号信息—对于收到电子邮件的人来说，这只是一封与我们其中一人的纯文本电子邮件。他们喜欢这样。

在产品方面，客户支持是一个关于产品需要改变的惊人的洞察力宝库。这就是为什么我们热爱支持，并总是要求人们与我们联系。我定期与 Carolyn 同步，听取最常见的问题或功能需求，然后我们直接基于这些见解来优先考虑我们的路线图。这有助于我们改进我们的产品，并发布我们预先知道人们会喜欢的功能。

Buffer 的一个关键价值观是同理心和谦逊，当我们受到批评时，这有助于我们。人们很容易收到一封电子邮件，却忘记了其中的人际互动——忘记了在另一端是一个真实的人花时间一个接一个地打出所有这些单词。我个人处理这类邮件的方法是大量感谢他们。如果他们对某件事不满意，并花时间联系，这表明他们真的在乎。没有多少人愿意告诉我们哪里搞砸了，所以这是非常有用的信息。最酷的是，通过真正从他们的角度看问题，并与他们产生共鸣，最直言不讳的批评者可以成为你最大的福音传播者。

#### 现在，如何在开发新功能和管理现有功能之间分配时间？

这是一个很好的问题。事实上，我们最近一直在思考这个问题。太多的新功能和产品将随着时间的推移而恶化。在现有功能上做太多工作，增长可能会放缓。我们试图取得平衡。这是我和我的联合创始人最近一次头脑风暴的照片:

因此，我们通常的目标是在新功能和修复及改进现有功能之间各占一半。我们还有一种文化，所有工程师都定期提供客户支持。这有助于工程师看到真实的人在用他们正在构建的功能做什么，通常这意味着更容易理解任何错误或困惑，并且他们更快地得到修复。然后，我们有 10%的时间在重构上，这意味着“照料花园”,随着时间的推移，使我们的代码库成为一种乐趣。

#### 你会给那些想从事类似项目的人什么建议？

如果你是第一次创业，我建议你试着找出真正的痛点，这是我在这里 详细写过的 [。一个关键是，你假设你可能还没有找到一个，(所以要严格验证——与人们交谈，并检查用户是否会回来)。](http://blog.bufferapp.com/idea-to-paying-customers-in-7-weeks-how-we-did-it)

另一件事是考虑如何过渡到全职创业。对我来说，最好的方法是通过对产品收费来尽早创收。作为第一次在英国创建公司的人，我几乎不可能获得资金来全职从事 Buffer 的工作。当我们筹集到 45 万美元的种子资金时，我们已经有了 55000 名用户，每个月赚 13000 美元。如果没有这种吸引力，我们真的很难筹集到资金，所以我建议其他人开始尝试建造人们愿意支付的东西。
