# 克里斯·格兰杰——编码不是新的文化

> 原文：<http://www.chris-granger.com/2015/01/26/coding-is-not-the-new-literacy/?utm_source=wanqu.co&utm_campaign=Wanqu+Daily&utm_medium=website>

尽管让人们编码的运动背后有良好的意图，但基本前提和方法都是有缺陷的。该运动基于“编码是一种新的读写能力”的想法，但这是对什么是真正的读写能力的狭隘看法。

如果你问谷歌如何定义读写能力，它会给出一个机械的定义:

> 读写能力。

这当然是准确的，但是将识字定义为在一张纸上解释和做标记是非常不够的。阅读和写作是我们用来利用更重要的东西的物理行为:外部的、可分配的头脑存储。有文化不仅仅是能够把单词写在纸上的问题，它还能固化我们的思想，使它们能够被书写出来。解释和应用别人的思想相当于阅读。**我们称这些为作文和感悟。它们才是真正的文化。**

### 编码不是基本技能

当我们说编码是一种新的文化时，我们认为挥舞铅笔和纸是一种旧的文化。编码和写作一样，是一种机械的行为。我们所做的只是升级存储介质。编写 if 语句和 for 循环对人们来说很简单，但并不能让他们变得更有能力。就像写作一样，我们必须知道如何固化我们的想法，并把它们从我们的脑海中抹去。然而，就编程而言，如果我们设法以某种方式做到这一点，计算机就不仅仅是存储它们了。它可以用它们来计算。

阅读和写作给了我们外部和可分配的存储空间。编码给了我们外部的和可分配的计算。它允许我们卸下为了执行某个过程而不得不做的思考。为了实现这一点，似乎我们所需要的只是向人们展示如何给计算机下达指令，但这是在教人们如何把单词写在纸上。我们需要相当于作曲的技巧，让我们思考事物是如何计算的。这一次，我们没有记录我们的想法，而是记录世界的模型，让我们一开始就有想法。

我们为一切事物建立心智模型——从如何系鞋带到宏观经济系统的运作方式。有了这些，我们做决定，预测，理解我们的经历。如果我们希望计算机能够为我们计算，那么我们必须准确地从我们的大脑中提取这些模型并记录下来。编写 Python 不是我们需要教给人们的基本技能。建模系统是。

### 建模是一种新的文化

就像写作和理解不依赖于纸张一样，建模也不依赖于计算机。可以是身体上的，也可以是精神上的。它发生在纸上和 Excel 中，或者用乐高和轻木飞机。这是一项非常强大的技能，通过将我们的模型移植到计算机上，我们可以更好地利用它。为了理解我们如何做到这一点，我们必须更深入地了解建模意味着什么。

> 建模是创建一个可以探索或使用的系统(或过程)的表示。

这个定义包含了一些技能，但最重要的是规范。为了表现一个系统，我们必须明白它到底是什么，但是我们的理解却深陷于假设之中。一个有趣的例子是，我的联合创始人罗伯特(Robert)第一次试图对一系列名字进行排序，结果却很失望。我问了他一个简单的问题:“按字母顺序排列某物*实际上*是什么意思？”他从来没有考虑过这到底意味着什么。这是一些模糊的概念，他有一个心理模型，但他从来没有被迫指定确切的结果(例如，我们如何处理非英语字符？).

定义一个系统或过程需要将它分解成几个部分并定义它们，然后可以进一步分解。这是一个帮助承认和消除歧义的过程，也是教会人们建模的最重要的方面。在分解零件的过程中，我们可以把极其复杂的东西用我们理解的术语和我们知道如何做的行动组织起来。这里编程的相似之处非常明显:编写程序就是分解一个系统，直到你得出一个*计算机*理解的想法和它知道如何做的动作。在这两种情况下，我们都必须详细说明我们的系统，并且通过一个迭代的过程来完成。

### 创造就是探索

我们用材料制作模型。有时我们的材料是木头、金属或塑料。其他时候是来自我们感官的数据或信息。不管怎样，我们从我们塑造的媒介开始我们的模型。这有助于我们逃离空白页面的陷阱——从一大块粘土(或数据，或一些物理运动，或乐高积木……)开始，我们不必想象从虚无到某物的飞跃。相反，我们可以通过操控眼前的事物来开始工作。我们可以磨练，削，弯，拉，粘。我们可以按照自己认为合适的方式塑造它。

制作模型的过程是一个发现的过程——我们在塑造材料时会发现我们需要什么样的零件。这意味着我们不需要完全指定一个系统来开始，我们可以在进行中简单地制作新的作品。我们最终会在创建系统时探索它，而不需要获得一个“完整的”模型来获得价值。我们可以简单地修补。我们可以刮掉一些或者粘上一些，看看会发生什么。在这个过程中，我们增强了对系统行为的直觉。

### 探索就是理解

物理建模告诉我们能够把东西拆开的价值，无论是拆下每一个螺丝，把整个发动机组放在车库地板上，还是只是拆下交流发电机看看。通过把某物拆开，我们可以直接探索它是由什么组成的。这就是为什么像钢铁侠这样的电影中的界面如此引人注目——它们让我们可以一头扎进去。

想象一下，如果我们有能力分解任何东西，深入其中，看看那一点点有什么作用，我们会学到什么。我们找到的表示系统的方式越多，我们保留这种能力的方式就越多，我们理解复杂事物的能力就越强。

我们通过探索获得理解，通过摆弄模型和“玩出来”看看会发生什么。我们可以把轮子变大，或者插入不同的利率数字，然后模拟结果。通过这个过程，我们经历和想象了一些我们无法想象的事情，经历越多，我们就越能理解这个世界是如何运作的。

### 数字自由

虽然物理建模的特性对我们来说是有用的指导原则，但数字世界为我们提供了一个走出局限的机会。我们可以自由地创建零件的副本或制作它们的许多不同版本。我们可以对单个部分进行更改，系统会适应它们。我们可以在不同的环境中模拟我们的模型，而复杂的工具验证我们的预期。通过将我们的模型移植到计算机上，我们可以卸下改变、模拟和验证的必要工作。我们甚至可以让我们的模型建议行动，并让其他系统为我们执行这些行动。因此，我们解放自己去探索，而不需要每一个变化都要求我们去隐喻的机械商店。

有许多工具已经帮助我们做到了这一点——从 [Matlab](http://www.mathworks.com/products/matlab/) 到[Quartz Composer](http://en.wikipedia.org/wiki/Quartz_Composer)——但是 Excel 无疑是王者。通过 Excel，我们可以对任何系统建模，我们可以用网格上的数字来表示，事实证明，这是很多数字。我们已经模拟了从整个商业到市场到家庭度假的一切。数百万人能够使用电子表格来模拟他们生活的方方面面，可以说，除了互联网，它是我们在电脑上可以使用的最重要的工具。它通过提供一套简单直观的工具来塑造一种材料，即数字网格，从而获得了这种力量。然而，如果我们想做更多的工作，我们必须编码。

### 根本性的脱节

编码要求我们将系统分解成计算机能理解的动作，这代表了意图上的根本脱节。大多数程序并不试图指定事物如何在内核间分配，或者对象应该如何在内存中布局。我们并不试图模拟计算机是如何工作的。相反，我们正在模拟人类互动、天气或宇宙飞船。从那个角度来看，这就像试图用焊接工的焊枪来绘画。我们使用了一套设计用来模拟计算机如何工作的工具，但是我们展示的系统与它们完全不同。

即使在我们具体讨论机器应该如何运行的情况下，我们的工具也没有真正按照建模的概念来设计。例如，我们的编辑器和调试器很难在不同的抽象深度挑选出片段。相反，我们必须从整体上审视这个系统，并试图弄清楚所有的螺丝钉是从哪里来的。大多数主流语言也使探索性创作变得困难。当我们在构建一个系统时，探索这个系统会给我们一个更好的直觉，让我们知道我们拥有什么和我们需要什么。这就是为什么*被*设计用来探索的语言(LISP、Smalltalk 等)看起来很神奇，并拥有狂热的追随者。但是，即使是这些，也不得不用单一的工具来模拟每一种材料。尽管不同的物理材料有不同的工具，但在编程中，我们试图只用一种工具来构建几乎所有的东西:通用编程语言。

从表面上看，拥有“一个工具来统治所有人”似乎是可取的，但现实是我们最终试图用厨师的刀来锤打金属。相比之下，Excel 将我们限制在它有意设计的单一材料上。通过这种约束，我们获得了一个工具，它有一个非常直观和强大的界面来处理网格。当然，问题是 Excel 做其他任何事情都很糟糕，但这并不意味着我们应该试图将厨师的刀概括为锤子。相反，我们应该使用合适的工具来完成这项工作，并寻找一种可以让我们将不同材料结合在一起的胶水。

现在存在的编程迫使我们建模，但它是以一种不自然的方式进行的。虽然教会我们自己如何编程将有助于我们学习如何打破系统，但这样做的风险是关注错误的事情。 <sup id="fnref:risk" role="doc-noteref">**我们不希望一代人被迫关心 Unicode 和 UI 工具包。我们需要能够利用计算机的一代作家、生物学家和会计师。**</sup>

### 我们如何教育孩子

我们是天生的建模者，我们通过创建我们对世界进行验证的表示来学习。我们的模型往往开始时过于简单，甚至明显错误，但这是完全可以接受的(也可以说是必要的),因为我们可以继续更新它们。任何父母都可以给你举一些例子，说明这种情况如何在他们的孩子身上发生，尽管他们可能没有完全意识到这就是正在发生的事情。一个很好的例子是，婴儿最初有一个依赖于看到物体的存在模式。这就是为什么看不到他们的父母是如此令人苦恼——这意味着他们不再存在。<sup id="fnref:permanence" role="doc-noteref">物体持久性的概念是儿童在几年磨练他们的物理物体行为模式后才完全发展起来的。</sup>

所以如果我们自然地这样做，我们有什么要教孩子的呢？本能的神奇之处在于，我们不必意识到它实际上是如何发生的。我们通过模型了解世界，但这并不意味着我们知道我们最初是如何创造它们的。因此，我们必须教会孩子们建模是如何发生的，我们可以将其分为四个不同的过程:

**规格**:如何分解零件，直到得到你理解的想法和行动。

**验证**:如何对照现实世界或者对照我们头脑中的期望来测试模型。

**调试**:如何分解一个模型中的 bug。一个非常重要的教训是，无效的模型不是失败，它只是表明系统的某个部分的行为与我们建模的不同。

**探索**:如何利用模型来更好地理解可能的结果，并了解如何用它来预测或自动化某个系统。

专注于这四个领域抓住了我们创建、磨练和使用模型的基本过程，它让孩子们在学习过程中成为积极的参与者。或许更重要的是，专注于建模将教育推向了这样一种理念，即教育学实际上是在引导儿童有意识地创造和探索他们周围的世界。

### 我们应该如何教育成年人

实际上，我们应该教自己同样的东西，但不幸的是，成人教育很少允许无指导的探索(我们只是想做点什么)。相反，我们可以根据如何在上下文中使用模型来构建建模。例如，如果您是一名会计师，每天都要经历相同的过程来创建一份报告，那么您可以将其分解，看看如何实现部分自动化。创建该报告所需的基本操作是什么？有没有工具可以帮你做这些事情？

**如果我们假设在某个时候会出现更好的建模工具<sup id="fnref:us" role="doc-noteref">，那么能够对一些系统或过程建模可能就是你自动化所需要的全部。</sup>**一旦出现这种情况，有针对性地探索如何对特定领域进行建模就有了明显的价值，因为它将我们解放出来去做其他任务。一个极端的例子可能是对你的整个业务建模，从与客户的互动到履行。在这一点上，你可以自动化 it 的大部分，并专注于如何发展业务，这本身可能涉及探索性的模型。

### “计算机革命还没有发生”

1997 年，艾伦·凯在 OOPSLA 做了一个题为“计算机革命还没有发生”的演讲，他在演讲中认为我们还没有意识到计算机能为我们提供的潜力。十八年过去了，我还是同意他的观点——还没发生。教人们如何循环列表也不会让它发生。为了实现计算机的潜力，我们必须专注于让我们能够驾驭外部计算的基本技能。我们必须创造出新一代的工具，让我们不用转换职业就能表达我们的模型，以及使用它们的新一代建模者。

> 简而言之，人类能力的下一个巨大进步来自于能够将我们一生所创造的心智模型具体化。

这是新的文化。这是我们期待已久的革命。
