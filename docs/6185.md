# 这场战争微软本应获胜

## 微软有许多优势成为主导的移动操作系统，但许多问题，许多是本土的，让它失去了与谷歌和苹果的早期开端。

2012 年,《名利场》[发表了一篇关于鲍尔默领导下的微软和“失去的十年”的长篇报道。我重读了很多遍，有一句话一直在我脑海中闪现:](http://www.vanityfair.com/news/2012/07/microsoft-downfall-emails-steve-ballmer)

“你看着 Windows Phone，不禁会想，微软是如何浪费他们在 Windows CE 设备上的领先优势的？他们遥遥领先，领先数年。他们彻底搞砸了。因为官僚主义，他们彻底搞砸了。” [艾德·麦卡希尔](https://www.linkedin.com/in/edmccahill)

这句话特别针对我，因为我在 2002 年作为实习生加入了微软的 Windows Mobile 团队，然后在 2003 年成为全职员工，当时我真的觉得我们有机会成为即将到来的移动电话向互联网连接计算设备发展的领导者之一。

对于那些可能只记得智能手机竞赛是 Android 与 iOS 之战的人来说，有必要简要总结一下我们的由来:

我们向智能手机的共同发展始于个人数字助理(PDA)。众所周知，苹果拥有命运多舛的牛顿。 [Palm](https://en.wikipedia.org/wiki/PalmPilot) 是第一款面向大众市场的移动 PDA。微软很快推出了基于其 WindowsCE 代码库的 PocketPC。Symbian(由欧洲移动操作系统公司 Psion 制造)与诺基亚合作，在 2001 年推出了 9210 Communicator，并集成了 PDA 功能(值得注意的是，诺基亚早在 1996 年就推出了装有操作系统的手机，Communicator 9000 被许多人认为是第一部“智能手机”，98 年的 9110 也运行 GEOS)。



Nokia 9110 Communicator



所以在 1999 年，当比尔·盖茨宣布基于 WinCE 内核的智能手机计划时，操作系统世界已经一片混乱。但有人可能会说，微软有很多优势赢得这场战斗:

*   一个坚实稳定的嵌入式操作系统代码库，带有 WinCE 和 PocketPC 中不断增长的 PDA 平台，集成了桌面上熟悉的应用程序和用户体验
*   与芯片制造商和原始设备制造商的关系应该允许它将 Wintel 时代的模式复制到智能手机上，微软提供操作系统、参考设计和营销资金，原始设备制造商制造硬件并将其推向市场
*   一群管理良好且广泛的应用程序开发人员，他们与微软生死相依，肯定会支持微软向移动平台的新转变。这也包括微软自己的应用程序，如 Outlook、Word、Excel、MSN Messenger、Internet Explorer 和 XBox 资产。
*   银行里的大量资金用来购买客户和市场份额

那么，问题出在哪里？微软是如何失去成为领先移动操作系统的机会的？【注意:在 2002 年至 2006 年间，我只是 Windows Mobile 团队中一个微不足道的 60 后 MBA。这些见解仅仅是我的一些看法，这些看法来自于我在微软的经历，来自于我从 WAP 早期就进入了移动互联网领域，也来自于几年后参与了谷歌的移动努力。我相信，在 117 号楼的会议室里，会发生更多战略性的、深思熟虑的讨论]

*   **微软在太多的战线上作战**:如下图所示，早期的移动操作系统战争不仅仅发生在诺基亚和微软之间，也发生在 Palm 和加拿大由寻呼机转变为药品的 RIM/Blackberry 之间。诺基亚/塞班拥有与美国以外运营商的关系(以及大部分手机和智能手机市场份额)。Palm 在 PDA 有一群追随者，并通过 Treo 进入了连接设备。RIM 公司采取了一个聪明的策略，将它的优秀产品植入银行业 MDs，成为有抱负的 i-bankers 的必备设备。要打所有这些仗，Windows Mobile 必须为太多人提供太多的东西。一款和 Palm 一样好的 PDA 替代品，一款和诺基亚 phone 一样好用的手机，一款和电子邮件以及 RIM 一样同步的联网设备。最后它太多了，塞不进一个小盒子里。



*   **移动原始设备制造商不是微软惯于摆布的家伙**:惠普、戴尔、IBM 都是 Wintel 卡特尔中心甘情愿的同谋。但它们都没有在移动生态系统中扮演重要角色。在 2000 年代早期，主要的手机厂商是不同的厂商诺基亚、西门子、索尼爱立信、三星…



Source: [http://www.economist.com/node/1454300](http://www.economist.com/node/1454300)



看到硬件制造商在个人电脑世界的边缘化，他们中的许多人都不愿与微软合作，这就是为什么微软被迫推出其第一代智能手机，采用未知的“ODM”[HTC](https://en.wikipedia.org/wiki/HTC)。[注意:2003 年我在微软的工作是与三星和摩托罗拉这两家一线手机制造商合作，与摩托罗拉和世界各地的运营商合作推出第一批品牌智能手机 MPx200 和 i700。两者都不会取得巨大的初步成功】。SonyEricsson 直到很多年以后才同意开发 Windows Mobile 设备，而诺基亚……我们知道 15 年后,[的燃烧平台](http://blogs.wsj.com/tech-europe/2011/02/09/full-text-nokia-ceo-stephen-elops-burning-platform-memo/)是如何运作的。



Motorola MPx200 launched with AT&T in September 15, 2003



*   企业、消费者、企业、消费者:Windows Mobile 未能成功的最大驱动力是其关键资产:微软。2000 年，微软从 Psion(Symbian 的制造商)聘请了 Juha Christensen，在我加入团队时，他的号召是让智能手机之战成为诺基亚/Symbian 与微软之间的“两匹马的比赛”。默认情况下，这种策略意味着大量的电话播放，因此是以消费者为中心的产品。这意味着把微软的消费者资产(MSN Messenger，internet explorer，XBox)放在最前面和最中心的位置。但 Windows Mobile 是微软内部的一个小企业，摇钱树仍然是 Office 和服务器产品。RIM 的 Blackberry Enterprise Server 减缓了 Exchange 新版本的升级速度，并下令扼杀 RIM(呃，抱歉，在 DOJ 事件之后，微软员工不再被允许说“扼杀”……“RIM-compete”是 DOJ 批准的正确的战略名称)。因此，这个小部门的重点转向了以前景和生产力为核心的以消费者/企业为中心的设备的更狭窄的出货量。在未来几年，消费者/企业的转变还会发生几次。2003 年夏天，朱哈[离开了](http://www.zdnet.com/article/microsoft-mobile-phone-exec-to-depart/)，两马竞争从诺基亚 vs 微软(消费者)变成了 RIM vs 微软(企业)。
*   消费者推动了购买决策:Windows 标志卖出了个人电脑。因此，windows 标志(以及它所暗示的所有熟悉度)*肯定*也能销售手机……但事实证明，消费者并不在乎手机运行的是什么操作系统。消费者只想要他们性感的 RAZR 翻盖手机，或者 800 万像素的诺基亚照相手机，或者低价的 LG 手机。在那个时候，它仍然只是一部电话，主要功能仍然是打电话(一些早期的 Windows Mobile 设备并不总是那么擅长……是的，有一种方法可以让 WinMo 设备 CTRL-ALT+Del)。
*   **单手对双手，键盘对触摸屏**:pocket PC 掌上电脑是双手设备，使用手写笔作为输入。智能手机的愿景一直是成为单手设备。它实际上是一个很好的收件箱分类工具……但是市场反馈令人困惑。PocketPC 计划逐步淘汰，专注于智能手机，但双手输入作为消费者的选择仍在推动销售。由于 iPhone 和 Android 设备将很快被证明，如果它提供更丰富的功能和更大的屏幕，消费者愿意接受双手输入。微软将 RIM 竞争视为一项法令，专注于键盘而非触摸屏。
*   商业模式向后看而不是向前看:微软通过向原始设备制造商出售 Windows 许可赚了大钱。这是科技界的可口可乐糖浆。同样的模式当然也适用于手机:向原始设备制造商收取 Windows 许可费，让他们管理台湾的工厂。但在移动世界，权力动态和客户期望是不同的。运营商仍在为消费者补贴手机，权力更多地掌握在运营商而非 OEM(或操作系统制造商)手中。10 美元的 Windows Mobile 许可证对材料清单(BOM)产生了巨大的影响，因此也影响了手机的批发价。正如 WinMo 集团的新财务主管曾经对我说的“*,我们想错了。我们需要从一次性的许可费支付转向来自服务的经常性收入*“…如果他的声音被听到就好了【小米正在[使用](http://www.wired.com/2015/02/may-never-use-xiaomis-phones-theyll-change-life-anyway/)同样的策略积极地为手机定价，但专注于服务的长期价值】。苹果颠覆了旧模式，推出了一款消费者垂涎欲滴、愿意花 500 美元购买的设备(无补贴！).Android 的改变更大，它让一个比 WinMo 更丰富、更开放的操作系统完全免费。旧的微软商业模式从未革新过。其他人围绕微软进行创新。
*   **“网络”不是“网络”**:WinMo 棺材里的最后一颗钉子是在 2007 年和 2008 年，当时苹果推出了 iPhone，谷歌推出了 Android。这不是关于 iPhone 的设计，也不是安迪·鲁宾为安卓 G1 打造的令人惊叹的铰链，而是更哲学地关于消费者希望他们的移动设备上有什么样的网络。微软、RIM 和诺基亚都想出了将网络压缩并重新格式化成更小屏幕的方法。这些手机和操作系统制造商似乎认为他们有权决定网络在移动设备上应该是什么样子。Android 的愿景一直是在移动设备上拥有完整的 HTML 网页体验(非常谷歌化)，iPhone 和 Android 平台都推出了 [webkit 浏览器](https://en.wikipedia.org/wiki/WebKit)和完整的 HTML 支持。消费者用拇指投票……他们希望“网络”是网络。[注意:移动网络的概念是几年后我将负责巩固谷歌、诺基亚和 RIM 之间伙伴关系的核心问题，但那是未来文章的内容。简而言之:谷歌是对的。]

所以事后看来，比尔·盖茨在 1999 年对移动技术的破坏力所做的预测是正确的。微软在移动操作系统上的赌注很有可能成功。但是在其巅峰时期，微软从未超过移动操作系统市场的 15%,正如 Benedict Evans 指出的那样，现在很明显谷歌和苹果赢得了这场移动战争。



多年前，我身处战争初期的第一线，有时间反思我在微软内部以及从外部看到的微软的各种错误，并得出结论，这不是一个错误，而是许多错误夺走了一个奖项，想想 2000-2005 年的情况，这个奖项本应属于微软…

在这篇文章的结尾，如果没有一个臭名昭著的视频链接，鲍尔默在 iPhone 发布时嘲笑它，那将是不诚实的……“它对商业客户没有吸引力，因为它没有键盘……”“现在我们每年卖出数百万部手机，而苹果却零销量……”“我喜欢我们的策略……我非常喜欢！”花了几年时间和几十亿美元买下诺基亚，后来却不确定这有多好笑。这个[反思](https://www.youtube.com/watch?v=v9d3wp2sGPI)2014 年的查理·罗斯访谈也值得一看。
