# 有没有想过为什么最受欢迎的应用程序开始变得千篇一律？这可能是一件好事

## 现在设计师可以花更多的时间关注真正重要的东西。



Copyright MSTQ, Inc.



打开 Instagram、 [Airbnb](https://www.inc.com/yazin-akkawi/the-surprising-technique-airbnb-uses-to-better-sell-an-experience.html) 、Apple Music、Twitter、Dropbox 或 Lyft，乍一看，你可能会觉得很难区分哪个应用程序是哪个应用程序:大而粗的标题，采用圆形无衬线字体；极简的黑白界面，有大量的负空间；几乎没有颜色。

现代的界面风格让人感觉如此简单。具有讽刺意味的是，许多最受欢迎的应用程序让人感觉它们实际上根本就不是设计出来的。

Instagram 深蓝色页眉怎么了？Lyft 的 app 里的亮粉色去哪里了？最大的科技品牌忘记了他们的品牌了吗？或者他们只是共享相同的模板？

更重要的是，有明确的信号表明谷歌也在加入无色趋势。今年，谷歌推出了对其材料设计指南的重大修改，并发布了重新设计的 Gmail 预览版。

你猜怎么着？看起来和其他人一模一样。它的大多数标志性的红色 UI 元素已经完全消失了(比如红色的顶栏),我们得到了更多的白色:白色的底栏、白色的浮动动作按钮、白色的输入框和白色的背景。

我们都知道硅谷有互相窃取创意的倾向，但至少在过去十年里，设计一直是差异化的关键。这种设计一致性的趋势似乎非常违反直觉。



An updated version of Michael Horton’s [Complexion Reduction: A New Trend In Mobile Design](https://medium.com/amazing-stuff/complexion-reduction-a-new-trend-in-mobile-design-cef033a0b978)



那么到底是怎么回事呢？

嗯，我认为产品设计的一致性是一件好事。这里有几个原因。

# 更加注重内容和结果

App 疲劳是真事。大多数人已经厌倦了在太多应用程序之间来回切换，或者在每次新下载后学习如何使用新界面。

近四分之一的下载应用程序在使用一次后被删除。研究表明，大多数人只有几个他们经常使用的应用程序——节省时间或金钱(或两者兼而有之)并让生活更轻松的应用程序。随着日常数字产品的一致性，设计师可以更加专注于这一点。

正如因创造了“UX”一词而闻名的唐·诺曼(Don Norman)所说，“界面的真正问题在于它是一个界面。接口碍事。我不想把精力放在界面上。我想专注于工作。”

# 更高的一致性意味着更好的可用性

你上一次尝试在网上购物，却没有立即浏览屏幕右上角进入购物车是什么时候？几乎所有电子商务网站的标准化设计惯例意味着人们不需要重新学习如何在每个在线商店购物。

大多数应用程序上的任何约定都将被铭记在大多数人的大脑中，不幸的是，偏差会导致困惑和沮丧。

以 Snapchat 为例，它在今年早些时候发布了一个有争议的设计更新。这种非传统的设计——包括混淆功能和改变导航模式——导致[用户情绪几乎立即下降了 73%](http://today.yougov.com/topics/technology/articles-reports/2018/05/10/consumer-sentiment-drops-snapchat-redesign),该应用的用户数量和股价也大幅下跌。

正如 Snapchat 所证明的那样，设计新颖的东西是有风险的，但一致的设计(即使是在竞争对手的应用中)可以降低让用户不安的可能性。

# 品牌不仅仅是看起来如何，而是如何运作

将用户的需求和意图与一个解决方案相协调是伟大体验的来源。差异化的*体验*是无形的，对一个品牌来说，可以说比差异化的外观和感觉更有价值。

诚然，风格是重要的，它也可以在体验中发挥很大的作用。当人们发现一个令人困惑的界面在视觉上很吸引人时，他们会更加宽容和原谅它。

但是设计师的精力应该花在创造人们完成任务的最佳方式上，而不是修饰调色板或戏剧性的动画。设计好一些东西，一些对最终用户有价值的东西，需要一个看似无限的测试和迭代设计的循环，以更接近有用和可用的东西。

巴拉克·奥巴马、史蒂夫·乔布斯、马克·扎克伯格；他们都因每天穿同样的服装而出名。原因是心理学家所说的决策疲劳。他们每天必须用有限的精力做大量的决定，限制决定的细节，比如穿什么，为真正的东西保留精力。

设计师应该以同样的方式思考他们的能量。随着界面设计趋向整合，设计师可以少花时间担心按钮应该是什么颜色或多大，而更多地考虑为什么它应该在那里。

# 界面设计的雷同不会扼杀创造力

现在，我知道你在想什么——是的，你最喜欢的应用程序看起来都一样的缺点是创造力和创新会减少。老实说，我认为这是真的。

但是，即使每个产品设计师都加入了精简界面的运动，仍然有大量有意义的设计工作要做。

事实上，事实是，设计上的雷同在短期内是有益的(希望我已经说明了原因)，但它们对长期有更大的影响。

首先，增强现实和人工智能的兴起，我们与计算机的互动可能会变得更加无形；屏幕正在缩小，手势变得越来越熟悉，我们现在比以往任何时候都更多地使用无屏幕界面交谈或聊天。

即使在我的 iPhone 上，由于它漂亮的屏幕和交互设计，我通常会选择 Siri 来完成设置提醒、创建日历事件、选择音乐、发送信息和查看天气等任务。我可能可以一整天不使用屏幕而不影响我的 iPhone 提供的任何功能。或者，正如诺曼所说，我可以“完成任务”。

但是，对于设计师来说，改进 Siri 的设计还有很多工作要做。我不是唯一一个这样认为的 UX 集体作家。凯文·尤金有一篇关于如何重新设计 Siri 体验的[整篇文章](/redesigning-siri-and-adding-multitasking-features-to-ios-70c2f1a1569b)。

也就是说，将会有更多的机会来考虑以新的、创造性的方式来吸引感官(视觉之外)、注意力、情感或感知。

将我们认为是好的设计局限于我们在屏幕上看到的东西，至少破坏了我们对未来梦想的可能性。
