我从一名工程师开始我的职业生涯，并很快进入工程管理领域。在泡沫时期，我可能雇佣了一百多名工程师。我学到了很多关于招聘的知识，主要是通过犯错。当我过渡到产品管理时，我能够运用我在雇佣技术人员方面的一些经验，但我也学到了一整套新的经验。上周，一个朋友打电话来说他们需要招聘一名产品经理，想听听我的建议。我意识到没有太多关于采访项目经理的好信息(一般来说，没有太多关于产品管理的好信息)。更重要的是，不管你在什么样的环境中——初创公司[还是大公司](/newsletter/advice-for-early-career-product-managers.html)，对产品经理都没有太多的要求。所以我想我应该把我学到的东西集中起来。

* * *

### 记住朋友，没人让你出现

产品管理可能是一项组织没有也能做得很好的工作(至少在一段时间内)。没有工程师，什么也建不成。没有销售人员，什么也卖不出去。没有设计师，产品看起来就像垃圾。但是在一个没有经前综合症的世界里，每个人都只是简单地填补空白，继续他们的生活。记住这一点很重要——作为项目经理，你是可以牺牲的。现在，从长远来看，伟大的产品管理通常会决定成败，但你必须证明这一点。产品管理也结合了许多其他专业的元素——工程、设计、营销、销售、业务开发。产品管理是一门奇怪的学科，充满了古怪的人和被拒绝的人，在其他任何地方都不适合。就我而言，我热爱工程的技术挑战，但鄙视编码。我喜欢解决问题，但我讨厌别人告诉我该做什么。我想成为战略决策的一部分，我想让*拥有*产品。营销吸引了我的创造力，但我知道我不喜欢离技术太远。工程师们尊重我，但他们知道我心不在焉，并且普遍认为我太“市场化”像我这样的人自然倾向于产品管理。

### 1.雇佣所有聪明的人

那么，我在 PM 中需要什么呢？最重要的是，原始的智力马力。我会选择一个非常聪明、没有经验的首相，而不是一个智力一般、经验丰富的首相。产品管理从根本上来说就是站在你的立场上思考，比你的竞争对手领先一步，并且能够把你自己投射到你的同事和客户的头脑中。我通常会问面试者一系列分析性问题，以评估他们的智力和解决问题的能力。通常我会问问题，直到我确定候选人比我聪明。出于某种原因，我认识的很多人都不愿意这么做。他们认为这是对候选人的侮辱。我认为合适的候选人会喜欢这个挑战。事实上，这是第一个测试——当我说“我想提出一些理论问题，可以吗？”时，他们会有什么反应这群人中的佼佼者通常会兴奋地从椅子上跳起来。超级聪明的人有时会用自己的问题来反驳。

***2022 年更新:**请不要问“脑筋急转弯”或逻辑谜题。自从我最初写这篇文章以来，我对分析性采访的观点，就像这个行业的许多观点一样，已经发生了变化。这些问题充其量也就是一文不值。最糟糕的是，它们是歧视性的。大多数优秀的产品公司已经转向结构化面试(T4)，强调没有“正确答案”的工作样本和情境问题请参见[我的回顾展](/essays/happy-10th-birthday-to-hthapm.html)了解更多。*

### 2.强大的技术背景

我认识的一些经理坚持只雇佣拥有计算机科学学位的项目经理。我没有那么势利——也许这是我自己的文科本科教育——但我确实倾向于喜欢那些担任过技术职务的人。扎实的工程背景给了项目经理两个重要的工具— [与工程师沟通的能力](/essays/how-to-work-with-software-engineers.html)以及对驱动产品的技术细节的掌握。当然，这取决于产品——从事底层开发 API 的项目经理肯定比从事个人网站前端工作的项目经理需要更多的技术知识。但是基本原则是适用的——[有技术背景的产品经理](/newsletter/product-manager-technical-enough.html)将更成功地向工程师传达产品需求，并向非技术同事和客户传达复杂的细节。也就是说，有些陷阱你需要避免。最重要的是，一个前工程师的项目经理需要意识到他或她仅仅是一个*前*工程师。来自工程的项目经理仍然试图负责技术决策和实现细节，他们将会崩溃。出于这个原因，我喜欢雇佣那些在之前的工作中已经转向产品管理的技术人员。他们已经经历了具有挑战性的适应期，通过查阅参考资料，你可以感受到他们进化得有多好。我不会用面试问题来烦你评估技术能力。它们取决于技能组合，有数百个网站提供招聘工程师的好建议。相反，这里有一些很好的产品经理面试问题，用于评估技术项目经理适应这个角色的程度以及他们与工程师合作的能力:

*   你为什么决定从工程转向产品管理？
*   有技术背景最大的优势是什么？
*   最大的*劣势*是什么？
*   当你从工程转向产品管理时，你学到的最大的一课是什么？
*   你希望当你是一名工程师时知道些什么？
*   你如何[赢得工程团队](/essays/how-to-work-with-software-engineers.html)的期望？

### 3.“蜘蛛感应”产品本能和创造力

下一个类别非常主观，难以评估，而且非常重要。我坚信某些人生来就有天生的产品本能。这些人只知道什么是伟大的产品。他们并不总是对的，但他们的直觉通常指向正确的方向。他们往往是某一观点的热情拥护者，有时会让他们的同事懊恼不已。我有幸和很多这样的人一起工作，这是产品经理的一个基本特质。而且可以调，但是学不来。产品管理，尤其是在像网络这样高度动态的环境中，涉及许多小的决策。当然，这里有很多宏大的想法和战略。但正是这些小决定让一个伟大的首相与一个体面的首相拉开了距离。你知道，当他们建议团队中没有人想到的方法时，他们有“蜘蛛感应”产品本能，但当他们听到这些方法时，每个人都会立即感到很明显。在面试中评估产品本能是最具挑战性的。但这是可以做到的。我经常做的一件事是检查候选人在一个小时的面试中是否完成了以下任务:

*   **独立回应我自己对我的产品的一些担忧** -如果你是一个优秀的项目经理，你会有一大堆让你担心自己产品的事情。也许它们是 UI 缺点、缺失的特性，或者需要解决的架构缺陷。你知道这些事情需要解决。对于一个具有强烈产品直觉的聪明的局外人来说，至少其中一些应该是显而易见的。我期待着面试中的那个时刻，我微笑着，点点头，说“是的，我知道——那也让我们疯狂。”
*   **让我对我的产品有了新的认识** -这可能是我从未考虑过的明显改进，针对竞争对手的新想法，或者他们遇到的需要解决的问题。当我从一个候选人身上学到一些东西时，我知道两件事:(1)他们不害怕批评，(2)他们可能比我聪明。我希望产品经理两者兼备。
*   让我对一些新的有趣的东西产生了兴趣——具有优秀产品直觉的人往往比其他人更先注意到优秀的产品。如果我在面试一个顶尖的候选人，我通常会在离开时发现[一些新的和创新的东西](/essays/10x-not-10-percent.html)。

以下是一些很好的判断产品直觉的面试问题:

*   说说你最近遇到的一个很棒的产品。为什么喜欢？【顺便说一句，当候选人在面试中提到我的一个产品时，我都快疯了。我很难在雅虎雇佣任何人。谁告诉我他们最近遇到的最酷的产品是雅虎！好悲伤。]
*   是什么让[在此插入产品]取得成功？[我通常选择受欢迎的产品，比如 iPod 或易贝，它们在拥挤的市场中轻而易举地赢得了消费者。]
*   您不喜欢我的产品的哪些方面？你会如何改进它？
*   一年后我们会遇到什么问题？两年？十年？
*   你怎么知道一个产品是设计好的？
*   你有过的最好的想法之一是什么？
*   最糟糕的是什么？
*   你怎么知道什么时候走捷径把产品推出去？
*   你从用户界面设计中学到了什么？
*   你如何决定建造什么*而不是*？
*   你最大的产品错误是什么？
*   你觉得产品管理的哪些方面最没意思，为什么？
*   你认为自己有创造力吗？

***2022 年更新:**参见[我的回顾](/essays/happy-10th-birthday-to-hthapm.html)更多关于“蜘蛛感应”是什么产品的意思。自从我最初写这篇文章以来，产品设计师已经上升为与项目经理和工程师并列的“凳子的第三条腿”。评估设计敏感性和产品本能的一个方法是让产品设计师参与面试环节。*

### 4.赢得的领导力

产品经理通常是他们组织中的领导者。但是他们通常对其他人没有直接的指挥权。这意味着他们赢得了自己的权威，并通过影响力来领导。[领导力和人际交往能力对产品管理至关重要](/essays/please-make-yourself-uncomfortable.html)。有一千本关于领导力的书，所以我不会把这篇文章变成一篇关于这个主题的论文(反正大多数书都是废话)。我发现证明人调查是衡量领导技能的最有效的方法，尤其是涉及与候选人共事但不隶属于候选人的同事和个人贡献者的证明人。但这里有一些我过去用过的面试问题:

*   共识总是一件好事吗？
*   管理和领导有什么区别？
*   你喜欢和什么样的人一起工作？
*   你发现什么类型的人很难共事？
*   告诉我一个团队不团结的时候。你认为为什么会这样，你学到了什么？
*   你如何让一个团队致力于一个时间表？
*   有人会做什么让你失去信心？
*   你对不同职能部门的人进行不同的管理吗？如果有，如何实现？
*   关于说不，你学到了什么？
*   谁对运输产品负有最终责任？
*   你有没有遇到过这样的情况，你的团队让你失望，你不得不承担责任？
*   这些年来，你对错误的容忍度有什么变化？
*   你喜欢哪个，好消息还是坏消息？
*   你的招聘方法是什么？

### 5.引导多个观点的能力

做产品经理需要身兼数职。我经常开玩笑说，很多时候你的工作是为那些目前不在房间里的人辩护——客户、工程、销售、高管、营销。这意味着你需要有能力做其他人的工作，但足够聪明知道不要做。伟大的项目经理知道如何引导不同的观点。他们经常唱反调。他们往往对简单的答案不满意。在一次谈话中，他们可能会告诉你这些要求在技术上似乎不可行，然后询问这些对销售人员来说有什么意义。有一个显而易见的方法可以评估候选人从多个角度思考问题的能力——让很多人参与面试。我总是坚持认为，至少，来自工程、设计和营销的代表要会见一个潜在的项目经理候选人。根据具体的角色，这个列表可能会增加—售前工程、支持、开发人员关系、业务开发、法律或客户本身。最终，任何将和这个人一起工作的人都应该见见他们。注意，我没有说*每个人*都需要满足他们。每个关键功能的一个精心选择的代表就足够了。这也不意味着每个人都要竖起大拇指——随着面试官名单的增加，在面试过程中很难达成共识，所以要适当考虑反馈。但是没有人能够像销售人员一样判断产品经理对销售过程的理解程度。我还强烈建议你给面试官一些具体的指示，比如“我想让你看看这个人对你在渠道开发中所面临的问题有多了解，以及他们在这个领域对你有多支持。”以下是我使用的一些具体的产品经理面试问题(这些只是例子，可以随意替换功能名称):

*   你是如何学会与销售打交道的？
*   与客户互动的最佳方式是什么？
*   营销的动力是什么？
*   你如何知道设计何时步入正轨？
*   产品经理应该如何支持业务发展？
*   关于向上管理，你学到了什么？
*   与高管合作的最佳方式是什么？

### 6.给我一个运过东西的人

最后一个特征可能是最容易评估的。除非这个职位非常初级，否则我通常会雇佣那些实际生产过产品的产品经理。我的意思是从开始到结束，从概念到发布。没有什么比以前做过更能表明一个人有能力运送伟大的产品了。过去的表现*是*未来成功的标志。更好的是，它在无形的海洋中提供了一些有形的东西来评估。当检查参考资料时，我总是确保与以前项目的重要同事交谈，特别是项目经理和他们的工程和销售或营销同事。(顺便提一下，这些规则是有原因的，正如我在第一条中提到的，我仍然会选择非常智能的 PM，而不是有调光经验的 PM，即使前者以前没有发货)。

*这是我最初的序言:*

我在创业公司招聘已经有一段时间了，在创业公司招聘和在大公司招聘有很大的不同。在雅虎！搜索，好像我们一直在招人。我平均每周接受 5-8 次采访。这是一场永不停息的简历、面试和录取通知书的宣传。我并不总是招聘经理。我在那里的时候只雇佣了少数几个产品经理。但是有人总是在招聘产品经理，而我通常在面试团队中。在大公司，你首先注意到的是专业化程度。在初创公司，每个人都会做一点，所以你需要强大的多面手。更重要的是，未来很难预测，所以你需要能适应的人。你可能认为你是在雇人做某件特定的事情，但是几个月后事情可能会发生变化。在大公司不是这样。通常当你在招聘的时候，你心里会有一个非常具体的角色，而这个职责发生变化的可能性很小。雅虎雇佣了很多人。这可能不适合初创公司。我记得很多面试后的对话都是这样的——“嗯，我不确定他们是不是完美的候选人，但他们看起来确实适合这个非常特殊的角色，所以让我们雇用他们吧。”这在大公司可能行得通，但在初创公司却是致命的想法。
