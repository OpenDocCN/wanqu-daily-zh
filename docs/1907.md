*下图:克里斯·万斯特拉斯、首席执行官和 [GitHub 联合创始人](http://github.com/)的问答。这是我们的[“自举、盈利、&骄傲”](http://www.google.com/search?q=bootstrapped+profitable+proud&sitesearch=37signals.com)系列的一部分，该系列介绍了收入超过 100 万美元、未接受风险投资且盈利的公司。GitHub 的 Chris 和 Tom 也在这篇文章的评论部分回答了读者的问题。*

你的公司是做什么的？
我们使用 git 或 Subversion 为公司和开源项目提供公共和私有源代码托管。我们想做的是降低为公共和私人项目做贡献的门槛。向开源项目提交补丁应该是关于代码，而不是提交补丁的过程。与你的同事一起工作，无论是在同一个办公室还是在世界各地，都应该是关于推进你的项目，而不是管理笨拙的工具。

我们还提供 git 培训，提供 git 学习材料，并赞助开源项目。

你如何向“正常”的人解释(例如，你的妈妈，你在鸡尾酒会上遇到的人，等等)。)你们公司是做什么的？
GitHub 就像程序员的维基百科。您可以编辑文件，查看谁修改了什么，查看文件的旧版本，并从世界上的任何地方访问它——除非您使用的是源代码而不是百科全书数据。公司用它来构建软件和网站，而业余程序员用它来寻找和分享项目。

商业模式很简单:如果你想和所有人分享你的源代码并公开，你不需要支付任何费用。如果你需要隐藏你的源代码，因为它是私有的，并且运行你的业务，你需要付费。

培训在企业中占多大比重？
培训并不是我们收入的很大一部分，但是如果我们可以通过教人们如何更好地使用 git 来赚钱，那么大家都是赢家。亲自与客户见面并与他们建立真正的关系也很棒。此外，我们喜欢把斯科特·沙孔(我们的常驻 git 向导)派到世界各地。

赞助开源和 git 学习材料对整个世界都有好处，但我们这样做是因为我们热爱它。如果一个 GitHubber 想要花一些时间在一个开源项目上，这让我们非常高兴。

生意是如何开始的？
起初 GitHub 是一个周末项目。Tom Preston-Werner 和我在一个体育酒吧闲逛，在一次当地的编程聚会后，他告诉我他对 git 托管网站的想法。这将是一个轻松分享代码和了解 git(git hub)的地方。这更多是出于需要:我们都喜欢 git，但是没有可以接受的方式与他人共享代码。汤姆认为我会有兴趣帮助解决这个问题，我也是。

我们开始在周六开会，一点一点地建设网站。我们一起吃早午餐，谈论我们对网站的想法，然后开始工作。汤姆会设计页面或功能，我会实现它们。基础工作就绪后，我们开始在日常工作中使用 GitHub，这是我和 PJ·海特共同创办的一家初创公司。这是改进网站的一个很好的方式，因为 PJ 和我每天都在使用 GitHub，并真正感受到了什么是缺失的，什么是有效的。

汤姆在他之前的公司 Gravatar 学到的一件事是，免费提供资源密集型服务是一个失败的提议。在这种情况下，它是高流量图像托管，但在 GitHub 的情况下，它是 git 托管。存储和传输代码会给我们带来一大笔服务器费用。我们需要一种收回成本的方法。

考虑到这一点，我们为我们的朋友推出了免费的公测版。该网站立即开始起飞。你可以免费创建公共或私人的存储库，人们开始使用这个网站来存储他们的业务代码——考虑到 PJ 和我也在做同样的事情，这并不奇怪。然而，很快就有人给我们发电子邮件，询问他们如何支付私人储存库的费用。

在这一点上，我们意识到 GitHub 可能不仅仅是收回成本。这可能是真正的生意。我们决定继续免费提供无限的公共存储库，但我们会对私人存储库收费。换句话说，我们会向要求付费的人收费。

PJ 成为 GitHub 的联合创始人，我们停止了创业。GitHub 现在是我们的创业公司。我们三个人在 2008 年 4 月 10 日正式启动了这个网站，从那以后一直在运营这个网站。

你需要多少现金来启动和运行？你怎么拿到那笔钱的？
这一切都始于一个域名、Slicehost 的一份廉价切片和一些股票艺术品。当我们决定开始一项真正的业务时，我们有建立一个法律实体的通常费用——我们一起筹集的几百美元。

真正的成本是支持我们的网站和业务增长。PJ 和我做咨询来支付账单，而汤姆做全职工作。随着业务的增长，我们想出了一个能让我们逐渐达到全职工资的系统。

我们会从小额工资开始，然后每个月，假设我们在那个月达到了一定的货币目标，我们会增加工资。最终，如果一切顺利，我们将获得全职薪水。

效果很好。几个月来，我们未能实现我们设定的目标，结果，我们的薪水没有增加。但是我们坚持了下来，最终拿到了我们计划的薪水。

你的生意有多成功？
对我来说，最激动人心的事情是，我们能够说服优秀的人和我们一起工作，并给他们丰厚的报酬，而且不需要任何外部资金。从这个角度来看，我们非常成功。

就数字而言，我们有几十万用户、几万付费用户和近一百万个存储库——每天还会增加几千个。这一切都发生在两年多一点的时间里。

你的工作环境是什么样的？
混乱，但我们喜欢这样。我们没有经理，而是由我们作为一个公司来决定哪些功能或想法是优先考虑的。谁对某个特性最感兴趣，谁就会实现并拥有它。

这听起来可能很疯狂，但它确实很有效。这是确保人们对他们正在做的事情感兴趣的好方法，而且作为一家公司，我们正在做有意义的事情。如果它是蹩脚的，没人会做它。我们都在使用这个网站，所以每个人都很清楚缺少什么或者什么没有用。我们也是许多客户的朋友，这有助于决定什么是优先事项，什么不是。

在文化上，我们像一个分布式公司一样运作。我们在旧金山市中心有一个办公室，但很少有人同时在同一个地方办公。真正的办公室，每个人每天出现的地方，是我们的营火房。起初这是出于需要——我们租不起办公室，所以我们在家里或咖啡店工作，并在网上保持联系。但是现在我们相信这是一种更好的操作方式。GitHubbers 可以自由地在某一天走进办公室一起工作，然后跳上飞机，第二天在世界各地继续一起工作。没有办公时间，人们只是专注于完成工作，而不是打卡上下班。

我们都想把事情做好。我们非常幸运地(主要)在一个网络应用上工作，这意味着快速做出改变是非常容易的。我们认识到，现在发布它，以后再修复它，一旦你能看到人们是如何使用它的，这比让它永远停留在开发阶段要好得多。直接发货吧。

![](../Images/7cc6abff86886d6b1de310886b476970.png)

**为什么现在发货，以后修复会好很多？想起什么例子了吗？**
你不可能总是对的，没有什么是完美的——接受这一点是一个巨大的竞争优势。尽早发布可以让你看到人们实际上是如何使用你的网站的，并让你做出相应的反应。你搁置的那个功能有关系吗？是一个你没有想到的非常需要的特性。有人打过你担心的那个 bug 吗？太靠近某物很容易变得有点近视。

分清轻重缓急并不容易，你也没有无限的时间，所以让你的客户来帮忙吧。看看他们想要什么，只要有用就尽快发货。

优先考虑运输也更有趣。截止日期感觉像工作，运输感觉像挑战。

我参与的最近一次发布是我们的新组织功能。一旦我们有了一些有用的东西，我们就邀请几个朋友进行私人测试。观察他们如何使用新系统帮助我们完善了一些工作流程，增加了一些急需的功能，并消除了令人讨厌的错误。

你在公司的目标是什么？
五年后，我想热爱我的工作，热爱和我一起工作的人。我们确实想保持增长，赚更多的钱，雇佣更多的人，做出顾客喜欢的东西，但最重要的是我们继续享受乐趣。我希望 GitHub 永远是一个令人敬畏的工作场所和一个令人敬畏的网站。

只要我们有真正优秀的员工，他们热爱自己的工作，享受自己的工作，我们就会继续为我们的客户(和我们自己)提供真正优秀的服务。

你无视他人的建议/意见，一意孤行的例子有哪些？
汤姆有[一篇关于拒绝 30 万美元去 GitHub](http://tom.preston-werner.com/2008/10/18/how-i-turned-down-300k.html) 工作的很棒的文章。

> 我将比预期的更早面临选择。我要么签约成为微软的员工，要么辞职去做 GitHub 的全职员工……最后，就像印第安纳·琼斯永远不会拒绝寻找圣杯的机会一样，我也不会放弃为自己真正热爱的事情工作的机会，不管这种选择有多安全。当我老了，快要死的时候，我计划回顾我的生活，说“哇，那真是一次冒险”，而不是“哇，我确实感到安全。”

通常我们会忽视他人的建议/意见。GitHub 是令人信服的论点的公司——每个决定都需要有自己的价值。在讨论中，某个成功(或不成功)的人曾经尝试过某件事可能很重要，但更重要的是这个想法本身如何适用于这种情况。

很多人告诉我们放弃我们的 git 训练(“它没有伸缩性！”)或者不提供网站的本地版本(“SourceForge 做不到！”)，但这两款产品都在为我们和我们的客户工作。每个公司都是不同的，我们喜欢保持开放的心态，倾听我们的客户，也倾听我们自己。

作为一家公司，你必须克服的最大挑战是什么？
在这家公司的第一年左右，让我想起了许多自我发现的尴尬的青少年时期。GitHub 公司是从这个副业项目中发展起来的，所以我们从来没有什么大的愿景、梦想或抱负。我们只是想做些酷的东西。我很想说这就是你所需要的一切，但我们已经知道还有更多:你需要有一个愿景和一种哲学。每个人(至少是所有的创始人)都需要保持一致。难的是找到那一页。

我们是做 web 应用，还是只做源码控制？我们付给员工什么？我们应该在会议上发言吗？我们如何获得客户支持？所有这些看起来像是孤立的问题，但它们都与公司的哲学直接相关。一旦你知道公司代表什么，回答这些问题就容易多了。在我们真正搞清楚 GitHub 公司是怎么回事之前，我们很难单独回答每个问题。

你把这种哲学写在什么地方了吗？还是在那里工作的人都只是知道它是什么有机的？
我们在招聘过程中会谈到这个问题，我们对此非常重视。我们希望任何潜在的 GitHubber 知道他们正在进入什么，并确保它是一个很好的适合。其中一部分是吃晚餐，谈论文化、哲学、我们犯过的错误、计划等等。

早期，我们根据他们的技能雇用了一些人，很少考虑他们如何融入公司文化，或者他们是否理解公司的理念。很自然，这些雇佣没有成功。因此，虽然我们关心潜在员工的技能，但他们是否“理解”我们也是一个重要因素。

你的故事还有什么有趣的地方？
GitHub 三位创始人中有两位是大学辍学生。

你认为这是有原因的，还是仅仅是巧合？
我不认为这是巧合——汤姆和我都辍学去找编程工作。他在一家初创公司工作，我在一家更传统的公司工作，但我们都离开了，因为我们迫不及待地想进入现实世界。不可避免的是，我们最终会开始自己的事业。

PJ 获得了计算机科学学位，但他拿到毕业证书的那一刻，基本上已经在飞往旧金山的飞机上了。毕业前几个月，他在 CNET 找到了一份工作(我和他是在那里认识的),在他整个大学生涯中，他一直在做副业和生意。

我个人对你是否应该获得大学学位没有意见，因为我在两边都遇到过迷人和成功的人，但我在学校表现不佳的原因与我在公司结构中表现不佳并创办了自己的公司的原因是一样的。我想很多人会更乐意为自己工作，但从来没有意识到这一点。你的工作不一定很糟糕。

你对考虑创业的人有什么建议？
做吧，但要聪明。动动脑子。想想什么对你最有意义。担心你在盈利之前(甚至之后)花掉的每一块钱。专注于重要的事情，不要浪费时间无休止地重新设计你的网站或修补新技术。以后会有足够的时间做那件事。
