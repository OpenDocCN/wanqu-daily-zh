再说影响力。作为一名工程师，你是如何获得影响力的？影响力看起来像什么，它植根于什么，你如何运用它或失去它？它与你作为经理可能拥有的权力和影响力有什么不同？[0]

这种情况经常出现在 IC 中，他们迫切地想成为经理，以便有更多的**获取信息**和**影响** **决策**。这是一个不好的信号，尽管可悲的是它非常普遍。

当这种情况发生时，你需要做一些自我反省。您的组织是否为高级 IC 留出了领导和拥有决策的空间？您是否有一条与经理路线平行的 IC 路线，至少与董事级别的经理路线平行？他们得到平等的补偿了吗？你![](../Images/7b3a2e57e9e2d821b3260ed5a2df4ed5.png)

有职业阶梯吗？对于不是经理的人来说，你的决策过程是不是很神秘？不要认为对你来说显而易见的事情对其他人来说也是显而易见的；你得四处打听。

如果是这样，那很可能是他们自己的私人包袱在说话。也许他们不相信你。也许他们只在管理者拥有所有权力的组织中工作过。也许他们甚至在很多地方工作过，说了和你说的完全一样的话，关于集成电路如何产生巨大影响，但这都是谎言，现在他们被烧毁了。可能因为各种原因，他们不习惯感觉强大。

不管怎样，那些想成为经理以延续糟糕的权力结构的人是你最不想成为经理的人。[1]

但是工程影响力是什么样的*？你的能力是如何体现的？*

我将避免讨论性别、种族和阶级的重叠和相互关联的问题，让我们承认，对一些人来说，行使权力比其他人更困难，好吗？

### 创造的力量

做是工程的超级力量。我们只用一台笔记本电脑和我们的大脑来创造东西！太不可思议了！我们不需要不断地说服、哄骗和胁迫别人来代表我们建设，我们只需要*建设*就可以了。

这看起来很基本，但是很重要。创造是我们所有形式力量的源泉。除非我们同意建造，否则什么都不会建造(这也是一个道德问题)。

脸书有一张海报，上面写着“代码赢得争论”。在很多方面都有问题，当然。但是你见过多少次技术纠纷是由愿意做这项工作的![](../Images/5518408e8b66c1144c817ffb6de72d83.png)

来解决的呢？或者用一种方式“解决”..然后通过做逆转？做结束辩论。实践证明理论。做是强大的。(而且“做”不仅仅是指“写代码”。)

此外，构建软件是一种创造性的活动，大规模地进行是一种强烈的集体活动。作为一种创造性的行为，当我们对工作充满动力、灵感和热情时，我们会成为更好的建设者(与劈柴相比)。作为一种合作行为，当我们拥有高度信任和社会凝聚力时，我们会做得更好。

工程能力和判断力、自主性和目标感、社会信任和合作行为:这是伟大工程的原材料。每个人都有一两种他们觉得最舒适和最权威的操作模式:我们可以将它们粗略地归类为原型。

(这些例子来自我多年来共事过的一些非常棒的高级工程师，以及我作为工程师喜欢炫耀自己的方式。)

### **影响力的原型**

*   “做着极其艰难和迫切需要的工作——而且常常极其枯燥。”SOC2 合规性，备份和恢复，可怕的重构，任何授权集成:如果它推动业务向前发展，他们不会在乎工作有多无聊。如果你是这位工程师，你会受到深深的尊敬和感激。
*   "**最后的调试器**。"通常是在那里工作时间最长或最初建立系统的工程师。如果你对自己的历史和背景乐于助人，这是一笔巨大的财富。(实际上，人们倾向于过分高估这个人的不可或缺性；请不要鼓励这种行为。)![Image result for engineer software meme manager](../Images/f8bbc531c0acc539886a4c0fd3b7ac14.png)

*   “**专家**原型密切相关。如果你是某个技术组件的深度主题专家，你对使用或接触该组件的任何东西都有很大的影响力。(你应该关注即将发生的变化，以保持你的优势。)
*   有些人提供了持续输出的令人困惑的强大的消防水管，有时同时在多条战线上取得进展。一些人长时间工作，另一些人对如何最大化影响有着准确无误的直觉(这有时反映在初级/高级表现上)。没有人想激怒那些人。他们的同意对…一切都至关重要。他们的参与通常会加速一个项目，或者把一个失败的努力推到终点。

并非所有的影响力都源于原始的技术实力或产出。只是众多创新/协作/人际优势中的几个:

*   一些工程师无限好奇，并且总是比其他人领先几步。他们可能看起来在玩一些毫无意义的东西，你想骂他们；然后他们把你从灾难中拯救出来。你学会珍惜他们的玩耍。
*   一些工程师通过在行业内交朋友、交换技巧、修理和恩惠来解决社会问题。不要低估**社交调试**，这往往是获得正确答案的最快路径。![](../Images/dca226b85cf4e195bb34c815628205d0.png)T3![](../Images/dca226b85cf4e195bb34c815628205d0.png)T5】
*   有些人懒得令人眼花缭乱，他们优雅的捷径和正确的捷径让你大吃一惊。
*   有些人是招聘磁铁，为了那些想和他们再次共事的人，支付他们的薪水是值得的。
*   有些人擅长在利益相关者之间达成共识。
*   有些是杀手级的解释者、教育者和故事讲述者。
*   有些人是每个人都默默想成为的高级工程师。
*   有些人可以讲述这样一个鼓舞人心的明天的故事，每个人都会跑去把它变成现实。
*   一些人通过将代码评审变成一种教学艺术形式来进行教学。
*   有些人让他们周围的人变得更有生产力和效率。有些人创造了无情的前进动力。有些人擅长说不。

有几个特殊的力量源泉，承受着这样的召唤。

*   已经成为经理的工程师价值连城。他们可以用自己的母语为初级工程师翻译业务目标，可信度无可挑剔(这是经理们从来没有过的，尤其是在初级工程师眼中)。).他们拥有强大的技术领先优势，他们可以将项目分成具有挑战性的组成部分，但不会在截止日期前压倒每个贡献者。
*   一些工程师是令人讨厌的皇家讨厌鬼，因为他们质疑![Image result for engineer software meme individual contributor](../Images/ed9eca516721b627dcad393c9ef0e614.png)

    并且挑战每一个系统和层级。但这些是锋利、强大的岩石，可以打磨出伟大的团队。尽管他们确实需要一个强有力的经理，将他们的精力用于富有成效的对话和改进，并防止他们惹恼整个团队。
*   我们也不要忘记随叫随到的工程师。如果你有一个健康的随叫随到的文化，你对生产的**所有权**创造了一个很深很深的权力和道德权威的源泉——提出要求，推动变革，分清轻重缓急。随叫随到不应该是端给那些无法拒绝的人的狗屎沙拉，它应该是每个运送代码的工程师肩负的荣誉和严肃的徽章。(它不应该是悲惨的或经常影响生活的。)

…我可以说一整天。工程是如此强大的角色和技能。绝对值得去拆包自己的影响力从何而来，去了解别人是如何感知你的长处的。

### 大多数形式的权力都可以归结为“影响力，运用”。

但是仅仅敲出代码是不够的。你可能有信誉，但拥有它并不等于使用它。要将影响力转化为力量，你必须利用它。你使用它的方式是通过交流。

你脑子里的东西对我们没有影响。你必须把它拿出来。

你可以通过很多方式做到这一点:通过写作，一对一，与小团体交谈，公开招募盟友，用明确的权威说服某人，在![Image result for engineer software meme individual contributor](../Images/3fe59677ed06548b8dd2d13edd79773b.png)

公开广播，等等。

因为工程是一项创造性的活动，**独裁权力实际上是非常脆弱和具有破坏性的**。唯一可持续的权力形式是所谓的“软实力”，如影响和激励，这就是为什么优秀的经理会自由地使用软实力，而很少/非常不情愿地使用硬实力。如果您的领导在常规上调用权威，那就是反模式。[2]

如果你不大声说话，你就没有权利坐下来为自己缺乏影响力而愤愤不平。大声说出来确实意味着在其他人面前变得脆弱，有时甚至是错误的。

### 这不是零和游戏。

你们中的大多数人拥有远比你意识到的或习惯于使用的更多的潜在力量，因为你没有感觉到强大或没有意识到你在这些方面做了什么。

经理可能有硬权力和权威，但是关于技术交付和卓越的真正实质性决定由他们最亲近的工程师做出更合适。这些理应属于实干家，很大程度上是因为他们必须承担这些决定的后果。![Image result for engineer software meme individual contributor](../Images/2305bee369eb2c68fec76035e0ea3e78.png)

权力倾向于流向经理，因为他们知道更多的信息。这使得雇佣意识到这一点并依靠这一点将权力推回他人手中的经理变得非常重要。

正如在健康的 BDSM 关系中，顺从者拥有最终权力一样，工程师实际上在健康的团队中拥有最终权力。你有最终的否决权:你可以拒绝创造。对你技能的需求很高。你通常有能力去寻找更好的条件。你们中的许多人可能应该。

当技术和管理优先级发生冲突时，谁会赢？理想情况下，你们共同努力，为企业*和*员工找到最佳解决方案。那些感觉🔥着火🔥两者之间始终保持紧密对齐。

### 选择你的战斗。

最后一个想法。如果你培养自己的影响力，并明智地使用它，你就可以对建设什么和如何建设有很大的发言权。但是你不能对每件事都有发言权。事情不是这样的。

把它想成@mcfunley 著名的“[创新令牌](http://mcfunley.com/choose-boring-technology)”，但为了关注和 fucks 给。

而一旦有了影响力，别忘了**代别人用**。注意那些没有被听到的人，放大他们的声音。付出你的时间，给予你的支持和信任，最重要的是，把那些让你变得强大的技能传授给那些需要它们的人。
