# 问数据科学家更好的问题

> 原文：<https://hbr.org/2016/11/better-questions-to-ask-your-data-scientists?utm_source=wanqu.co&utm_campaign=Wanqu+Daily&utm_medium=website>

大数据和业务的交集日益增多。虽然企业已经研究分析几十年了，但数据科学是一项相对较新的能力。在一种新的数据驱动的文化中互动可能会很困难，尤其是对于那些不是数据专家的人来说。

这些人面临的一个特殊挑战是如何向数据科学家请求新数据或分析。他们不知道该问什么样的问题，该用什么样的术语，也不知道获取所需信息需要考虑哪些因素。最后，分析师不确定该如何进行，而当经理们得到的信息不是他们想要的时，他们会感到沮丧。

在[数据孵化器](https://www.thedataincubator.com/)，我们与数百家公司合作，这些公司希望招聘数据科学家和数据工程师，或者让他们的员工参加我们的企业培训项目。我们经常回答招聘和培训客户提出的关于如何与他们的数据专家互动的问题。虽然不可能给出详尽的说明，但在与数据科学家交流时，尤其是在开始数据搜索时，需要考虑一些重要因素。

我们应该问什么问题？当你开始与数据分析师一起工作时，请明确你希望实现的目标。考虑您希望数据产生的业务影响，以及公司根据该信息采取行动的能力。通过了解您希望从他们的帮助中获得什么，数据科学家可以与您合作来定义要回答的正确问题，并更好地理解要寻找的信息。

即使是最微妙的模糊也可能产生重大影响。例如，广告经理可能会问分析师，“利用广告增加销售额的最有效方式是什么？”虽然这看起来很合理，但这可能不是正确的问题，因为大多数公司的最终目标不是增加销售额，而是实现利润最大化。来自广告从业者协会的研究表明，利用广告来降低价格敏感度的利润通常是试图增加销售额的两倍。所获得的洞察力的价值在很大程度上取决于所提的问题。尽可能具体可行。

我们需要什么数据？在您定义正确的问题和分析目标时，您和您的数据科学家应该评估数据的可用性。询问是否有人已经收集了相关数据并进行分析。不断增长的公共数据通常为常见问题提供了易于获取的答案。[医疗保健 IT 解决方案供应商 Cerner](http://www.opendata500.com/us/cerner/) ，使用美国卫生与公众服务部的数据集来补充他们自己的数据。 [iMedicare](http://www.opendata500.com/us/imedicare/) 使用来自医疗保险和医疗补助服务中心的信息来选择政策。考虑公共数据是否也可以用于解决您的问题。您还可以与组织中的其他分析人员合作，以确定内部其他人以前是否出于类似原因分析过该数据。

然后，评估可用数据是否足够。数据可能不包含回答您的问题所需的所有相关信息。它也可能受到难以识别的潜在因素的影响。想想私人借贷数据中的[年份效应](https://www.orchardplatform.com/blog/2014523vintage-analysis-how-loans-perform-with-age/):即使是看似相同的贷款，根据发放时间的不同，其表现也有很大的不同，尽管事实上它们可能在那个时候有相同的数据。这种影响来自于发行时基础承销标准的波动，这些信息通常不会在贷款数据中体现出来。

你还应该询问数据是否是无偏的，因为样本大小本身不足以保证其有效性。最后，询问数据科学家是否有足够的数据来回答问题。通过确定需要哪些信息，您可以帮助数据科学家规划未来更好的分析。

我们如何获得数据？如果需要更多信息，数据科学家必须在使用公司通过正常业务过程(如通过观察研究)收集的数据和通过实验收集新数据之间做出决定。作为与分析师交谈的一部分，询问这些选项的成本和收益。例如，观察性研究可能更容易安排，也更便宜，因为它们不需要与受试者直接互动，但它们通常远不如实验可靠，因为它们只能建立相关性，而不是因果关系。



实验允许更多的控制，并提供更多关于因果关系的可靠信息，但它们通常昂贵且难以实施。即使看似无害的实验也可能带来道德或社会影响，并带来真正的经济后果。例如，脸书[因操纵自己的新闻源](http://www.nytimes.com/2014/06/30/technology/facebook-tinkers-with-users-emotions-in-news-feed-experiment-stirring-outcry.html?_r=2)来测试情绪如何在社交媒体上传播而面临公众愤怒。尽管这些实验完全合法，但许多用户对在不知情的情况下参与脸书的实验感到不满。管理者必须超越数据思考，考虑数据收集带来的更大品牌反响，并与数据科学家合作了解这些后果。

在投入资源进行新的分析之前，验证公司能够以富有成效和有意义的方式使用从中获得的见解。这可能需要与现有技术项目集成，向自动化系统提供新数据，并建立新流程。

**数据是否干净且易于分析？**一般来说，数据有两种形式:结构化和非结构化。顾名思义，结构化数据是结构化的，很容易添加到数据库中。大多数分析师发现操纵起来更容易、更快。非结构化数据通常是自由格式的，无法轻松存储在企业中最常用的[关系数据库](https://en.wikipedia.org/wiki/Relational_database)中。据估计，非结构化数据占全球数据的 95%，[根据瑞尔森大学的 Amir Gandomi 和 Murtaza Haider 教授的报告](http://www.sciencedirect.com/science/article/pii/S0268401214001066)，对于许多大公司来说，存储和处理非结构化数据可能需要投入大量资源来提取必要的信息。在定义初始目标时，与您的数据科学家合作，评估使用非结构化数据的额外成本。

即使数据是结构化的，仍然需要清理或检查其不完整性和不准确性。如果可能，鼓励分析师首先使用干净的数据。否则，他们将不得不浪费宝贵的时间和资源来识别和纠正不准确的记录。市场研究公司 Ascend2 在 2014 年进行的一项调查发现，近 54%的受访者抱怨“缺乏数据质量/完整性”是他们最突出的障碍。通过搜索干净的数据，您可以避免重大问题和时间损失。

**模型是否过于复杂？**分析数据的统计技术和开源工具比比皆是，但简单往往是最佳选择。更复杂、更灵活的工具容易过度适应，需要更多的时间来开发。与您的数据科学家合作，确定更简单的技术和工具，只有在简单的技术和工具不足以解决问题时，才转向更复杂的模型。遵守*吻*规则很重要:“保持简单，笨蛋！”

可能无法避免与数据收集和分析相关的所有费用和问题。但是你可以采取措施降低这些成本和风险。通过向您的分析师提出正确的问题，您可以确保适当的协作，并获得自信地向前迈进所需的信息。
